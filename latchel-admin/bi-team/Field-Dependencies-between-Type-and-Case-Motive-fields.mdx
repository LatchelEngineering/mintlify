---
title: "Field Dependencies between Type and Case Motive fields"
---

<Note>
  - Cases currently have two fields that describe its content: Type and Case Motive. The Case Motive field options are based on the Type previously selected.
  - It’s important to read this document when creating or updating any of these fields.
  - Previously, instead of Case Motive, we had a field called Case Reason. Unfortunately, we couldn’t find a way to use it in the Field Dependencies settings, this is because it was a Standard object.
  - Since 06/21/2022 the Case Reason field was deactivated (information is still saved on previous Cases) and Cases have instead Case Motive.
</Note>

## How to add, hide or delete options in the Type or Case Motive field:

1. Click the Gear Icon and go to Setup.
2. Enter the Object Manager tab.
3. Enter the Case object.
4. Enter Fields & Relationships
5. Select Type or Case Motive depending on the field you need to update.

<Frame>
  ![Fields and Relationships](/images/MC-7.png)
</Frame>

## How to update field dependencies:

1. Click the Gear Icon and go to Setup.
2. Enter the Object Manager tab.
3. Enter the Case object.
4. Enter Fields & Relationships
5. On the top right corner, click the Field Dependencies button.
6. Click on Edit on the Type controlling field.

<Frame>
  ![Field Dependencies button](/images/MC-8.png)
</Frame>

7. Select or unselect the Case Motive option on the necessary Type:

<Frame>
  ![Case Motive options by Type](/images/MC-9.png)
</Frame>

8. Save your changes.
