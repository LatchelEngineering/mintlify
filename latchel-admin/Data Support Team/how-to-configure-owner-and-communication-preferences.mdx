# Objective

**Purpose**

The purpose of this SOP is to guide you through setting up owner profile and their communication preferences. Owners can choose to receive SMS and/or email communications on certain status of work orders and approval requests; 

**ATTENTION**

1. Owner communication only works if the Property Manager has a Gmail account, so follow [step 1 ](#Howto:ConfigureOwnerandCommunicationPreferences-step1)closely
2. We don't notify the Property Manager about the emails we sent; they will just show up on their outbox. Any reply to those emails will go directly to your email inbox
3. Owner approvals get logged automatically in our system

# Content

## **Step-by-Step Guide**

### **Step 1. Assess Owner Notification Customization Request**

1. If you are receiving this request at the moment the property and resident data is being added to the Latchel Console for the first time, use this SOP to upload the owner information into the Latchel Console together with property information. [How to: Add Properties in the Latchel Console to a New Account](https://latchel.atlassian.net/wiki/spaces/TST/pages/795934730)

    1. At the end of this process, owner accounts should have been created and the Owners will already show up on each property
    2. After creating the properties, resident and owner data, execute [step 2](#Howto:ConfigureOwnerandCommunicationPreferences-step2) to link the Property Manager Gmail account to our system, then skip to [step 4 ](#Howto:ConfigureOwnerandCommunicationPreferences-step4)to continue with the owner communication configuration 
    
2. If the Property Manager already has properties in the Latchel Console, but the list received in this email are new properties, you can also follow this SOP: [How to: Add Properties in the Latchel Console to a New Account](https://latchel.atlassian.net/wiki/spaces/TST/pages/795934730) to add those new properties to our system along with owner information

    1. To verify if these properties are really not in our system, you can manually research the property in the Latchel Console ([↔ How to: Navigate Properties Tab in the Latchel Console](https://latchel.atlassian.net/wiki/spaces/OPS/pages/464388136)), or use this SOP to pull a list of properties and compare with the list you have received: [How to: Handle Property and Resident Information Update](https://latchel.atlassian.net/wiki/spaces/TST/pages/796296121). (this last SOP is recommended when the Property Manager is providing a long list of properties)
    2. After creating the properties, resident and owner data, execute [step 2](#Howto:ConfigureOwnerandCommunicationPreferences-step2) to link the Property Manager Gmail account to our system, then skip to [step 4 ](#Howto:ConfigureOwnerandCommunicationPreferences-step4)to continue with the owner communication configuration 
    
3. If the Property Manager is requesting to add Owner Communication to properties already in the Latchel Console, you will move to [step 2](#Howto:ConfigureOwnerandCommunicationPreferences-step2)  to link Property Manager Gmail account to the Latchel Console and create owner profiles


### **Step 2. Work with the Property Manager to Set-up Gmail the Connection to the Latchel Console**

1. If the Property Manager doesn't have a Gmail account, let them know they will need to create one so we can use that account to send communication to owners on their behalf
2. Once the Property Manager has a Gmail account, you will ask them to set-up the connection between their Gmail account and the Latchel Console. In the same communication, you will ask which type of notifications each owner should receive

    1. In Zendesk, send an email to the Property Manager with the link to this customer guide that has the instructions [(Draft) How to: Handle Customer Success emails in Zendesk](https://latchel.atlassian.net/wiki/spaces/CS/pages/849314225)
    
        1. Use this email template: [(Archive) Email Template: Set-up Owner Communication](https://latchel.atlassian.net/wiki/spaces/CS/pages/848855062)
        2. This is the customer guide the Property Manager will use: [https://latchel.com/blog/how-to-set-up-owner-communication/](https://latchel.com/blog/how-to-set-up-owner-communication/)
        
    


### **Step 3. Create Owner Profile and Assign Property Ownership**

1. Access the Property Manager profile that has the property that needs to have an owner assigned - [How to: Access the Property Manager Profile Page](https://latchel.atlassian.net/wiki/spaces/TST/pages/813729656)
2. Go to the Properties tab and click on the property address

    1.   
    
3. In the Property Details tab, scroll down to the "Owner" section, click on the three dots to open the menu with the option to "Change Owner". Click on it.

    1.   
    
4. You will be asked to either "select and existing owner" or to "Create New Owner".

    1. If the owner is already in the system, type their name and you will see their information showing up on the search bar. Click on it 
    
        1.   
        2. Verify the information and click on "Submit"
        3.   
        
    2. If the owner is not in the system, click on "Create New Owner" 
    
        1.   
        
    3. Enter the owner name and contact information and click on "Continue"
    
        1.   
        
    4. A confirmation screen will be displayed. Verify the information and click on "Submit" 
    
        1. 
        
    


### **Step 4. Access the Owner Profile page**

1. Access the Owner profile page by going to the "People" menu and clicking on "Owners"

    1. 
    
2. Type the owner name in the search bar. Once you find the owner name in the list, click on it to open the account profile

    1. 
    


### **Step 5. Set-up the Owner Profile and Preferences**

1. In the details tab, you will be able to add critical notes, edit contact information, and set-up communication preferences. 

    1.   
    
2. Critical Notes

    1. Skip this section as we are not using it currently
    
3. Contacts

    1. If the contact information needs to be updated, click on the field you want to update, re-type the information and click on "Save"
    
        1. 
        
    
4. Contact settings - here you can configure if this owner wants to receive notifications via SMS or email or both. To set to receive that type of communication, the button should be set to "blue" 

    1. 
    2. To remove that option, toggle the button to show as grey and click on "save"
    
        1. 
        
    
5. Maintenance Communication Settings - Here you can configure when communications or notifications should be sent to owner. Notifications can be sent when a work order is created, when owner wants to pre-approve new work orders, and when owner wants to approve budget increases. If the button is showing as grey, the option is disabled, if it is showing in blue, the option is enabled. To change it, click on the button and click on save.

    1.
