## **Overview**

Enhancing the process of posting Task Share Jira tickets is essential for promoting effective collaboration and efficiency within the team. In the current Confluence document –[ Job-Aid: How The Task Share Program Works](https://latchel.atlassian.net/wiki/spaces/TST/pages/863141952/Job-Aid+How+The+Task+Share+Program+Works), it has been observed that there is room for improvement regarding the creation of tasks in Jira Software under the Task Share Project, specifically in using standardized titles listed in [Task Share Program: Standardized Task Titles.](https://latchel.atlassian.net/wiki/spaces/TST/pages/2485321730/Task+Share+Program+Standardized+Task+Titles) 

‌

## **Objectives**

* minimize miscommunication,
* ensure precise task assignments, 
* and facilitate seamless project management. 

‌

## **Outcome**

* increased productivity
* timely task completion
* Decrease ticket creation time
* Accurate quality measurement of the requests posted by the Customer Success, the Launch team, and the Client Care team.

## **The Jira Ticket**

1. Each new request posted in the [#task-share-jira-request](https://latchel.slack.com/archives/C05CGVC9FPV) Slack channel will be reviewed by the Jira project manager. The current Task Share Project Manager is <custom data-type="mention" data-id="id-2">@Malen Macasaquit</custom>.
2. Once approved, a Jira ticket will be posted in the Task Share project which will be included in the current active sprint. The approval and ticket creation must be done within 24 business hours of completing the request form.
3. All updates and progress status will be shared with the requester of the task via the comment section of the Jira ticket. 

**Note:**  Each time an update is made on the Jira ticket, _**an email notification is sent to the requester.**_  

4. For any tasks that are blocked or in NEED HELP, the requester will be notified as soon as possible. The requester will be given a maximum of 72 hours or 3 business days to respond. Otherwise, the ticket will be canceled.

## **The Jira Sprint**

	Every week the Customer Success Team, the Launch Team, and the Client Care team will be given a brief report of the sprint accomplishments and progress that consist of the following:

1. Total number of requests received in the [#task-share-jira-request](https://latchel.slack.com/archives/C05CGVC9FPV) Slack channel
2. Total number of Jira tickets prioritized
3. Total number of Jira tickets completed
4. The number of Jira tickets in the backlog with their corresponding status such as:

    1. To Do
    2. In Progress
    3. Blocked tickets and the reason
    4. For Auditing
    
5. The number of Jira tickets canceled or called off, and the reason for cancellation.

## **Schedule of Jira Ticket Request for Customers with Automated Reports**

* For **Excalibur**, please submit a Jira ticket request to Update their Properties and Residents based on their Latchel Concierge Report twice a month. 

    * **The First Jira ticket** request is to be submitted **every 1st Monday of the month**, the **deadline is on Friday of that same week.**
    
        * For example, if you submitted a request on Dec. 04, 2023, the deadline for the task will be on Dec. 08, 2023.
        
    * **The second Jira ticket** request is to be submitted **every 3rd Monday of the month**, the **deadline is on Friday of the same week.**
    
        * For example, if you submitted a request on Dec. 18, 2023, the deadline for the task will be on Dec. 22, 2023.
        
    
* For **GC Realty**, please submit a Jira ticket request to Update their Properties and Residents **every 3rd Friday of the Month**. Indicate a **deadline of Wednesday of the following week**.

    * For example, if you submitted a request on Nov. 17, 2023, the deadline for the task will be on Nov. 22, 2023.
    
* For **Chosen Property Management**, please submit a Jira ticket request **every Friday of the week**. In the request, indicate a **deadline every Tuesday of the following week**. 

    * For example, if you submitted a request on, Nov. 24, 2023, the deadline for the task will be on Nov. 28, 2023.
    
* For **DS Huber**, a request must be submitted **every last weekday of the month** which must be **completed within 3 working days of submission.**

    * For example, if the request is submitted on February 29, 2024, the deadline for the task will be on March 05, 2024 since the 2nd and 3rd of March fell on the weekend.
