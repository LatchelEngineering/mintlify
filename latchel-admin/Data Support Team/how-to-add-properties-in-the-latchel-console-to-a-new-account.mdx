# Objective

**Purpose**

The purpose of this document is to provide an overview process on how to set-up properties to a new Property Management account and provide links with detailed process for each step of the process.

# Content

**What is happening prior to this process?**

The customer has created their account in the console using an account creation link provided to them by the Sales team. 

The Sales team has also requested Property information by sending them our spreadsheet and / or requesting access to their database with that information 

# **Step-by-Step Guide**

## **Process Overview**

1. Once the Property Management account has been set-up into the Latchel Console, you will be able to start adding and configuring properties to the Property Management account.  
2. There are multiple steps involved in fully setting up properties:

    1. 
    
        1. The process starts with collecting basic information from Property Managers on the properties they service. For Property Management companies with multiple units, we should explain the concept of portfolios (see step 2 and 3 of [Customer Onboarding Process & Checklist](https://latchel.atlassian.net/wiki/spaces/CS/pages/791347310)) and work with them to group properties into Portfolios.
        
            
        
            1. The property information is usually either provided via a download from the current system the property management company uses to manage their properties, or via a tracking spreadsheet they maintain, or by filling up our spreadsheet [(located here)](https://docs.google.com/spreadsheets/d/1OLC1S4VVp755qXpbxSUZcGS3enlmLF2YwdQDRi1B7kE/edit?folder=1fPMYgh3niGJUwrAZFBP5dFtOeIkeWzlP#gid=1924383467)
            2. To provide guidance for the minimum required information we request to set-up the account, we will send Property Managers our spreadsheet showing required and optional information. However, they can also use any other format to send the required information to us. See warming box on [ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step2) for a call out on requiring property and resident information so the ops team can provide the maintenance coordination service.  
            
        2. After the spreadsheet is received, it needs to be converted into the format required by the Latchel Console import tool, which you will use to upload (create) the properties in the system (see [ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step2))
        
            1. Properties can be created in the system under the overall Property Manager account ([ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step3)), or
            2. Properties can be created in the system directly into a Portfolio ([ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step4))
            
                1. Note at the end of each section (step 3 or step 4) there are instructions on how to add/ move properties to a portfolio or remove from the port
                
            
        3. After all properties are created in the system, you will be able to go to each one of those to finalize configuration and customize.
        
    


## **Step 1 - Get Properties Information**

There are multiple ways to collect information on properties: (1) Property Managers can provide a data extract from their current system or they can provide their login information to us and we will do the data extract; (2) Property Managers can fill in the spreadsheet we provide to them (see templates in this [folder](https://drive.google.com/drive/folders/1fPMYgh3niGJUwrAZFBP5dFtOeIkeWzlP)); (3) Property Managers can provide any other spreadsheet they use to keep track of the properties they manage; (4) Data sync, i.e. we use our plug-in application to "copy" information from the Property Manager current system to Latchel Console - this is available for Appfolio only

The initial request for Service Provider information is done by the Sales team before the onboarding meeting. If this information is not received, please follow-up with the Property Management company using the template in this link: [Job-Aid: Onboarding Flow & Communications](https://latchel.atlassian.net/wiki/spaces/CS/pages/851247114)

If we are doing the data extract on their behalf, the process will depend on the system they use. Please use the appropriate SOP below to guide you on how to pull the data:

| **System** | **SOPs** |
| --- | --- |
| Appfolio | [How to: Install the Chrome Plug-in (Appfolio)](https://latchel.atlassian.net/wiki/spaces/CS/pages/813729953) |
| Buildium | [How to: Export and Import Data from Buildium](https://latchel.atlassian.net/wiki/spaces/TST/pages/795148345) |

| Rent Manager | TDB - tech is working on the integration |

**ATTENTION**

We require Property Managers to provide property address and resident information to properties we will be coordinating maintenance on their behalf. We will only not require resident information if the property is vacant or the property manager is saying it is a short-term rental (i.e. something like an Airbnb) 

Without property information, the ops team is unable to provide any type of service. Without resident information, the work order cannot be scheduled, so the workflow stops at collecting resident availability and the ops team will need to contact property management to get resident information to continue to coordinate the maintenance request. Therefore, having accurate and complete property and resident information in the system is crucial for the account creation process

To learn more about how ops team uses property information to create a work order, see the SOP below:

* [↔ How To: Create a New Work Order from stage.latchel.com/maintenance-request & Incognito Mode](https://latchel.atlassian.net/wiki/pages/createpage.action?spaceKey=LTG&title=%E2%86%94%20How%20to%3A%20Handle%20Intake%20of%20Phone%20Maintenance%20Requests%20via%20Incognito%20method)


## **Step 2** - **Prepare Properties Data for Upload**

1. Because the information on Properties can be provided to us in different formats and the Latchel Console has some requirements to enable the data upload, you will have to make some adjustments to the data to meet those requirements.

    1. Overall, the data will need to be converted into another spreadsheet showing the information of each Property on each line. This spreadsheet needs to be saved in ".csv" format. The spreadsheet does not need to have "headings" or field names, but it is recommended to use the first line to show the field name so you can use it to make sure you are matching the information correctly in the upload tool, as described in the [ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step3).
    2. Therefore, the spreadsheet should show the field name in the first line (for example: Resident Name, Property Address, City, State, ZIP, etc), and each other line will contain the information for one Property per line.
    
        1. Note that the field name in the spreadsheet does not need to exactly match the field name options you will see in the Latchel Console, but you need to make sure that the information in the subsequent lines are related to the field name, so the information populated in that section of the Service Provider profile is correct. See example below: 
        2. 
        
            Note: Phone and email formats are not enforced, so you can enter it anyway  

        
        
            
        
    
2. If the input data is coming from a system extract, you will follow the SOPs listed in [ 796131349](#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step1) to change the input data into the required format to upload into Latchel Console
3. If the input data is coming from the spreadsheet we provided (see templates in this [folder](https://drive.google.com/drive/folders/1fPMYgh3niGJUwrAZFBP5dFtOeIkeWzlP)), use the following SOP to change the input data into the required format to upload into Latchel Console — [How to: Convert our Property Information Template into "Latchel Imports" Required Format](https://latchel.atlassian.net/wiki/spaces/TST/pages/889782300)
