title: "How to do Monthly Report for Piñata"
---

<Note>
  The objective of this SOP is to provide a step-by-step guide on how to conduct the Monthly Review and prepare the Monthly Report for Piñata. It also outlines the steps for updating the Monthly Report and sending it to the stakeholders.
</Note>

<Warning>
  In **Piñata**, once a contract reaches 12 months, **<u>it auto-renews unless the property manager cancels</u>**. Monitor the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) Slack channel regularly for any “**Desire-to-Churn”** **notifications** or **cancellation notices**. Make sure to check regularly the service start date for your PMs assigned in Piñata.
</Warning>

## Step-by-step guide

<Warning>
  To prepare the Monthly Report, you should first follow the steps below to complete the Monthly Review.
</Warning>

<Accordion title="Perform the Monthly Review before preparing the Monthly Report.">
  ### **Step 1: Generate Rent Roll Report**

  a. Log in to the Property Management Software (PMS) – AppFolio. Go to Reports \> Rent Roll (Itemized).

  <Frame>
    ![](/images/CPT-475.png)
  </Frame>

  <Warning>
    **Important:** Ensure the Date is set to the last date of the month.
  </Warning>

  <Frame>
    ![](/images/CPT-476.png)
  </Frame>

  b. Select the following columns:

  - Unit
  - Status
  - Property Address
  - Tenant
  - Lease From
  - Lease To
  - Eligible for Rent Increase
  - Move-in
  - Move-out

  c. Click on **Actions \> Export as CSV**

  <Frame>
    ![](/images/CPT-477.png)
  </Frame>

  ### **Step 2. Process the Rent Roll Data**

  a. Open a new spreadsheet and apply a filter by clicking the top-left corner between column A and row 1

  b. File \> Import \> Upload the CSV file.

  c. Select Replace all data at selected cell.

  d. Ensure Unit **(Column A)** is in **Column C.**

  e. Apply Filter. Sort **Column B** (Property) A to Z.

  <Check>
    Note: _If many properties, filter twice and select all properties on the 2nd filter._
  </Check>

  f. Copy the report and paste it into the Benefits CPT Tracker (PM - GL codes).

  <Frame>
    ![](/images/CPT-478.png)
  </Frame>

  <Frame>
    ![](/images/CPT-479.png)
  </Frame>

  g. Focus on updates in **Columns J to M** of the Benefits Tracker PM’s tab (e.g., Rent-LBK GL codes).

  <Frame>
    ![](/images/CPT-480.png)
  </Frame>

  <Warning>
    **Reminders:**

    - If a property is deleted but has a GL code, take a screenshot for record keeping.
    - If a new resident has a GL code, search in the PMS to verify the GL Code and date. (Navigate to Resident Profile \> Recurring Charges Tab.)
    - If a property is marked as 'VACANT' and previously had a GL code, add a note stating 'Had a GL code (Month)' and highlight the row with color."
  </Warning>

  <Frame>
    ![](/images/CPT-481.png)
  </Frame>

  - Mark the new resident row in blue.

  <Warning>
    **For Piñata benefits:** Mark the date the GL Code was added (only when GL Code is present).
  </Warning>
</Accordion>

### **Step 1: Once you’re done with the Monthly review, proceed to download the Piñata Monthly report.**

a. In **PMS**, go to **Reports \> Report Builder (Tenant Directory)**.

<Frame>
  ![](/images/CPT-482.png)
</Frame>

<Frame>
  ![](/images/CPT-483.png)
</Frame>

b. Select the following columns:

- _Emails_
- _Phone Numbers_
- _First Name_
- _Last Name_
- _Property Street Address 1_
- _Unit_
- _Property City_
- _Property State_
- _Property Zip_
- _Lease From_
- _Lease To_
- _Rent_
- _Tenant Birthdate_
- _Past Due (from Rent Roll Itemized report)._

<Frame>
  ![](/images/CPT-484.png)
</Frame>

c. Tenants: “Show Active Tenants”

d. **Tenant Status:** Select the statuses **Current, Notice, and Evict**.

e. **Tenant Type:** Set the filter to **Financially Responsible**.

f. Properties: Leave the property filter as is (no changes required).

<Frame>
  ![](/images/CPT-485.png)
</Frame>

g. In the Filter Columns From Other Reports section, select the GL Code

h. Ensure Date = last day of the month.

i. Click **UPDATE**

<Frame>
  ![](/images/CPT-486.png)
</Frame>

j. **Click on Actions\> Export as CSV**.

### **Step 2: Process Piñata report Data**

a. Open a **new spreadsheet**.

<Frame>
  ![](/images/CPT-487.png)
</Frame>

b. **Import and upload** the CSV file \> replace data in the selected cell \> Import Data.

<Frame>
  ![](/images/CPT-488.png)
</Frame>

c. Arrange **columns in the correct order**.

- _Email_
- _Phone number_
- _First name_
- _Last name_
- _Shipping address_
- _Shipping address 2_
- _City_
- _State_
- _Zip_
- _Lease from_
- _Lease to_
- _Rent amount_
- **_New column for the Rent Due Day of Month_**
- _Past Due Balance_
- _DOB_

d. Add a **column (L onwards)** for **Rent Due Day of the Month** (manual entry based on previous month).

e. **Reapply filters** (they reset after rearranging columns).

f. Sort **Column C (Unit)** **A to Z**.

g. **Filter Column P** to show tenants with **GL Codes**. (select all numbers with Values)

<Frame>
  ![](/images/CPT-489.png)
</Frame>

h. Select CTRL \+ H to search for the Mobile, Home, and Phone options to remove in the Phone number column, and delete them.

<Frame>
  ![](/images/CPT-490.png)
</Frame>

<Frame>
  ![](/images/CPT-491.png)
</Frame>

<Frame>
  ![](/images/CPT-492.png)
</Frame>

i. All amounts in the Past Due column displaying this icon (–) are \$0.

<Frame>
  ![image-20250731-173000.png](/images/CPT-493.png)
</Frame>

j. Replace any blank cells caused by missing information with 'NA'.

<Frame>
  ![image-20250731-173446.png](/images/CPT-494.png)
</Frame>

k. Copy filtered data into the **Benefits CPT Tracker PM Piñata**.

l. Add the Rent Due Day Of Month for the column you created.

<Check>
  **Example:** _If you’re doing the report on July 02nd, it will contain the information of June, so the date in the Rent due day of the month column should be 06/01/2025._
</Check>

<Frame>
  ![](/images/CPT-495.png)
</Frame>

_m. Open the Piñata Report Tab: Go to the **Piñata Report** tab in the Benefits Tracker._

_n. Copy Data from PMS Report: From the updated PMS report, copy the required columns._

<Frame>
  ![](/images/CPT-496.png)
</Frame>

_o. Paste Data on the Left Side: Paste the copied information on the left side of the Piñata Report tab **(Column V)**, following the existing format._

<Frame>
  ![image-20250807-235100.png](/images/CPT-497.png)
</Frame>

### **Step 3: Open the Monthly Report**

a. Open the Monthly report from the last email sent.

<Frame>
  ![](/images/CPT-498.png)
</Frame>

b. Duplicate the tab (bottom of sheet) and rename it for the new month.

c. Right-click on the tab, select 'Duplicate,' and rename the duplicated tab with the corresponding date and month of your review, following the required format.

<Frame>
  ![](/images/CPT-499.png)
</Frame>

d. Copy data from Benefits CPT Tracker (Piñata tab for the PM), then paste data into the Piñata Monthly Report.

e. Copy from the Tenant email to the unit column and paste it in the report’s tab. Then repeat the process to copy from the Property city till the DOB columns, respecting the merge column in the report.

<Frame>
  ![](/images/CPT-500.png)
</Frame>

### **Step 4: Unit Count & Validation**

a. In Column J (Merge column) of the Piñata report:

- Apply the formula to count units being charged.
- Copy formula from previous month.
- Drag the formula down to all rows.

b. Total at bottom of report – Ensure it matches the GL codes total (For the Month to be reported) in the PMs tab.

<Danger>
  **Note:** If the total number is different, please check the properties one by one to identify the discrepancies. And add the missing properties in the Monthly report.
</Danger>

<Frame>
  ![](/images/CPT-501.png)
</Frame>

### **Step 5: Update the Billing Inputs Doc**

a. Go to Billing Input Spreadsheet

b. Update Column Q with the new counts of Piñata Enrollments

<Frame>
  ![](/images/CPT-502.png)
</Frame>

### **Step 6: Send the Report.**

a. Draft email using the previous template

- Recipients:  Check the “To” and “CC” refer to the photos below. If this is the first time you are sending the report, you should request the CSM for the POC information to send the report.
- Subject: **PM Company Name \+ Piñata Rent Roll Update**

<Frame>
  ![](/images/CPT-503.png)
</Frame>

b. Click the “drive” Icon

<Frame>
  ![image-20250730-195929.png](/images/CPT-504.png)
</Frame>

c. Search for the Piñata report by typing the name of the Property Manager (PM) and the report title. Example: New Age Realty Piñata Rent Roll Report.

<Warning>
  Attach the report and first verify that it is the correct file by clicking on it before sending. If the file is incorrect, make sure to unselect it and attach the correct one.
</Warning>

<Frame>
  ![image-20250730-193109.png](/images/CPT-505.png)
</Frame>

d. Click on **send**.

For newly onboarded PMs for Piñata, create a dedicated tab for the new PM. Please follow the steps outlined in this job aid: [How to Create a Piñata Tab (AppFolio Customers)](https://latchel.atlassian.net/wiki/spaces/AL/pages/3585114139/How+to+Create+a+Pi+ata+Tab+Appfolio+Customers).
