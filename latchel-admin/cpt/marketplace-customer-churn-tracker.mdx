<Note>
The purpose of this SOP is to provide a clear overview of the dashboard that gathers all churn events from Marketplace Benefit partners, enabling the CPT to track risk and record results in one place.
</Note>

<Check>
This tracker is **<u>only used</u>**<u> when property managers under </u>**<u>Marketplace benefits</u>**<u> are flagged in </u>[<u>#service-changes</u>](https://latchel.slack.com/archives/CKA2W1KJA)<u> with a </u>_**<u>Desire to Churn alert</u>**_. The tracker centralizes and monitors churn status across Piñata, Filters, and Coverpest (as well as future partners).
</Check>

## Overview

**<u>Dashboard structure</u>**

Each tab within the churn tracking spreadsheet is organized in a structured table format to ensure consistent monitoring, communication, and documentation of churn-related activities. This table helps stakeholders quickly understand the status of each account flagged for churn and provides transparency across teams.

<Frame>
  <img src="/images/CPT-777.png" alt="image-20250512-193056.png" />
</Frame>

| **Column:** | **Description: ** |
| --- | --- |
| **a) **Date | Day the “Desire to churn” workflow was triggered |
| **b) **Customer Name | Property-manager company flagged for churn |
| **c) **Deal Name | HubSpot deal ID / name |
| **d) **Churn Notification Link | Permalink to the Slack workflow |
| **e) **Churn Status | Reflects the current stage or outcome of the churn process. _<u>This field is manually updated based on the latest activity or feedback shared in the Slack thread or internal discussions with the CSM</u>_. Maintaining accurate status updates is key to ensuring alignment across teams and timely follow-ups. Examples include: - CSM follow-up meeting in process - Pending CSM updates - Churn Prevented – Customer Staying Onboard - CSM follow-up completed — client still canceled |
| **f) **Cancellation Alert Sent | A simple Yes/No dropdown indicating whether a formal cancellation notification has been sent to the appropriate partnership contact. This helps track communication progress and prevent missed handoffs. |
| **g) **Email/Slack Link — Cancellation Notification | Permalink to the partner email or Slack post |

Each partner (Piñata, Filters, Obie, Coverpest) has its own tab with the same structure.

For more reference, see the SOP guide [How: the Marketplace Customer Churn tracker process works](https://latchel.atlassian.net/wiki/spaces/AL/pages/3472982023/How+the+Marketplace+Customer+Churn+tracker+process+works)
