<Note>
The Objective of this resource is to provide all the details on everything we need from Property Managers with Rent Manager PMS. This resource is for the CSM to understand what access level is required for the Customer Profitability Team to be able to have the visibility needed to deliver insight and visibility to Stakeholders and to track what is needed for the Assisted Ledgering contract, Filters, Piñata and Optional Benefits.
</Note>

In **Rent Manager**, the User Role for Latchel purposes should be something similar to **Administrator**, for the CP team to have access to financial reports. <u>This access level ensures the CP team can do weekly, monthly reviews, and monthly scorecards, which gather accurate information essential for the benefit and success of the PM contract acquired with Latchel</u>. **If the Property Manager is not willing** to customize the User Role assigned to Latchel **to enable access to all the features the CP team needs**, <u>the CS team should set the right expectations that they must send the General Ledger report monthly, otherwise, they will be billed a full invoice amount.</u>

The PM may want to edit the user role to restrict certain sensible information, however, this should keep access to the Rent Roll Report, General Ledger Report, Property, Tenant, and Ledger Menu.

<Check>
For more information on how to add User Roles and assign them to a user in Rent Manager, please refer to this article: [https://rmxhelp.rentmanager.com/Topics/Express/CSH/User-Roles.htm?Highlight=administration%20user](https://rmxhelp.rentmanager.com/Topics/Express/CSH/User-Roles.htm?Highlight=administration+user)
</Check>
