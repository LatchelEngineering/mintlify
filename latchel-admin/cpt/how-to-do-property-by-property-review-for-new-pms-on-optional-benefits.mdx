<Note>
  The purpose of this guide is to provide the step by step on how to identify the Optional benefits added on the resident’s account.
</Note>

## Step-by-step

<Check>
  Once the rent roll is uploaded, the next step is to track the residents who have Optional Benefits added. To identify them, a property-by-property review must be completed before the Property Manager’s service date begins.
</Check>

For new PMs added in Optional Benefits, you need to do a Property by property review, once the Rent Roll report is added for the first time, to make sure all the residents were advised about the Optional Benefits Package, which of them opted in/out, and which ones have a charge.

### Step 1: Open the Optional Benefits tab of the PM you’ll be reviewing

a. Look for the tab of the PM you need to review. Once you locate it, select the name of the resident and look for his/her name in the corresponding PMS.

### Step 2: Check the resident information in the PMS

a. Once you find the resident’s profile, you need to check the following information:

<Frame>
  ![type of lease.jpg](/images/CPT-510.jpg)
</Frame>

<Frame>
  ![tyoe of lease b.jpg](/images/CPT-511.jpg)

b. **The type of lease:**

- **Residential Lease:** When a resident is under a residential contract, you will see it in this section stating the full date. Example: _LEASE END DATE 02/28/2025._
- Monthly Lease: When a resident is under a monthly contract, you will see it in this section stating the type of contract. Example: _LEASE END DATE Monthly._

c. Notes Section:

<Frame>
  ![notes section.jpg](/images/CPT-512.jpg)
</Frame>

**Notes:** In this section, you need to look for any note that would state the resident opted out of the resident’s benefits package.

d. **Text Messages Section:**

<Frame>
  ![text.jpg](/images/CPT-513.jpg)
</Frame>

**Text:** In this section, you need to look for any message that would state the resident opted out of the resident’s benefits package.

<Check>
  Make sure to look for all the messages sent close to the date when the PM started with the Optional Benefits contract.
</Check>

e. **Emails Section:**

<Frame>
  ![resb.jpg](/images/CPT-514.jpg)
</Frame>

Emails: In this section, you need to look for any email that the PM team sent over to the resident advising about the Optional Benefits Package.

<Check>
  In this same section, you can check if the resident replied to the PM.
</Check>

f. Letters Section:

<Frame>
  ![aqd.jpg](/images/CPT-515.jpg)
</Frame>

**Letters:** In this section, you need to look for any letter that the PM team sent over to the resident advising about the Optional Benefits Package.

<Check>
  In the emails and letters section, you will usually find a type of notification the PM team sent. This is a type of addendum that includes the charge amount, period when it will begin, the benefits included, and the option for the resident to opt-out by replying to the notification sent.
</Check>

g. **Recurring Charges Section:**

<Frame>
  ![aqz.jpg](/images/CPT-517.jpg)

**Recurring Charges:** In this section, you need to look for the GL code being added for the Optional Benefits Package.

<Warning>
  **Section 8, HOA, and Commercial units** are not supposed to have a GL code for Optional Benefits. However, we can make an exception if you encounter any of these cases with a GL code. Make sure to take note of this and report it to your TL.
</Warning>

h. **Lease Information Section:**

<Frame>
  ![wqe.jpg](/images/CPT-518.jpg)
</Frame>

**Lease Documents:** In this section, you will be able to find the leases available. Make sure to look for the current lease at the time you are checking the resident’s information. Make sure to check the addendum was added advising the resident about the Optional Benefits Package. The resident must have signed the document as a consent of the agreement.

<Frame>
  ![image-20240328-180917.png](/images/CPT-519.png)
</Frame>

<Warning>
  **<u>Most residents would opt out even after signing the lease and addendum, however this is not a practice that should be allowed</u>**. This is why it's very important to check the Notes, SMS, Emails, and Letters section to make sure the resident has not opted-out. If you find any of these cases make sure to report it to your TL.
</Warning>

If no lease is found in this section, you can also look at the **Files Section** that is at the bottom of the resident’s profile.

### Step 3: Classify the resident in the PMs tab

- After you gather all this information. Look for the tab of the PM you need to review and fill the row with the resident’s information you were checking accordingly.

<Frame>
  ![image-20240328-185404.png](/images/CPT-521.png)
</Frame>

a. The first column you will need to fill in is the **Opt-In column**:

<Frame>
  ![updated DS Huber.jpg](/images/CPT-522.jpg)
</Frame>

Make sure to select the correct option according to the information gathered:

**Yes -** We select this option when the resident has a signed lease and addendum as a sign of the agreement.

**No -** We select this option when the resident has responded to any of the notifications sent, either SMS, Email or Letter and has stated he/she want to opt out from the package.

No answer from the resident - This is the most common option you would probably select. We select this option when, after the resident has been advised via SMS, Email or Letter, no answer has been received. **NOTE:** This does not apply if the addendum is signed.

Vacant - As the option states it, this applies to vacant units only.

HOA - As the option states, it applies to all properties under the Home owners association.

**Notification not sent - This option applies to all the residents that for some reason, the PM did not inform them in any way about the Optional Benefits Package**.

b. In the next section, you will need to fill in the information about the resident’s lease:

<Frame>
  ![image-20240328-193011.png](/images/CPT-523.png)
</Frame>

Type: In this column, you will need to select the option according to the lease type of the resident.

**Future Tenant -** We select this option, for new residents that will begin their lease in a future date.

**Just Renew -** This option is selected if the resident has already finished the period of their previous lease and has renewed a new one, or the lease has ended, but there is no lease on file, however the resident’s profile is still active.

**Monthly -** We select this option for residents who are in a monthly lease. If the resident is a transfer from another PM company, make sure to note it.

New Resident - We select this option for new residents who just began their lease with the PM company.

c. Has Lease: For this option, we select Yes/No.

d. Lease sign date:  **Type the date when the lease was signed.** If No was selected in the previous column, just type **NA** = Not applicable.

e. Has future lease: This option only applies if there is a lease already added for a future date. Please **select Yes/No.**

f. Lease start date: This option only applies if there is a lease already added for a future date.

g. Lease sign date: This option only applies if there is a lease already added for a future date that is already signed. **<u>If there is a future lease added but not signed yet, please type: Out for signing.</u>** 

h. Did the resident Opted in?: Only fill this column if there is a future lease already signed in which the resident agreed on the addendum for the Optional benefits.

<Frame>
  ![ds huber up1.jpg](/images/CPT-524.jpg)
</Frame>

In this section, you will need to fill in the information related to the notification sent to the resident regarding the Optional Benefits Package.

<Frame>
  ![ds huber up2.jpg](/images/CPT-525.jpg)
</Frame>

i. Notification sent to the resident: For this option, please select **Yes/No**.

j. Notification sent through: For this option, please select any of the options below:

1. Email
2. Letter
3. Chat
4. Email and Letter
5. NA: _This option is only  selected when **No** was selected in the previous column._
6. Addendum added on the lease
7. Email, letter, addendum

k. **Notification sent date:** For this option, type the date when the notification was originally sent. _Note: If the addendum was also included, type the date of the notifications sent via Email, SMS and letter along with the addendum signed date._

l. Was the charge added: For this option, please select **Yes/No**.

m. Benefits added on: **For this option, type the date when the GL code was added to the resident’s profile.** 

Note: If the charge was not added due to the resident opting out, please type the date when the resident rejected the optional benefits package.

In the last section, you will need to fill in the GL code charge information.

<Frame>
  ![ben.jpg](/images/CPT-526.jpg)
</Frame>

n. Did the resident Opted-out?: For this option, please select **Yes/No**. 

o. **If so, when?:** Only fill out this option with the opt-out date in case you select **Yes** in the previous column. 

p. Has GL code but no Opt in/Opt out document or Opt-in agreement?:  For this option, please select **Yes/No**. 

q. **Notes:** Fill in all additional information that requires the PM's’s attention. **Example**: _Section 8. Might be a commercial unit, need to check with the CSM. Etc._

<Check>
  The same process will be repeated for all the residents in the list, **<u>even If it’s not showed in the Rent Roll they have a GL Code.</u>**
</Check>
