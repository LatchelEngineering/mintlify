---
title: "How to do the Monthly Review for Optional Benefits"
---

<Note>
  The purpose of this SOP is to provide a step-by-step guide on how to perform a Monthly Review for residents in AppFolio.
</Note>

<Warning>
  In **Optional Benefits**, **once their contract reaches the 12-month period, it will end**; you should stop doing their review. **<u>Instead, you will follow the steps of this SOP:</u>** [How to: Do the Monthly Review in Appfolio for Clients Whose Optional Benefits Contract Has Ended](https://latchel.atlassian.net/wiki/spaces/AL/pages/3574170015/How+to+Do+the+Monthly+Review+in+AppFolio+for+Clients+Whose+Optional+Benefits+Contract+Has+Ended)   **<u>if the PM had GL codes added</u>**. Make sure to check regularly the **service start date** for your PMs assigned to Optional Benefits.
</Warning>

## Step-by-step guide

### **Step 1: Download the Rent Roll Itemized report**

a. Log in to AppFolio.

b. Navigate to Reports \> Rent Roll (Itemized).

<Frame>
  ![](/images/CPT-528.png)
</Frame>

### **Step 2: Filter and Update the Rent Roll Report**

a. Search for the GL Code

- You can find the GL code for the Property Manager (PM) in the [**CPT Tracker**](https://docs.google.com/spreadsheets/d/1bQ45AVpZG5-ONwnSLFlJP8M5GCl7J__VczN1rfeCFQI/edit?gid=0#gid=0), under the **Optional Benefits Overall Details** tab.

<Frame>
  ![image-20250730-201400.png](/images/CPT-529.png)
</Frame>

b. Set the Filters

- **Units:** Select **"Show Active Units"**
- Search for the GL code  (If the GL code does not appear in the PMS, please download the report without selecting it.)
- **As of Date:** Enter the date of your review

c. Click "Update" to generate the filtered report.

<Check>
  **Note:** _Select the last day of the previous month as the reporting date. Since the Optional Benefits review is done once a month, the date should be the last day of the previous month to report. Example: It’s July 03rd, you will download the report to do the review of June. Please select 06/30/2025._
</Check>

<Frame>
  ![](/images/CPT-530.png)
</Frame>

### **Step 3: Customize and Export the Rent Roll Report**

a. Click on Customize Columns and ensure the following fields are selected:

- _Unit_
- _Status_
- _Property Address_
- _Tenant_
- _Lease From_
- _Lease To_
- _Eligible For Rent Increase_
- _Move-in_
- _Move-out_

b. Once customized, go to Actions and click Export as CSV.

<Check>
  Note: _Always verify the unit count (found at the right bottom of the report) before proceeding to ensure accuracy._
</Check>

<Frame>
  ![](/images/CPT-531.png)
</Frame>

### **Step 4: Set Up the Spreadsheet**

a. Open a new spreadsheet (Google Sheets).

b. Click the blank cell in the upper-left corner (between Column A and Row 1) to select the entire sheet.

c. Click the Filter icon (or go to Data \> Create a Filter) to enable filters for all columns.

<Frame>
  ![](/images/CPT-532.png)
</Frame>

### **Step 5: Import the Exported Rent Roll File**

a. In your new spreadsheet, go to **File \> Import**.

b. Click the **Upload** tab. c. Click **Browse** and select the **CSV file** you exported from PMS.

d. When prompted, choose **"Replace data at selected cell"**.

e. Click **Import data** to load the data into your spreadsheet.

<Frame>
  ![](/images/CPT-533.png)
</Frame>

### **Step 6: Rearrange Columns**

a. Move **Column A (Unit) to Column C** to match the required format.

<Frame>
  ![](/images/CPT-534.png)
</Frame>

### **Step 7: Sort and Verify the Data**

a. Click the Filter icon on the column headers.

b. Sort the data A to Z to organize the properties.

<Warning>
  **Note:** If the report contains more than 300 properties, click Column B, right-click, and select "Sort sheet A to Z".
</Warning>

c. Verify that the unit counts match the PMS report for accuracy.

<Frame>
  ![](/images/CPT-535.png)
</Frame>

### **Step 8: Paste Data into the CPT Tracker**

a. In the current Rent Roll sheet, copy **Columns A to J starting from Row 2 (A2 to J2)** down to the last row, ensuring the number of properties matches the count in the PMS report. (If no GL code column was found in the report, ensure to add a column next to the tenant column before you copy-paste the information from the report in the PMs tab in the CPT Tracker)

b. Open the CPT Tracker and go to the tab of the PM you are reviewing. Paste the copied data into the sheet starting at Column AI to AR, using CTRL \+ SHIFT \+ V to paste without formatting.

<Warning>
  **Note:** _Make sure not to click anywhere without first checking whether any properties need to be deleted at the end of the newly pasted report._
</Warning>

<Frame>
  ![](/images/CPT-536.png)
</Frame>

### **Step 9: Compare Current and Previous Reports**

a. Open the **CPT Tracker 2025** tracker and navigate to the tab corresponding to the PM you are currently reviewing.

b. Focus on Columns L to P (with "NO").

c. Compare the data with the newly pasted PMS report.

d. Identify any changes or status updates in columns **L to P**

<Frame>
  ![](/images/CPT-537.png)
</Frame>

"Opt In” **Column K -** verify in AppFolio whether the resident has been notified regarding the Resident Benefits. 

**If the resident signed a lease with an addendum**, we can consider that an Opt-in. **If the resident specifically stated he/she doesn’t want to pay for Optional benefits**, we consider that as an opt-out. 

**If the lease with the addendum isn’t signed and there’s no evidence the resident opted out**, then we will consider that as No answer from the resident.

e. Check notes, SMS/text, emails, letters, and Lease information.

<Frame>
  ![](/images/CPT-538.png)
</Frame>

<Frame>
  ![](/images/CPT-539.png)
</Frame>

<Frame>
  ![](/images/CPT-540.png)
</Frame>

**Note:**

If the property is **VACANT**, kindly select 'Vacant' in “Opt In” Column K.

- **Column Q** select “vacant”
- **Column R to T, V to X, AA, AC, AE, AG**\_ leave blank\_
- Columns U, Y, AB, AD, AF select _“NO”_
- Column Z _“NA”_

If the property is commercial, kindly select 'commercial' in “Opt In” Column K.

- **Column Q** select “commercial”
- **Column R to T, V to X, AA, AC, AE, AG**\_ leave blank\_
- Columns U, Y, AB, AD, AF select _“NO”_
- Column Z _“NA”_

If the property is HOA (Check LEASE in PMS), kindly select 'HOA' in “Opt In” Column K.

- **Column Q** select “HOA”
- **Column R to T, V to X, AA, AC, AE, AG**\_ leave blank\_
- Columns U, Y, AB, AD, AF select _“NO”_
- Column Z _“NA”_

f. If a PDF is available in the Lease Documents tab, please select the document accordingly .If a PDF is not available in the Lease Documents tab, please use the 'View in Browser' link to access the detailed lease information.

g. Search for 'Latchel' or any related resident benefits. Please note that in some cases, the Property Manager may have already included the resident amenity package addendum in the lease by default.

<Frame>
  ![](/images/CPT-541.png)
</Frame>

h. Go back to the PMs tab and look at the columns L-P to note any cells in NO marked in red. These will indicate the following:

_"Is Tenant Same” **Column L** - Check **Column AL** to determine if there is a new resident or if the property has recently become vacant._

_“Is  GL code same” **Column O - Check Column AM** to check if the GL code has been removed or added_

“Is same address” Column P -  Check **Column AJ** to determine if the property address has been changed or removed. If the property in **Row 2 (Column AI)** now appears in **Row 3 (Column AI)**, this may indicate that a new property has been added.

“Is Lease to same” - Check **Column AO** to verify if the lease end date has been changed. If there is an update, reflect the change in **Column G** accordingly.

- If a new property has been **ADDED**, copy the data from **Columns AI to AR**. Select the entire row above where the latest property should be inserted, right-click, and choose 'Insert Row Below.' Paste the copied data into the newly added row using CTRL\+SHIFT\+V. Next, enter the new address in Column A of that row. Then, select Columns AI to AR, right-click, and choose 'Delete Cells' \> 'Delete Cells and Shift Up'.
- If a property has been DELETED, when you see “NO” in **columns L–P** and the row is red, check if the property is missing from the new report. If it is, select columns **AQ–AZ** of the mismatched row in the new report, right-click, and choose "Insert cells" then "Shift down" to align the data. Then, right-click the row of the deleted **resident in the old report and choose "Delete row." After that, go back to columns L–P and drag the formula down to update the comparison. Once corrected, the row should turn green and display “YES.”**

<Check>
  **Note**: Always repopulate the formulas in **Columns L to P** to ensure the formulas are correctly updated throughout the report.
</Check>

<Frame>
  ![](/images/CPT-542.png)
</Frame>

<Warning>
  **Note: Before deleting,** if the Property Management removed a property from the PMS report you just pasted, please take a screenshot and save it for reference and count it to report it in the Billing inputs doc at the end of the month. To restore the formula, click on\*\* Columns L to P \*\*(do not drag your mouse across the sheet).
</Warning>

### **Step 10:  Update Property Details**

a. Update the **Type** _(**Column Q**)_ accordingly.

- **Signed before the Onboarding Date** of the PM _(The Onboarding Date is in the CPT Tracker (Optional Benefits Overall Details Tab, Column P)_

<Frame>
  ![](/images/CPT-543.png)
</Frame>

- Check the **lease document** to confirm the **date the resident signed the lease**. \_

<Frame>
  ![](/images/CPT-544.png)
</Frame>

- **HOA / Section 8** - To determine if a unit is classified under HOA or Section 8, open the lease document, press CTRL \+ F, and search for keywords such as "HOA" or "Section 8".
- **Monthly**- check the PMS (AppFolio). This information is located at the top-right corner of the resident’s profile page.
- New Resident - A tenant who recently moved in and started a new lease.
- Just Renew - An existing tenant who recently renewed their lease agreement.

<Frame>
  ![](/images/CPT-545.png)
</Frame>

b. Update Column R: (Has a lease)

- YES - check the lease on the PMS (If the lease is not showing on the “Lease Information tab, go to the “Attachments” tab

<Frame>
  ![](/images/CPT-546.png)
</Frame>

<Frame>
  ![](/images/CPT-547.png)
</Frame>

c. Next, proceed to **Column U** ("Has Future Lease") to check if a future lease exists. If applicable, complete **Columns V to X** accordingly.

d. Update **Column Y** (Notification sent to Resident) if YES

- Update **Column Z** (Notification sent through) to Column AG accordingly.
- Check for any communication records, such as **Email, Text, Letter, or Addendum** by reviewing the corresponding tabs in the PMS.
- If no notification was sent to the resident, please complete the columns as shown in the reference image below.

<Frame>
  ![](/images/CPT-548.png)
</Frame>

### Step 11: Update Review Date in Overall Details Tab

a. Once you’ve completed your review, **please update** the 'Optional Benefits Overall Details' tab with **the corresponding date of your review**.

b. Update active units in column T (**Only for PMs with GL codes every 1st of the month**).

<Frame>
  ![](/images/CPT-549.png)
</Frame>

<Warning>
  For **Urban Coast**, update the **active units (Column T)** until you finish the OB Review **only every 1st of the month**, as the active units total will be the same as the one reported and sent to the Data team each month. **<u>This applies to any other PM who has GL codes</u>**, if no GL codes are added the active units should be the same as in this [SOP](https://latchel.atlassian.net/wiki/spaces/AL/pages/3574497370/How+to+Update+Active+Units).
</Warning>

### **Step 12: Escalate Properties with GL codes as of the last day of the previous month to the Data Team and report GL codes Totals to Finance.**

a. Follow the [**SOP**](https://latchel.atlassian.net/wiki/spaces/AL/pages/3119808527/How+to+Request+activation+deactivation+of+properties+in+Latchel+console+for+OB+Customers) to send the review to the Data Team

b. Report to Finance in the [Billing inputs doc](https://docs.google.com/spreadsheets/d/1HG_bRF-JM8DqSz1lusDrfJVGIpohvJwBP4fNAipclvM/edit?gid=250560413#gid=250560413) the total number of GL codes as of last month.

<Note>
  **Example:** It’s July 03rd, you did the review for June, <u>you will report in the Billing inputs doc the amount of GL codes for June, even if the GL code was deleted</u>. You will also report deleted properties to the data team.
</Note>

<Check>
  For active property managers in Optional Benefits with GL codes, ensure you save a copy of the report escalated to the Data Team by accessing the appropriate folder in the [**CPT Drive**](https://drive.google.com/drive/u/1/folders/14gX1LldAU2iGekZF4NzAI2RW7fnl5Dtc). In the _Active Units CPT Review Internal_ doc, duplicate the previous month’s tab and customize it to include the totals reported to the Data Team.
</Check>

<Warning>
  **🔔 Reminders for Monthly Review Process** <u>New PMs must conduct a property-to-property review before their service date starts.</u>

  - If the lease is new with a future date: - Only fill out Columns U to X
  - If the lease is new but has already started: - Fill out all relevant details from Column K to the end
  - For vacant units: - Mark Column K and Column Q with the option “Vacant” - Column R should only be answered with Yes or No - Leave blank if the unit is vacant (do not select any option) - Always leave notes if a GL code was deleted. Example: It’s July 03rd and you’re doing the review for June. You need to leave a note stating: Had GL code in June. - If a property was deleted and had a GL code, ensure to take a screenshot and save it for reference. Make sure you add that property for the data team report.
</Warning>

The onboarding and Review Process for New PMs (Optional Benefits) begins with a property-by-property review once the Rent Roll report is added for the first time. Please refer to the following job aid: [How to do: Property by property review for new PMs on Optional Benefits](https://latchel.atlassian.net/wiki/spaces/AL/pages/2953609279/How+to+do+Property+by+property+review+for+new+PMs+on+Optional+Benefits)
