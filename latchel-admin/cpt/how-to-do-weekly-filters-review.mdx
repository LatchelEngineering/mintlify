---
title: "How to do Weekly Filters Review"
---

<Note>
The objective of the Weekly Filters Review is to keep the Benefits CPT Tracker accurate and up to date by reviewing active GL codes, identifying changes, and updating notes to ensure correct data for monthly reporting.
</Note>

<Warning>
In **Air Filters**, once a contract reaches 12 months, **<u>it auto-renews unless the property manager cancels</u>**. Monitor the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) Slack channel regularly for any “**Desire-to-Churn”** **notifications** or **cancellation notices**. Make sure to check regularly the service start date for your PMs assigned in Filters.
</Warning>

## Step-by-step guide

### **Step 1: Open the PM tab you’ll review. **

a. Go to the<u> </u>[<u>Benefits CPT Tracker</u>](https://docs.google.com/spreadsheets/d/1qd4oXlu6dBYwvPCVnl8fMwoulezF2MTZQoGOhqPqdo0/edit?pli=1&gid=0#gid=0)** **and open two tabs:
 - Main Tab: GL Codes

- Second Tab: (PM company name) AU

<Frame>
  <img src="/images/CPT-442.png" alt="" />
</Frame>

### **Step 2: How to Generate Rent Roll in PMS.**

a. Log in to the Property Management Software (PMS) – AppFolio.

<Frame>
  <img src="/images/CPT-443.png" alt="" />
</Frame>

b. Navigate to the top menu and click on **Reporting**.

c. Select **Reports** from the dropdown menu.

d. Click on **Rent Roll (Itemized)** to open the report.

<Frame>
  <img src="/images/CPT-444.png" alt="" />
</Frame>

### **Step 3: Filter and Update the Rent Roll Report.**

a. Search for the GL Code

b. You can find the GL code for the Property Manager (PM) in the **Benefits Tracker **under the **Filters Overview **tab.

c. Set the Filters

d. **Units:** Select **"Show Active Units"**

e.** As of Date:** Enter the date of your review. Since this is a weekly review, you must update the date to the current date on which you are doing the review. For the monthly review, ensure to set the date as of the last day of the month. _**Example:** It’s July 03rd, you will do the last review for June, please select 06/30/2025._

f. Click** "Update"** to generate the filtered report.

<Frame>
  <img src="/images/CPT-445.png" alt="" />
</Frame>

g. Ensure the following 9 columns are selected:

- Unit
- Status
- Property Address
- Tenant
- Lease From
- Lease To
- Eligible For Rent Increase
- Move-in
- Move-out

h.** Click on actions >** Export as **CSV**

<Frame>
  <img src="/images/CPT-446.png" alt="" />
</Frame>

### **Step 4: Prepare the Spreadsheet**

a. Open a **new spreadsheet**

b. Click the top-left corner to **select all** → **Enable Filter**

<Frame>
  <img src="/images/CPT-447.png" alt="" />
</Frame>

c. In your new spreadsheet, go to **File > Import**.

d. Click the **Upload** tab.

e. Click **Browse** and select the **CSV file** you exported from PMS.

f. When prompted, choose **"Replace data at selected cell"**.

g. Click **Import data** to load the data into your spreadsheet.

<Frame>
  <img src="/images/CPT-448.png" alt="" />
</Frame>

h. Move **Column A (Unit) to Column C** to match the required format.

<Frame>
  <img src="/images/CPT-449.png" alt="" />
</Frame>

<Check>
**Note:**_ If the report contains more than 300 properties, click Column B, right-click, and select "Sort sheet A to Z"._
</Check>

<Frame>
  <img src="/images/CPT-450.png" alt="" />
</Frame>

i. Select and copy all necessary data from the report.

j. Navigate to the _Benefits CPT Tracker_ and paste the copied data into the appropriate tab (PM company name + GL codes) and section (From **Column AC or AC depending on the tab of the PM**)

<Check>
** **_**Note: **Use **CTRL+SHIFT+V** to paste values only, ensuring formatting is not disrupted. Make sure not to click anywhere without first checking whether any properties need to be deleted at the end of the newly pasted report._
</Check>

k. Double-check that all data has been correctly pasted and aligned with the correct columns.

### **Step 5: Compare Current and Previous Reports**

a. Focus on** Columns** **J to M** (with "NO") to identify the properties that need to be changed.

b. Compare the data with the newly pasted PMS report.

<Frame>
  <img src="/images/CPT-451.png" alt="" />
</Frame>

### **Step 6:  Review the type of property**

a. If a Property is marked as  Vacant in the new report but had a GL Code in the previous report side:

- Leave a note in **Column AA. **_**Example: **You are doing the weekly review on July 15th, and the GL code was deleted. You will leave a note stating: Had GL code in July. _
- **Delete the date** in **Column O**
- Highlight the row in **blue** (from Column A to H)

b. If there is a New Resident with a GL code:

- Search for the resident in PMS
- Confirm **GL Code and Date**
- Update the tracker accordingly

<Frame>
  <img src="/images/CPT-452.png" alt="" />
</Frame>

c.** Check for GL Code**

- In the _Monthly Charges_ tab, verify if a GL code is present for the resident or property.

d. **Enter the GL Code Date**

- If a GL code is listed, input the date it was added in **Column O** of the _Benefits CPT Tracker_.

<Frame>
  <img src="/images/CPT-453.png" alt="" />
</Frame>

f. Type (Column N): You can check this information in PMS

- Residential
- Monthly
- Commercial

g. Check if the resident has a future-dated GL code in the PMS under the Monthly Charges tab.

- Has Future GL code (**Column X**): Date of Future GL code (**Column Y**)
- Add Notes (**Column AA**) if needed

For newly onboarded PMs for Air Filters, create a dedicated tab for the new PM. Please follow the steps outlined in this job aid: [How to Create a Filter Tab (Appfolio Customers)](https://latchel.atlassian.net/wiki/spaces/AL/pages/3585015876/How+to+Create+a+Filter+Tab+Appfolio+Customers)
