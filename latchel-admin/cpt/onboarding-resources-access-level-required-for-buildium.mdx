<Note>
  The Objective of this resource is to provide all the details on everything we need from Property Managers with Buildium PMS. This resource is for the CSM to understand what access level is required for the Customer Profitability Team to be able to have the visibility needed to deliver insight and visibility to Stakeholders and to track what is needed for the Assisted Ledgering contract, Filters, Piñata and Optional Benefits.
</Note>

In Buildium, the Property Manager User Role provides the access the CP team needs to conduct the processes required for keeping a precise record of the Property Managers' Road to Profitability. <u>This access level ensures the CP team can do weekly, monthly reviews, and monthly scorecards, which gather accurate information essential for the benefit and success of the PM contract acquired with Latchel.</u> However, Buildium provides many default user roles, which can be edited to reflect what areas of the software the PM would like members to have access to. **If the Property Manager is not willing** to customize the User Role assigned to Latchel **to enable access to all the features the CP team needs**, <u>the CS team should set the right expectations that they must send the General Ledger report monthly, otherwise, they will be billed a full invoice amount.</u>

The PM can also add a new user role if none of the default options fit his needs for a certain member. He can customize it according to his needs taking into consideration the information we require to have access to

<Check>
  For more information on how to add User Roles and assign them to a user in Buildium, please refer to this article: [BuildiumHelpCenter](https://help.buildium.com/hc/s/article/User-Roles-and-staff-member-access-1557495331958)
</Check>
