<Note>
  The purpose of this document is to answer some general and specific questions that can arise in different aspects of the Customer Profitability tasks.
</Note>

<Accordion title="Addendums">
  **1. What information must an Addendum have to consider it an addendum or ledgerable property?**

  No specific requirement is needed. As long as it mentions that there will be an additional charge for a resident benefit package (or similar) is enough for it to be considered an addendum.

  **2. When an HOA or section 8 has an addendum, do we consider that property ledgerable? Or do we need to report this as a courtesy to the PM so they can remove it?**

  - HOA: If an HOA has an addendum and GL code there is no further action needed. We will take it as the PM determining they want that property ledgered.
  - **Section 8**: If a Section 8 has an addendum and a GL code there will be no further action needed. We will take it as the PM determining they want that property ledgered.

  <Warning>
    - **Note:** We can report these scenarios as a courtesy to CSMs so they can check with the PM, however, no action item is needed from CSMs or PMs. As the CPT we do want to have a record of these cases for data input.
  </Warning>

  **3. Do we count as an addendum if the information is provided by an email notification instead of included in the lease?** If the email is sent in the middle of a lease notifying the resident of the resident benefit package charge we will count that as an addendum for now and keep a record of these cases for data input. However, we would expect the PM to add this information to the lease in the case that renewal occurs for the resident. If we notice that for the renewal the addendum is not added, we will report this to CSMs as an action item for PMs, and keep a record of scenarios for data input.

  **4. Does the lease need to be signed for it to be valid or the property to be able to be ledgerable?** Ideally yes. If there is an addendum or lease language with the signature of the lease we consider the addendum signed. This goes as well for any attachments to the lease. If it is sent as an email we will not focus on auditing if it is signed or not but instead just keep an eye out for instances similar to question number 3.

  **5. Does the addendum need to mention Latchel?** No.

  **6. If the addendum is not complete or included in the lease can we ledger the property? What happens if this is the case (there’s a GL code) but there is no addendum?** We will continue to report to CSMs when we have cases where there is a GL code but no addendum/lease language was added as an action item for PMs. However, as stated in question number 1 as long as there is a mention of an additional charge for the resident benefit package (or something similar) we will consider that as an addendum.

  The property can be ledgerable even if it doesn’t have an addendum.

  <Check>
    Even if the property seems to be non-ledgerable (Section 8, HOA or signed before Latchel contract), if the PM adds a GL code we will consider the property as ledgerable to the PM and will mark the property as ledgerable in the CPT tracker and fill in everything else accordingly but leaving a specific note.
  </Check>
</Accordion>
