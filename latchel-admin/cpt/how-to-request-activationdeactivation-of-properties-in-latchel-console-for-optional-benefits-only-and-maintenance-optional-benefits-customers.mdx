---
title: "How to: Request activation/deactivation of properties in Latchel console for Optional Benefits Only and Maintenance + Optional Benefits Customers"
---

<Note>
  The purpose of this article is to be a guide on how to send the request to #task-share-jira-request  channel to active/deactivate properties in Latchel console each month, for customers enrolled in **Optional Benefits** or **Maintenance \+ Optional Benefits**.
</Note>

<Check>
  After doing the monthly review for customers enrolled in Optional Benefits, the CPT will send a request in the \*\*#data-support-requests Channel, \*\*so all properties with a GL code for the month being reported (previous month) can show as \*\*Active \*\*in Latchel console, while all the properties that are no longer included in the list provided by the CPT need to be **deactivated**.
</Check>

## Step by step

### Step 1: Create the Active units CPT Review report

a. After the monthly review, you will create a spreadsheet where all the properties that need to be active  in Latchel console are listed. You may use this [spreadsheet](https://docs.google.com/spreadsheets/d/1HMZgLdqamr_VhFq4pIJ-EFZe6m10gXznb4kmES4RNPg/edit?gid=2101808614#gid=2101808614) as a template. Name the spreadsheet “Active Units CPT Review”, name the tab with the PM Company’s name and the month being reported. (You may duplicate the tab for each consecutive month). In <u>Column D,</u> change the month to the one being reported.

<Frame>
  ![20.png](/images/CPT-527.png)
</Frame>

Keep in mind that if a GL code was deleted during the month you are reporting, but this was still charged to the resident in that month, the property will remain in the list to stay active in Latchel console. The property can be removed the next month if a GL code is not added.

<Check>
  CPT will also create a similar spreadsheet to the one used to report the active properties, but for internal purposes. You may use this [spreadsheet](https://docs.google.com/spreadsheets/d/1eMgZwx9KQYfI4YtT0uRuhtp0ybJ7yBh_eqLUeoowwbM/edit?gid=267413095#gid=267413095)as a template. This will include an extra column “Is the property currently active in the PMS?”. A folder for each Property Manager will be created in this [folder](https://drive.google.com/drive/u/1/folders/14gX1LldAU2iGekZF4NzAI2RW7fnl5Dtc), to include the internal spreadsheet.
</Check>

### Step 2: Go to the Data Support Requests Channel

<Warning>
  **<u>If the customer has already ended their contract in Optional Benefits and had GL codes</u>**<u>, you should still submit the Typeform following the same steps</u>, except for the following question where you should **type this answer**:

  **Provide special instructions here \>** Customer had Optional Benefits. Although the contract has ended, please proceed with the same process.
</Warning>

a. After creating the report, go to [data-support-requests](https://latchel.slack.com/archives/C05CGVC9FPV) Channel, and use the workflow Account Update Request Typeform, fill in all the information as follows:

- **Is the customer disputing for subscription charges?** \> No
- **Is this an Onboarding request?** \> No
- **Requesters Information** Please provide your information so we can reach out to you if needed. Requester's Name: Type your name Email address: Type your Latchel email address
- **Company Details**
- **PM Company Name:** Type the PM Company Name
- **Company URL in Latchel:** Include the link of the Properties section in the PM’s profile in Latchel console.
- **Does the customer have an active third-party integration?** \> Yes
- **Identify the request category** \> Properties
- Property update request \> M. Optional Benefits Only (For CPT Use Only) N. Maintenance \+ Optional Benefits (For CPT use Only)
- **Do you also need to update the residents of the properties?** \> Yes
- **Identify resident update required** \> Update Status (Activate/Deactivate)
- **Provide special instructions here** \> _<u>This is for Optional Benefits Only,</u>_ or _<u>This is Maintenance \+ Optional Benefits.</u>_
- **Share the PM spreadsheet files here** \> Include the spreadsheet link (Active units CPT review)
- **Set your request priority** \> High
- **Set a completion deadline** \> Select the 7th business day of the month

<Check>
  If you have any questions, you can watch the following [**video**](https://drive.google.com/file/d/1uErr7Hd9aXg-pU3_PhAuDFXkDwjhEEXe/view?usp=drive_link) on how to submit the Typeform.
</Check>

b. After completing all the information as advised, click on **Submit.**
