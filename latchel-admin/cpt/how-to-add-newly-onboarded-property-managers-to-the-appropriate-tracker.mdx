---
title: "How to add newly onboarded Property Managers to the appropriate tracker"
---

<Note>
  The purpose of this SOP is to provide a step-by-step guide on how to add newly onboarded property managers to the tracker. It also highlights the information required from them before their service start date.
</Note>

## Step by step

<Check>
  It's always best to also check the PM’s profile in **HubSpot** to have full visibility of their information. See the following job aid for reference: [How to Use Hubspot](https://help.latchel.com/latchel-admin/cpt/how-to-use-hubspot)
</Check>

### Step 1: Review the Sales & CPT channel

a. Every time a new property manager signs up for any type of benefit handled by the Customer Profitability Department, a notification will appear in the Sales and CPT channel in Slack.

<Frame>
  ![image-20250813-184020.png](/images/CPT-395.png)
</Frame>

b. To learn what type of benefit they signed up for, review the Benefits section of the notification.

<Frame>
  ![image-20250813-184116.png](/images/CPT-396.png)
</Frame>

### Step 2: Add the Property Manager to the Correct Tracker

<Accordion title="Assisted Ledgering and Optional Benefits">
  <Check>
    Any property manager enrolling in any of these benefits should be added to the [**CPT tracker**](https://docs.google.com/spreadsheets/d/1bQ45AVpZG5-ONwnSLFlJP8M5GCl7J__VczN1rfeCFQI/edit?gid=0#gid=0) in the corresponding tab.
  </Check>

  Make sure to add the following information in the tracker:

  - PM company Name
  - CSM
  - Software
  - Price per unit \+ Product rate (**For example**: _if a PM has Optional Benefits and Piñata, you must add the rate of each product (found in the Benefits Selection section of the PM profile in HubSpot) to the price per unit_)
  - Service start date
  - Contracted units
  - Resident Charge amount (Found only in the PM’s profile in HubSpot)
  - GL code (If found in HubSpot, it will be under the Onboarding Information section in the Resident Charge Name/Number field.)

  <Warning>
    The PMS URL, username, and password information will be included once we receive the account activation link in the Assisted Ledgering email.
  </Warning>

  <Check>
    **Drag down the formulas for all other columns** so profitability can be calculated. Before that, ensure you create the PMs tab by following the appropriate job aid. [How to Create an Assisted Ledgering Tab (Appfolio Customers)](https://help.latchel.com/latchel-admin/cpt/how-to-create-an-assisted-ledgering-tab-appfolio-customers)  and [How to Create an Optional Benefits Tab (Appfolio Customers)](https://help.latchel.com/latchel-admin/cpt/how-to-create-an-optional-benefits-tab-appfolio-customers).
  </Check>

  <Danger>
    If the new PM will be using a PMS other than AppFolio, you should create a new tab and adapt the process to that PMS report. You can check the Appfolio, Buildium, Propertyware, Rent Manager, RentVine and DoorLoop sections under the [Assisted Ledgering](https://help.latchel.com/latchel-admin/cpt/assisted-ledgering) contract section for reference.
  </Danger>
</Accordion>

<Accordion title="Filters, Piñata and Coverpest">
  <Check>
    Any property manager enrolling in any of these benefits should be added to the [**Benefits CPT Tracker**](https://docs.google.com/spreadsheets/d/1qd4oXlu6dBYwvPCVnl8fMwoulezF2MTZQoGOhqPqdo0/edit?pli=1&gid=675879001#gid=675879001) in the corresponding tab.
  </Check>

  Make sure to add the following information in the tracker:

  - PM company Name
  - CSM
  - Software
  - Service start date
  - GL code (If found in HubSpot, it will be under the Onboarding Information section in the Resident Charge Name/Number field.)

  <Warning>
    The PMS URL, username, and password information will be included once we receive the account activation link in the Assisted Ledgering email.
  </Warning>
</Accordion>

### Step 3: Ensure You Have Everything You Need Before the Service Start Date

<Check>
  Ensure you have everything you need, **<u>ideally 15 days before the service start date</u>** of the newly onboarded property manager.
</Check>

For all property managers, regardless of the benefit type they signed up for, we need to ensure we have access to the following:

1. **PMS Information:**

- URL
- Username
- Password

1. **Reports:**

- Rent roll (itemized)
- General ledger

1. **GL code information**

<Warning>
  **For Filters**, you should also ensure you have the Filters size list. If you cannot find an attachment in HubSpot or their Drive folder, escalate the issue to the CSM
</Warning>

### Step 4: Follow-Ups for Missing Information

<Accordion title="Missing GL code information">
  <Check>
    If you’re missing the GL code information for any benefit, create a post requesting the CSM to provide this information.
  </Check>

  a.Go to the [#ledgering-mkt-opt-benefits](https://latchel.slack.com/archives/C03D4N7B0LF) and click in the workflows section located at the top of the channel.

  b. Then select Request from CS Help

  <Frame>
    ![image-20250813-191557.png](/images/CPT-397.png)
  </Frame>

  c. Fill in the workflow with the following information:

  <Frame>
    ![image-20250813-191841.png](/images/CPT-398.png)
  </Frame>

  Use the following template:

  > _Hi \[CSM Name\], I hope you're doing well. Could you please confirm the GL code for \[PM Company Name\]? We’ll need it as soon as possible to ensure everything is ready for the upcoming review. Thank you in advance for your assistance!_

  <Check>
    For the due date, please set it to one week from the day you’re escalating the issue.
  </Check>
</Accordion>

<Accordion title="Missing GL Report">
  <Check>
    If you’re missing the GL report for any benefit, create a post requesting the CSM to request access to the PM.
  </Check>

  a.Go to the [#ledgering-mkt-opt-benefits](https://latchel.slack.com/archives/C03D4N7B0LF) and click in the workflows section located at the top of the channel.

  b. Then select Request from CS Help

  <Frame>
    ![image-20250813-191557.png](/images/CPT-399.png)
  </Frame>

  c. Fill in the workflow with the following information:

  <Frame>
    ![image-20250813-191841.png](/images/CPT-400.png)
  </Frame>

  Use the following template:

  > _Hi \[CSM Name\], We just received the account activation link for the PM. I noticed that we currently do not have access to the GL report. Could you kindly request it from the PM using the following template below. Thank you!_

  > _Hi \[PM Name\],_ _We hope you’re doing well. We received the access‑update invitation and have now set everything up to track the \[Insert benefit type\]. However, we’ve noticed there’s one report we still can’t access. Could you please update our user permissions so we can view the General Ledger report under the Accounting section in AppFolio? Below is a step‑by‑step guide you can follow (all actions require an admin/president‑level login)._
  >
  > **_Steps to Grant General Ledger Access_**
  >
  > 1. In AppFolio, search for “Manage User Roles and Permissions” and select the matching result.
  > 2. Locate the Property Manager role (Latchel user), click the ellipsis on the right, and choose Edit
  > 3. In the permissions screen, find the options for financial reports—specifically General Ledger—and set them to Full Access.
  > 4. Click Review Changes, then Save to apply the updates.Once this is done, we’ll be able to access the General Ledger report.
  >
  > _Thank you very much for your help!_ _Please let us know if you have any questions._
  >
  > _Best regards,_
  >
  > _The Latchel Team_

  <Check>
    For the due date, please set it to one week from the day you’re escalating the issue.
  </Check>
</Accordion>

  </Check>

  b. Then select Request from CS Help

  <Frame>
    ![image-20250813-191557.png](/images/CPT-401.png)
  </Frame>

  c. Fill in the workflow with the following information:

Could you please follow up with the property manager? Their service starts on \[Service start date\], and we’d like to have everything ready by then since they’ll be enrolled in \[Benefit type\]._

  <Check>

After adding the new PM to the appropriate tracker, update the ‘Real Time Monthly Scorecard’ tab in the CPT Tracker with the PM’s details. Please refer to the following job aid: [How to Update the Real-Time Monthly Scorecard Tab with Newly Onboarded PMs](https://help.latchel.com/latchel-admin/cpt/how-to-update-the-real-time-monthly-scorecard-tab-with-newly-onboarded-pms)
