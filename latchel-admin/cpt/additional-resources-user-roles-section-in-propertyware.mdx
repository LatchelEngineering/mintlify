<Accordion title="Types of Users">
# Types of Users

Users have different types of privileges, they can create, edit, deactivate, and delete reports within a Propertyware account.

To see the different types of users follow the steps below:

1. Click **Setup** and expand **Administration Setup**. 
<Frame>
  <img src="/images/CPT-253.jpg" alt="" />
</Frame>

<Frame>
  <img src="/images/CPT-254.jpg" alt="" />
</Frame>

2. Expand **Manage Users**, and then click **User Profiles**.

<Frame>
  <img src="/images/CPT-255.png" alt="" />
</Frame>

1. The Profiles Menu will display, showing the different types of roles.

<Frame>
  <img src="/images/CPT-256.png" alt="" />
</Frame>

1. **Accounting User: **This user role can create/edit/delete custom report folders, schedule reports, administrative reports, contact reports, desktop reports, finance/financial reports, lease reports, maintenance reports, marketing reports, property reports statements. This user role doesn’t have access to screening privileges.
2. **Administrative Profile: **This user role has the same permissions as the Accounting User, with the exception that this role can access screening privileges. People with Administrator privileges can create, edit, deactivate, and delete users within a Propertyware account.
3. **Leasing Agent: **This user role has access to read only view.
4. **Maintenance User: **This user role has main access to work orders for maintenance coordination and also has permission to create, edit, delete and custom the same reports as in the Accounting User profile.
5. **Marketing User: **This user role has access to contact reports, desktop reports, lease reports, maintenance reports, marketing reports and property reports.
6. **Property Manager:** This user role can publish reports and schedule reports. It also has access to contact reports, desktop reports, finance/financial reports, lease reports, maintenance reports, marketing reports, and property reports statements.

1. You can view each profile, edit, or assign a role to a new account. You can also create a new profile.

<Accordion title="User Roles Permissions/Restrictions">
# User Roles Permissions/Restrictions

Every user in Propertyware is assigned a user profile, which restricts the data and functions accessible to its assigned people. With user profiles, a person with Administrator privileges can ensure that a group of people has the same level of access to the system. Each member must be assigned a profile, and there is no limit to the number of people that can be assigned to the same profile.

**Every profile is broken into three sections:**

1. _**Module Settings**: _These settings dictate what tabs people assigned to this profile have access to, as well as what functions within that tab they can use.

<Frame>
  <img src="/images/CPT-258.png" alt="" />
</Frame>

**a.**  **Profile Name:** User profile's name. The name relates to the role of the people assigned to the profile, such as “Property Manager”.

<Frame>
  <img src="/images/CPT-259.png" alt="" />
</Frame>

**b. Module Settings:** Indicates what tabs and activities in those tabs the profile can perform.

1. _**Read Only Settings:** _These settings dictate whether the person has “Full Access” or “Read Only” access to each particular tab. “Full Access” allows the person access based on the Module Settings as set above. “Read Only” access limits people to viewing the pages they have access to, but prevents them from making any changes regardless of permissions selected for the module above.

<Frame>
  <img src="/images/CPT-260.png" alt="" />
</Frame>

### **Settings:**

a. **Desktop: **Set to “Full Access” to allow the person to make calendar events, record or edit tasks, enter or delete conversations, and sync calendar events to Google or Outlook.

b. **Properties:** Set to “Full Access” to allow the person to create, edit, and delete portfolios, buildings, and units.

c. **Leases Summary:** Set to “Full Access” to allow the person to create, edit, and delete leases.

d. **Leases Ledger: **Set to “Full Access” to allow the person to create, edit, and delete tenant transactions.

e. **Maintenance:** Set to “Full Access” to allow the person to create, edit, and delete work orders and service requests, to record time tracking entries, and to create bills and bill payments for work orders.

f. **Contacts:** Set to “Full Access” to allow the person to create, edit, and delete contacts, enable or disable portal access, and set contact categories.

g. **Money In: **Set to “Full Access” to allow the person to create, edit, and delete income transactions for tenants, vendors, or other payers.

h. **Money Out: **Set to “Full Access” to allow the person to print checks, generate vendor 1096s and 1099s, and create, edit, and delete bills, bill payments, credits, and checks for vendors.

i. **Banking:** Set to “Full Access” to allow the person to open, edit, and close bank deposits, create checks, reconcile the bank accounts, and enter budget information.

j. **Management: **Set to “Full Access” to allow the person to bill management fees, issue owner statements, pay owner draws, and generate owner 1096s and 1099s.

k. **Marketing:** Set to “Full Access” to allow the person to publish or unpublish listings, edit marketing information for buildings and units, track marketing campaigns, and create, edit, and delete prospects.

l. **Reports: **Set to “Full Access” to allow the person to copy, import, publish, schedule, and update reports.

m. **Portals: **Set to “Full Access” to allow the person to edit the tenant and owner portal settings.

n. **Sites:** Set to “Full Access” to allow the person to set the domain name of your site, edit the listing widgets, and create, edit, and delete pages from your site.

o: **Dashboards:** Set to “Full Access” to allow the person to create, edit, and delete dashboards.

p. **Inspections:** Set to “Full Access” to allow the person to create, edit, and delete inspections.

1. _**Screening Privileges:** _These settings dictate what access a person has to the screening actions and details of prospects.

<Frame>
  <img src="/images/CPT-261.png" alt="" />
</Frame>

### **Settings:**

a. **Run Screenings:** Allows users to run screenings for prospects or leases. Regardless of this setting, if the screening profile at **Setup > Administration Setup > Screening Profiles** is not set up to screen prospects, the person cannot screen prospects. Additionally, if the screening profile is not set up to screen leases, the person cannot screen leases.

b. **View Detail Reports:** Allows users to view the Detail Reports of screenings run for prospects or leases.
