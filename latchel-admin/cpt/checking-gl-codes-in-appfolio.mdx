title: "Checking GL Codes in Appfolio"
---

<Note>
  The purpose of this document is to provide a guide on how to navigate the Appfolio tool to be able to track the GL codes for each new added property into the tracker.
</Note>

## Step-by-step

### **Step 1 - Locate and confirm the correct tenant's profile**

a. Log in to the PM’s Appfolio account.

b. Use the main search bar to look for the tenant by name.

c. Click on the tenant’s name in the search results, ensuring two things:

- The property address matches the one you are investigating.
- The status next to the tenant’s name says **(Current)**.

<Frame>
  ![](/images/CPT-68.png)
</Frame>

### **Step 2 - Inspect the recurring charges for the Latchel GL Code**

a. On the tenant's profile page, scroll down to the **"Recurring Charges"** section.

b. Carefully review the list of charges to find the Latchel-specific charge.

c. Confirm the following details if the charge is present:

- The **amount** being charged.
- The **start date** of the charge.
- If there is an **expiration date** listed.

d. This section is your definitive source of truth. If the code is not here, it is not being charged.

<Frame>
  ![](/images/CPT-69.png)
</Frame>

### **Step 3 - Review lease documents for supplemental information (Optional)**

a. If you need additional context or are troubleshooting, navigate to the **"Attachments"** or **"Lease"** section of the tenant's profile.

b. Open the lease document and look for any mention of Latchel services or a signed lease addendum.

<Warning>
  The presence of a lease addendum does **not** guarantee the GL code has been added to recurring charges. This step should only be used for investigation. Always rely on Step 2 as the final confirmation.
</Warning>
