title: "How to do Monthly Filters Review"
---

<Note>
  The objective of this SOP is to reconcile and update all properties with GL codes, ensure accurate filter counts, and deliver a Monthly Filters Report to the Stakeholders.
</Note>

<Warning>
  In **Air Filters**, once a contract reaches 12 months, **<u>it auto-renews unless the property manager cancels</u>**. Monitor the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) Slack channel regularly for any “**Desire-to-Churn”** **notifications** or **cancellation notices**. Make sure to check regularly the service start date for your PMs assigned in Filters.
</Warning>

## Step-by-step guide

<Warning>
  _The weekly review for Filters must always be completed first before proceeding with the Monthly Filter Review._
</Warning>

### Step 1: Search for the previous Monthly report sent.

a. Open Assisted Ledgering Gmail

b. Search for the PM's previous monthly filter report

c. Duplicate the last report tab and update the date

<Frame>
  ![](/images/CPT-454.png)
</Frame>

### **Step 2. Update the Monthly Report.**

<Warning>
  Between the 1st and the 8th business day of each month, we will conduct the monthly Filters reconciliation process. This process determines the total number of GL codes the PM company had during the month we will report. **Example:** It’s July 04th, and you’ll report the information for June.
</Warning>

a. Go to the **Benefits CPT Tracker**

b. Filter the properties with GL codes from the Benefits Tracker, then search for each property in the duplicated Monthly Report to verify whether it is already included. This will help identify any newly added properties that need to be reflected in the Monthly Report. To easily identify new properties to be added to the report, you can mark them with a color.

<Note>
  **Example**: 123 Almond road is marked in blue, which means this is either a new property that needs to be added in the report, or the property had a GL code during the previous month, and it should be reported in this month.
</Note>

<Frame>
  ![](/images/CPT-455.png)
</Frame>

<Frame>
  ![](/images/CPT-456.png)
</Frame>

c. Similarly, for the Notes column **(Column AA)**, filter the entries that indicate a property had a GL code (e.g., 'Had a GL code in July'). Then, search for each of these properties in the duplicated Monthly Report to confirm whether they are already included. This will help identify any properties that had GL code and should be included in this month’s report.

<Frame>
  ![](/images/CPT-457.png)
</Frame>

d. Cross-check the Monthly report with the AU tab and the GL Codes tab for accuracy.

e. This ensures the monthly filter list sent to the PM is always up to date.

<Check>
  **Note:**

  - Delete rows for properties with no GL Code (In the Monthly report) unless they had a GL code in the month you’re reporting.
  - _Update the filter size details as needed. If there are any changes, you can verify the updated filter size(s) in the Benefits Tracker under the AU tab._
</Check>

### **Step 4. Billing Count.**

a. Count filter sizes in the new report you have prepared; if any filter size is missing, that should be counted as 1 filter.

b. Go to **Billing Input Sheet**

c. Select the corresponding **month tab**

d. Search the PM’s name.

e. Update **Column P** with the filter count.

<Frame>
  ![](/images/CPT-458.png)
</Frame>

### **Step 5. Sending Report to PM.**

### **Final Steps:**

a. Draft email using the previous template

- Recipients:  Check the “To” and “CC” refer to the photos below. If this is the first time you are sending the report, you should request the CSM for the POC information to send the report.
- Subject: **PM Company Name \+ Latchel Filters Monthly Update**

  ![](/images/CPT-459.png)
</Frame>

<Frame>

b. Attach file via Google Drive (Make sure the report attached is the same you were updating, for this, click on the report attached **before** you send it)
