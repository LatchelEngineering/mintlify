<Accordion title="User Role General Details">
  # **User Role General Details**

  Once a Rent Manager user account is created, you can view and manage additional information regarding the user's account, such as contact information, from the **General** tab. Certain privileges, such as for property and bank access, can also be managed directly from the **General** tab.

  <Frame>
    ![](/images/CPT-289.jpg)
  </Frame>

  By default, the tiles below are available to view and edit on the **General** tab for every user.

  1. **Personal Information:** This tile displays basic information about this user and their active status.

  <Frame>
    ![](/images/CPT-290.png)
  </Frame>

  a. Is Active: **By default, this is checked, as it allows the user to log into Rent Manager. Once an account is no longer being used, uncheck to make this user inactive. An inactive user’s past actions are retained for historical purposes, but no one is able to log in using this username while it is inactive.** 

  **b.** First Name: The user’s first name. 

  c. Last Name: The user’s last name. 

  d. Username: The username used to log in and track the employee's activity throughout Rent Manager. 

  e. Password: The password used to log in to Rent Manager. 

  f. Supervisor: If applicable, select the user’s supervisor from the drop-down list of other users who are assigned to manage assigned users and privileges in the _System privilege group_. The supervisor controls the access of a limited list of other users for which they are selected as supervisor.

  <Danger>
    When creating a password, some restrictions may apply, and Rent Manager provides recommendations for a strong password. If a user forgets their **Password**, you can change it here to provide them access to Rent Manager.
  </Danger>

  2. Contact Information: This tile displays this user's primary phone number, fax number, and email address.

  <Frame>
    ![](/images/CPT-291.png)
  </Frame>

  a. SMS/Text Ready: **Check if the phone number is able to receive text messages. This is necessary for the phone number to be included in a text broadcast.** 

  **b.** Phone: The user’s primary phone number. 

  c. Fax: The user’s fax number. 

  d. Email: The user’s primary email address.

  **If you use per-unit Rent Manager licensing, an email address for each user is required. Setup instructions are sent to this address.**

  3. Options: This tile displays additional assignments that can be applied to the user's account.

  <Frame>
    ![](/images/CPT-292.png)
  </Frame>

  The following fields are available on the tile:

  a. Read-Only: Restricts a user to read-only access. The user can see information but cannot change the data. 

  b. Administrator: Enables the user as an administrator account. Rent Manager administrators have the ability to manage other Rent Manager user accounts that have limited privilege access. Administrators are automatically granted new privileges when privileges are added to the system in future updates. An administrator is not automatically granted all privileges. Privileges must still be specified for each administrator. 

  c. Sales Rep / Leasing Agent: Enable the user to create sales estimates (for projects and/or products and services) and identify requirements (parts and labor) to complete a project, itemize the costs, and provide an extended total amount due based on any additional markups and/or taxable items. This option also allows the user to be selected as the Leasing Agent for a prospect, which can be tracked in reports such as the Box Score report. 

  d. Inspector: This allows the user to perform field inspections on a smartphone or tablet. Users with the appropriate privileges may also set up and manage inspection information in the program. 

  e. rmServices User: Provides this user access to the Service feature, a licensed feature that pulls live issue data from Rent Manager and loads those issues in rmAppSuite Pro. 

  f. Mobile Web Access: Provides this user access to mobileRM. This web portal allows the user to view and manage information in the Rent Manager database from any web-enabled device like a smartphone or tablet. 

  g. API Access: Provides this user access to the open Application Programming Interface (API). This is a licensed feature that allows developers to write programs for you that communicate directly with your database. 

  h. Owner Sales Rep: Enables the user to be selected as the Sales Rep for an owner prospect, which can be tracked in reports such as the Owner Prospect Box Score report.

  <Danger>
    IMPORTANT: An **Administrator** assignment should be _very limited_. A user with this assignment has the potential to change _any_ user in the system, as well as create new users (including a new one for themselves) with _all_ privileges and access to _all_ reports.
  </Danger>
</Accordion>

<Accordion title="How to edit the User Roles Menu">
  # **How to edit the User Roles Menu**

  You can make all the changes you need to customize a profile in the User Role Menu. In this section, we will go over the different section you can find here and the information you can customize.

  <Frame>
    ![](/images/CPT-293.jpg)
  </Frame>

  The privileges assigned to a user via user roles are denoted by a check mark   on the Privileges tab for the User details page and User Roles page.

  When a user is assigned a role, you cannot remove any privileges that make up that role from the **Privileges** tab. You may, however, add or remove privileges not associated with the role for individual users by going to Admin Setup **Users**.

  1. **General Tab**

  a. The _Role Information_ tile displays the **Role Name**, which you may view or edit.

  <Frame>
    ![](/images/CPT-294.jpg)
  </Frame>

  b. The _Users_ tile allows you to remove users from the user role.

  <Frame>
    ![](/images/CPT-295.png)
  </Frame>

  To remove users from this user role, select the desired Username and **click Delete**. The privileges that were granted to this user by way of assigning this user role are then removed.

  **2. Privileges Tab**

  a. The Privileges tab displays privilege groups, individual privileges, and the permission types (**Enabled**, **Add**, **View**, **Edit**, **Delete**) associated with each privilege.

  <Frame>
    ![](/images/CPT-296.png)
  </Frame>

  b. By default, the **Privileges** tab opens with each privilege group collapsed into a summary row. To expand or collapse all privileges, click  **Expand** or **Collapse**. The boxes in the row indicate the status of each permission type for the group:

  <Frame>
    ![](/images/CPT-297.png)
  </Frame>

  To assign privileges to the user role, do the following:

  1. Optionally, check or uncheck the box to the right of expand in the top left of the privileges list to select all or deselect all privileges.
  2. Click a group arrow to expand the group into its individual privileges.
  3. Check or uncheck privilege boxes to grant access or deny access to the privilege.
  4. Select additional privilege groups as needed, and assign privileges to the user role.
  5. Click Save to accept your changes for that user role. Privileges are added or removed from all the users assigned to the role.

  **3. Reports Tab**

  a. The user's Reports tab is where access to Rent Manager reports and custom Report Writer reports can be assigned to a specified user role. Reports are displayed in groups, and you are able to assign an entire group of reports or specific reports from a group to a user role.

  <Frame>
    ![](/images/CPT-298.png)
  </Frame>

  b. By default, the **Reports** tab opens with each report group collapsed and a box that summarizes access to the group’s reports:

  <Frame>
    ![](/images/CPT-299.png)
  </Frame>

  To grant a user role access to reports, do the following:

  1. Optionally, check or uncheck the box to the right of  Expand in the top left of the reports list to select all or deselect all reports.
  2. Optionally, check a report group to grant access to all reports in that group.
  3. Click a group arrow to expand the group into its individual reports.
  4. Check or uncheck report boxes to grant access or deny access to each report.
  5. Click Save to accept your changes for that user role. Privileges to reports are added or removed from all the users assigned to the role.

  **4. Letters Tab**

  a. _The User's Letters tab_ is where access to Letter templates, and document packets can be assigned to a user role. Letters and packets are displayed in groups, and you are able to assign an entire group of letters or specific letters from a group to a user role.

  <Frame>
    ![](/images/CPT-300.png)
  </Frame>

  b. By default, the **Letters** tab opens with each entity group collapsed and a box that summarizes access to the group’s letters and packets:

  <Frame>
    ![](/images/CPT-301.png)
  </Frame>

  To grant a user role access to letters and/or packets, do the following:

  1. Check or uncheck the box to the right of expand in the top left of the letters and/or packets list to select all or deselect all letters and/or packets.
  2. Click a group arrow to expand the group into its individual letters or packets.
  3. Check or uncheck boxes to grant access or deny access to each letter or packet.
  4. Click Save to accept your changes for that user role.
</Accordion>
