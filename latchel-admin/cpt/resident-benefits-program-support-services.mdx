## Purpose

This SOP establishes the procedures for:

1. GL Code Management for New, Future, and Renewed Leases
2. Resident Outreach for Resident Benefits Package (RBP) Awareness and Enrollment Support

The objective is to improve operational efficiency, ensure accurate lease setup, increase resident awareness of available benefits, and provide Property Management teams with enhanced visibility and reporting.

## Instructions

### Step 1: Lease Identification

Review incoming records and identify:

- New leases
- Future leases
- Recently renewed leases

Verify:

- Resident name
- Property address
- Lease start date
- Lease status

### Step 2: Addendum Verification

Confirm that:

✓ Resident Benefits Addendum is signed

✓ Documentation is complete

✓ Lease qualifies for RBP enrollment

If documentation is incomplete:

- Mark as Pending Documentation
- Add to the exception report
- Notify the designated PM contact

---

### Step 3: Authorization Validation

Confirm Property Management approval exists for:

- GL code entry
- Lease processing support

If authorization is not available:

- Hold processing
- Document status
- Escalate to a supervisor if necessary

---

### Step 4: GL Code Entry

Access PMS and:

- Locate the resident account
- Verify lease details
- Add applicable GL code
- Confirm successful entry

Document:

- Date completed
- Team member completing task
- Lease type
- Processing notes

---

### Step 5: Quality Assurance Review

Conduct secondary verification:

- Correct resident selected
- Correct GL code assigned
- Billing setup reflects intended configuration
- No duplicate entries exist

---

### Step 6: Tracking and Documentation

Update tracking report with:

- Resident Name
- Property
- Lease Type
- Move-In Date
- Addendum Status
- GL Code Status
- Processing Date
- Processor Name
- Notes

---

# Exception Handling

Examples:

- Missing addendum
- Incomplete lease information
- PMS access issues
- Duplicate records
- Authorization not received

All exceptions must:

- Be documented
- Be included in every 25th of the month reporting
- Remain open until resolution

# RESIDENT OUTREACH SUPPORT

## Objective

To educate residents with newly renewed, new, and future leases about the Resident Benefits Package and provide support regarding available benefits.

---

## Scope

Applies to:

- New Leases
- Future Leases
- Newly Renewed Leases

Where:

- The resident has not yet received benefit information
- Outreach has been approved by Property Management

---

## Communication Channels

### Email

Used for:

- Initial introduction
- Benefit summaries
- Follow-up information

### Text Message

Used for:

- Awareness reminders
- Follow-up communications
- Contact verification

### Phone Calls

Used for:

- Resident education
- Question handling
- Direct engagement

---

## Outreach Procedure

### Step 1: Resident Identification

Generate outreach list containing:

- Resident name
- Contact information
- Lease type
- Property information
- Outreach status

Verify contact details prior to outreach.

---

### Step 2: Initial Outreach

Contact the resident using an approved communication method.

Objective:

- Introduce Resident Benefits Package
- Explain available benefits
- Answer general questions
- Provide support resources

Document:

- Date contacted
- Method used
- Outcome

---

### Step 3: Resident Education

Provide information regarding:

- Package features
- Included services
- Enrollment process
- Frequently asked questions

Maintain approved messaging standards.

---

### Step 4: Resident Questions

Address questions within approved guidelines.

If the question requires Property Management review:

- Document inquiry
- Escalate appropriately
- Track until resolution

---

### Step 5: Follow-Up Materials

If resident requests additional information:

Send:

- Benefit overview
- Program information
- Contact resources

Verify preferred email address before sending.

Document completion.

---

### Step 6: Follow-Up Process

If no response is received:

#### First Follow-Up

Within designated follow-up period:

- Email reminder
- Text reminder

Document attempt.

#### Second Follow-Up

If still no response:

- Additional phone outreach
- Final communication attempt

Update tracking system.

---

### Step 7: Outreach Completion

Mark outreach as complete when:

- Resident successfully contacted
- Information delivered
- Resident declines information
- Maximum outreach attempts completed

---

# REPORTING REQUIREMENTS

## Every 25th of the Month Reporting

Provide Property Management with:

### GL Code Report

- New leases processed
- Future leases processed
- Renewed leases processed
- Pending records
- Exceptions

### Outreach Report

- Residents contacted
- Outreach method used
- Successful contacts
- Follow-up required
- Open issues
