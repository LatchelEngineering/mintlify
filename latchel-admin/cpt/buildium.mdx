## **Logging into Buildium**

<Warning>
  Keep in mind each PM has their own personal link to Buildium so make sure you are always using the correct one. Once you select the link it will ask you for the email and password which you will find in the tracker.
</Warning>

### **Step 1 - Locate the correct PM and their unique login credentials**

a. Navigate to the **"Overall Details"** tab in the CPT Tracker.

b. Find the PM you will be tracking and confirm that their PMS is listed as "Buildium."

c. Locate the following three columns for that PM:

`- PMS URL` (This is the PM's unique Buildium link)

`- Username`

`- Password`

c. Keep this information ready for the next step.

<Frame>
  ![](/images/CPT-138.png)
</Frame>

### **Step 2 - Initiate the login and trigger the security verification**

a. Click the unique `PMS URL` link from the tracker to open the correct Buildium login page.

b. Enter the `Username` and `Password` you found in the tracker.

c. The system will prompt you for a secondary verification code if:

- It is the first time you are logging into this account.
- There have been 30 days of inactivity.

### **Step 3 - Complete the time-sensitive two-factor authentication (2FA) process**

a. Once you trigger the 2FA, the verification code will be sent automatically to the SMS inbox in the Latchel console.

<Warning>
  The verification code expires in 10 minutes. Immediately check the console's SMS inbox. Be aware that an Ops associate who is unfamiliar with this process might accidentally close the chat.
</Warning>

b. Copy the code from the SMS message.

c. Return to the Buildium page, paste the code into the corresponding field, and click **"Sign In"**.

<Frame>
  ![](/images/CPT-139.png)
</Frame>

<Warning>
  If the code expires or you encounter any other error during the login process, do not continue trying to log in. Immediately contact your Team Lead (TL) for further instructions.
</Warning>
