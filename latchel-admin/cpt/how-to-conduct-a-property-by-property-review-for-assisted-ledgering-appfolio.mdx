---
title: "How to Conduct a Property-by-Property Review for Assisted Ledgering (AppFolio)"
---

<Note>
The purpose of this SOP is to provide a job aid on how to conduct a property-by-property review for customers who are new to Assisted Ledgering using AppFolio PMS. This review is a one-time process performed before the property manager's service start date. It ensures that the CPT agent classifies each property listed in the rent roll report to identify any opportunities that can be ledgered.
</Note>

## Step by step

<Check>
Ensure to download the Rent roll (itemized) report before proceeding with the Property by property review.Follow the steps on the SOP [How to: Generate a Rent Roll Report in Appfolio](https://latchel.atlassian.net/wiki/spaces/AL/pages/2805923844/How+to+Generate+a+Rent+Roll+Report+in+Appfolio)
</Check>

### **Step 1: Verify the  Property and Resident in AppFolio **

a. Click on the resident’s name (_current tenant_).

b. Identify and select the affected property.

c. Open AppFolio and search for the resident and property.

d. Cross-check that the information in AppFolio matches the data in the CPT Tracker.

### **Step 2:  Review Resident Details**

a. Scroll down and check the **Recurring Charges** section.

b. If **no lease is found**, it is **not ledgerable**. **(Update Column L**)

c. If the **Lease Information tab** is not visible in PMS, go to the **Attachments** tab to locate and review the lease document.

d. Click on the **file/lease document** to confirm that it is under the **Property Manager (PM)** you are currently reviewing.

</Warning>

### **Step 3:  Update Property Details**

a. Update the **Type** _(**Column R**)_ to:

- _ _**Residential**_- If the lease document explicitly states **"Residential or is a 12 month lease"**, then the unit should be classified as **Residential**._
- **HOA / Section 8**_ - To determine if a unit is classified under HOA or Section 8, open the lease document, press CTRL + F, and search for keywords such as "HOA" or "Section 8"._
- **Monthly**- check the PMS (AppFolio). This information is located at the top-right corner of the resident’s profile page.

### **Step 4:  Go to the Lease Documents section. Press Ctrl + F and search for any of the following keywords.**

a. “**Latchel”**

b. **“Resident Benefits”**

<Note>
**Important:** The lease must clearly show one of the following: **“Resident benefits package  or anything similar”** with the **corresponding amount**, or the word **“Latchel”**
</Note>

c. If confirmed, **update Column U** to reflect that the addendum is included.

d. **If a GL code was added**, go back to the PMS. At the top of the resident’s profile, click “Current Balance.”

e. **Look for the month** you are reviewing. Check if there is a GL code charge listed for that month.

h. Complete Columns **AC to AE** if the resident has a future lease. Leave these columns blank if there is no future lease.

### **Step 5: What are the Missed Opportunities**

a. New or renewed leases are sometimes entered without a GL code in their recurring charges. This happens even when there’s a Latchel addendum or a lease uploaded. Without the GL code, these charges can’t be tracked properly, leading to reporting and accounting issues. Ensure you mark these accordingly by selecting any of the following options if that’s the case:

<Frame>
  <img src="/images/CPT-136.png" alt="" />
</Frame>

<Note>
_**Reminder:** Will renew in the next 15-60 days - If there is a current lease that is about to expire soon, and the resident is likely to renew_
</Note>

- _Just renewed, and no GL code was added; enter the lease start date._
- _Month to Month with no GL code added- If the resident is on a month-to-month lease _
- _New tenant and no GL code added - If the resident moved in after the PM’s onboarding date, and no GL code has been added in the PMS_

b. ** **Insert the lease start date in **Column X. **If no matching information is found, proceed to **Step 14**.

### **Step 6: Future Lease columns and Notes section**

a. Complete **Columns Z **and **AA**, and enter the date of your review in **Column AB.**

<Frame>
  <img src="/images/CPT-137.png" alt="" />
</Frame>

b. ** In Column AG **_(Notes)_, enter any relevant notes from your review in **Column AG**. For example, note 'Had a GL code in July' if the GL code was recently removed but was present in the previous month. Leave the column blank if no notes are necessary.
