<Note>
The purpose of this article is to find all the resources and processes related to the Filters Benefit.
</Note>

The Air Filters service is an optional enhancement for Property Manager subscriptions, allowing them to provide a filters' delivery service to their residents. We work together with HomeDepot that will handle the filter shipments.

<Warning>
In **Air Filters**, once a contract reaches 12 months, **<u>it auto-renews unless the property manager cancels</u>**. Monitor the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) Slack channel regularly for any “**Desire-to-Churn”** **notifications** or **cancellation notices**. Make sure to check regularly the service start date for your PMs assigned in Filters.
</Warning>

Each Property Manager that is part of this benefit, will be listed in the [Filters Overview tab](https://docs.google.com/spreadsheets/d/1qd4oXlu6dBYwvPCVnl8fMwoulezF2MTZQoGOhqPqdo0/edit#gid=0), in the Benefits CPT Tracker. Here all the necessary information for each Property Manager is included.

</Warning>

<Accordion title="Filters + Piñata">
When a Property Manager has both Air Filters and Piñata Benefits, some steps of the reports' processes will be merged.

For Property Managers that only have Piñata, they will have the Piñata Report Tab, and the GL Codes Tab, in the Benefits CPT Tracker. For Property Managers with Air Filters only, they will have the GL Codes Tab, and the (AU) Tab.

When a Property Manager has both benefits, the GL Codes Tab will be used for both purposes, therefore a PM with both benefits will have 3 tabs in total.

The first days of the month the CPT will work on the Piñata’s report, the GL codes tab will serve as the base to create the report, as only the tenants with GL code will be included in the report. (Keep in mind that if a tenant had a GL code during the month you are reporting this will still be included, even if the GL code was removed during the current month).

When the GL codes tab is being updated this will serve for both Piñata’s report and Air Filters shipments.

In the GL codes tab you will use the GL code removed last month, Need to add in this month’s invoice column to register the properties that even though they went vacant or the GL code was removed, they still need to be included in the last month’s invoices for both Piñata and Filters benefits. These properties will also be included in Piñata’s report if the month being reported had still a paid GL code.

<Check>
Example: On May 25th a property went vacant, but checking in the PMS, May’s GL code was charged, therefore, in June when we are working on Piñata’s report and the invoices amounts for May, that property will still be included in Piñata’s report and in both Piñata and Air Filters May’s invoices.
</Check>

In the GL codes tab, the columns for “Paid months”, “First Filters shipment made on expected date?”, and “Property Filters' Amount” will only serve for Air Filters purposes.

While working on updating the GL codes tab and Piñata’s report, you will also be able to update FilterTime if necessary.

<Check>
When all steps have been completed, report to Finance the amount of filters the Property Manager needs to be charged in the previous month, and the amount of GL codes for Piñata. To report this, go to the Billing Inputs Master Doc and add the amounts in Column Q and Column R.
</Check>
</Accordion>

<Accordion title="Filters + Piñata + Cover Pest">
For customers with this combination of benefits the process will be almost the same as the one with Filters + Piñata, since customers with these 3 benefits will not have an extra tab for Cover Pest.

For Cover Pest you will use the same GL codes tab used for Filters and Piñata, and report all the properties with a GL code up to the month you are reporting.

This list will go directly to the Cover Pest report in the [PM’s folder](https://drive.google.com/drive/folders/1K-OxCEHlyfI6GNhukN9aNPsZho1yzz7T)that is shared with the customer and Cover Pest.

See how to create the report in this job aid: [**Cover Pest**](https://latchel.atlassian.net/wiki/spaces/AL/pages/3093692426/Cover+Pest)
</Accordion>

**Filters** are billed to the property manager only when a GL code is present for the property; otherwise, no charge is applied—even if a filter size is set or missing. Properties with a missing filter size and a GL code are billed as 1.
