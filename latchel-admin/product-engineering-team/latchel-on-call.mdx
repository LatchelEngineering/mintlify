#### **Intake Exception Management (11/7)**

In On-Call, we do not create new property/resident information as part of the call center intake process. Instead, we respect the “source-of-truth” which is the upstream integration system. This is different than in traditional Latchel where, if we can’t find a given resident/property etc. we will try to create them. The only exception to this is that we will temporarily add a caller’s phone number to the resident record if the caller confirms they are calling on behalf of a tenant & unit that does exist in the system AND the caller indicates they want to receive notifications for the work order. The next time the sync happens (usually the next day) the secondary number will be removed from the tenant record.

### **FAQ 1: Will Latchel Call Center Associates receive calls from Leo residents?**

(10/01/2025) No. For now, we’re routing all Leo resident calls to an external call center (TMP) that offers English & Spanish support.

### **FAQ 2: Can On-Call clients outside of the Leo partnership use this?**

(10/31/2024) In general, yes! For some clients, a traditional Latchel setup might be better. For example, features like the benefits hub and some workflow customizations are not currently available in the Latchel On-Call product. Check with @Jordan Caley  if you’re not sure.
