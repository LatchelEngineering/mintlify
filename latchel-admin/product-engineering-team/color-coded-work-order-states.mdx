## What is the Product Release?

Color-coded states provide simple visual clues to help new Latchel users understand Latchel's automation and WO state designations. This will be visible initially on the Board and List views of the Work Orders Overview page.

## Colors de-coded

The table below explains what each of the colors are intended to communicate.

| Color Scheme | Intended Meaning | Example |
| --- | --- | --- |
| Yellow | Needs PM or owner attention |  |
| Blue | Latchel's automation is running. Good outcomes are possible without PM intervention. |  |
| Green | On track or done. No action needed. |  |
| Gray | No action needed. |  |

## What Are Some Risks or Ways Things Could Go Wrong?

Mapping states to colors is an imperfect approach: There are some instances where a simple state:color mapping will not perfectly match the actual real-world status of a work order. However we think this is close enough of a heuristic to be worth testing folks reaction to it.

In-app guidance: We're not starting with any in-app guidance or language to explain the colors. Users could have questions about what the colors mean.
