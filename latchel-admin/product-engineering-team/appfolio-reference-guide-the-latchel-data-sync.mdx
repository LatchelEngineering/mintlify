This guide helps customer support agents understand and use the Latchel Data Sync Chrome Extension. It provides instructions and information to support customers with data synchronization between Appfolio and Latchel.

# **Activating the Latchel Data Sync (New Customer)**

### **Step 1: Activate Chrome Plugin in Latchel - Latchel (Internal Only)**

1. From your admin login, search the PM account and navigate to the **Integrations** tab. Click **Activate** under the Chrome Plugin Integration.
2. Enter the client's AppFolio database **App Slug**. You can locate this by navigating to the client's AppFolio account URL and copying the database name. In the example below, the **App Slug** is highlighted in red:
   - [https://rinconpm.appfolio.com/users/sign\_in](https://rinconpm.appfolio.com/users/sign_in)
3. Copy/Paste the **App Slug** into the corresponding field, and set the **Starting Sync Date** as today’s date, then hit **Save**.
4. Navigate to **Miscellaneous Settings**
   - Designate which **Integration Sync Method** the client prefers (most clients choose the default Assigned Work Orders). 
     - Reference the [<u>Latchel Data Sync Extension Reference Guide</u>](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/Latchel-Data-Sync-Extension-Reference-Guide#latchel-data-sync-extension-reference-guide "https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/Latchel-Data-Sync-Extension-Reference-Guide#latchel-data-sync-extension-reference-guide") for additional details on the Sync Methods. 
   - Disable **Property Modification** and **Resident Modification**.
5. Loom Video for reference.

### **Step 3: Force Run Property & Resident Sync - Property Manager **

If the properties and residents do not begin syncing right away (i.e. do not see imports or properties populating in [<u>app.latchel.com</u>](http://app.latchel.com)) then direct the customer to run a forced property sync. Below are the steps

1. Have the property manager share their entire screen and open the Chrome Developer Console (right click \+ Inspect, or Ctrl/Cmd \+ Shift \+ I, or three dot menu on Chrome \> More Tools \> Developer Tools.)
2. Go to the Console tab: First paste in “allow pasting”, you’ll see a warning telling you to type in “allow pasting” and hit enter/run. (yes, it’s a weird security measure)
   - Sometimes it still does not work. If that’s the case: **Reset DevTools settings:**
     - Open DevTools and navigate to the "Preferences" settings (gear icon \> Preferences).
     - Scroll to the bottom and locate the "Restore defaults and reload" button, typically found under the "Sync" section if a profile is active.
     - Click this button to revert DevTools settings to their default state, which includes paste protection settings.
3. copy and paste `SyncService.bgSync()` and run

If the sync does not begin within 1 minute, then troubleshoot the permissions granted to the logged in Appfolio user.

# **Data Synchronization Overview**

## **Types of Syncs**

### **Background Sync (Appfolio \> Latchel)**

**Purpose**: Transfers data from Appfolio to Latchel

The background sync now runs on two independent cadences

**Work Order Sync (every 15 minutes)**

- Data included: Work orders only
- Frequency: The plugin checks every 15 minutes. If it has been more than 15 minutes since the last work order sync, a work order sync runs.
- Runs 24/7 — no longer restricted to business hours.

  **Daily Full Sync (every 24 hours)**

- Data included: Properties, tenants, owners, and work orders
- Frequency: The plugin checks every 15 minutes. If it has been more than 24 hours since the last full sync, a full sync runs.
- Runs 24/7.

### **Foreground Sync (Latchel \> Appfolio)**

- **Purpose**: Updates work orders in Appfolio based on changes in Latchel.
- **Data Included**: Creates new work orders and updates existing ones that have been either completed or canceled since the last sync.
- **Conditions**: Only occurs automatically when automatic work order sync is on. Otherwise can be initiated manually.
- **Frequency**: Manually initiated or, if automation is enabled, runs immediately following the automated background sync.

# **Data Sync Details**

## **Properties**

### **Newly Created Properties**

When a property is first synced from Appfolio to Latchel, the following fields are mapped:

| **Appfolio Field** | **Latchel Field** |
| :-- | :-- |

| **Appfolio Field** | **Latchel Field** |
| :-- | :-- |
| PropertyId | ref\_property\_id |
| UnitId | ref\_unit\_id |
| PropertyStreet | address1 |
| PropertyStreet2 | address2 |
| PropertyCity | city |
| PropertyState | state |
| PropertyZip | zip |
| Visibility: Active/Hidden | is\_active \*If property is “hidden” in Appfolio, the property will be marked as inactive in Latchel |
| PropertyName | integration\_data (property\_name) |
| PropertyAddress | integration\_data (property\_address) |

### **Previously Created Properties**

- **Updates**: Only changes from active to inactive status are synced. If a property is unhidden in Appfolio, this change will not propagate to Latchel.
- **Address Data**: If any address data differs between Appfolio and Latchel (for example if service ops needs to update a “bad” address), the exact platform address is stored to provide the "Source Address" feature for customers. This can be viewed on the work order or property pages.

### **Exclude properties from import**

Based on customer requests to exclude certain units from import to Latchel (ex: commercial units, non-revenue units, parking spots, storage units, etc.), PMs now have more control via the "Exclude from Property Sync" button now showing in Appfolio.

### **Common areas**

Upon customer request, Latchel can create common area units for PM’s to manage building-level maintenance for jobs such as landscaping, garbage removal, elevator maintenance and more.

How it works:

1. A PM (or Latchel team member on behalf of a PM) submits a one-off request for common area maintenance on a specific building(s) via this Typeform: [<u>https://latchel.typeform.com/to/P8jj3FBx</u>](https://latchel.typeform.com/to/P8jj3FBx)
   1. Why one-off requests only? We only want to enable common areas on specific addresses for customers who are actually going to create building-level work orders due to billing implications (they will appear as active units in Latchel for billing purposes).
2. Latchel creates the common area unit via API.
   1. This results in a record with a building ID but no ref unit ID.

## **Tenants/Residents**

### **Newly Created Residents**

When a tenant is first synced from Appfolio to Latchel, the following fields are mapped:

| **Appfolio Field** | **Latchel Field** |
| :-- | :-- |
| SelectedTenantId | ref\_user\_id |
| Name | name |
| Phone (Mobile, Home, Other) | Primary phone number (first one we receive in the report) All other numbers set as secondary numbers |
| TenantPortalLogin | Primary email |
| Occupancy ID | integration\_data (occupancy\_id) |
| Move In Date | move\_in |
| Move Out Date | move\_out |

### **Previously Created Residents**

- **Updates**:
  - Move in and move out dates are updated.
  - The user\_property status in Latchel is updated based on move in and move out dates.
  - The property and unit association may be updated.
- **No Updates**: Contact information and names are not currently updated once a resident is created.

## **Work Orders**

### **Background Sync (Appfolio \> Latchel)**

**Conditions for Sync**:

- Work order must be assigned to a “Latchel” user. See “[<u>Integration Sync Method</u>](https://latchel.atlassian.net/wiki/spaces/PROD/pages/edit-v2/2608824321#Integration-Sync-Method "https://latchel.atlassian.net/wiki/spaces/PROD/pages/edit-v2/2608824321#Integration-Sync-Method").”

When a work order is first synced from Appfolio to Latchel, the following fields are mapped:

| **Appfolio Field** | **Latchel Field** |
| :-- | :-- |
| WorkOrderId | ref\_job\_id |
| ServiceRequestId | ref\_service\_request\_id |
| WorkOrderNumber | order\_number |
| First 50 characters of job description | name |
| Description | description |
| Instructions | vendor\_description |
| Max Cost | max\_cost |
| Tenant\* | tenant\_id \*ref\_user\_id on job import |

#### **Work Order Update conditions**

A customer can specify if they want WO updates to happen less often than by default. This settings is located  on a PM company page → Integrations tab → Work Order Update conditions. This setting is only accessible by admin users.
