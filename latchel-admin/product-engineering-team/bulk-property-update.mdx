## What is the Product Release?

A tool designed to help the data support team and eventually customers more easily align Latchel's records to their source-of-truth property data.

<Frame>
  ![Bulk property update tool overview](/images/Eng-275.png)
</Frame>

Our initial goal is to reduce the time the Data Support team spends on property/resident listing tasks from 5.5 hours to \<2.75 hours.

<Frame>
  ![Time savings goal](/images/Eng-276.png)
</Frame>

## Who Is Impacted?

We expect that the Data Support team will be the only initial users of the tool. Admin users in Latchel have access to the tool.

## When Is It Relevant?

When performing a Property/Resident Listing update per Data Support team SOPs.

## What was the Previous State?

Data support team members would open potentially hundreds of individual property tabs in Latchel in order to execute changes.

## How Is It Used?

The tool can be accessed by going to a PM Company's profile page in Latchel → clicking on the "Properties" tab → clicking the three-dot "ellipsis" in the top right corner → select "Bulk Update Properties"

<Frame>
  ![Accessing the bulk update properties tool](/images/Eng-277.png)
</Frame>

### Features

#### Bulk Create Properties

Demo - Bulk create properties

<iframe src="https://www.loom.com/embed/f1acfe8a0e434750b6046015159392dc" title="Loom video player" frameborder="0" className="w-full aspect-video rounded-xl" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen />

Similar to the existing "Import Properties" tool (bulk update will eventually replace this), users can create many properties at once. The fields you can set on creation are:

- Address 1
- Address 2
- City
- State
- Zip
- Is\_active (1/0)

You must also include a blank column for "property\_id" - this is how the tool knows you are creating a new property vs. updating an old one.

#### Bulk Update Properties

Demo - Bulk update properties

<iframe src="https://www.loom.com/embed/e9127f5d5f98493c9ecaec29c94b4528" title="Loom video player" frameborder="0" className="w-full aspect-video rounded-xl" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen />

You can also update many existing properties at once by specifying the property\_id you want to update. You can update these fields on the property:

- Address 1
- Address 2
- City
- State
- Zip
- Is\_active (1/0)

##### Query to get existing Latchel records

```sql
SELECT p.property_id, p.address1, p.address2, p.city, p.state, p.zip, p.is_active
FROM property p
WHERE p.pm_company_id = [enter PM company ID here]
```

#### Duplicate Prevention

Demo - Duplicate Prevention

<iframe src="https://www.loom.com/embed/7f1295cfed874761896569b322a3d1ff" title="Loom video player" frameborder="0" className="w-full aspect-video rounded-xl" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen />

As long as you supply a property\_id, you will never create duplicates!

If you leave the property\_id column blank, you will create a new property!

The Bulk Update tool respects the "Property Modification" setting on the PM profile.

## What Are Some Risks or Ways Things Could Go Wrong?

The biggest risk is in accidental duplicate creation. We will work with Malen after the initial release to make sure we have guardrails in place to prevent this.

## When Will This Change Be Effective?

Live as of 10/7/2024!

## Troubleshooting

### I submitted awhile ago but I have not received an email yet

You can check the status of your update request using this query

```sql
SELECT *
FROM bulk_property_imports bpi
WHERE bpi.company_id = [enter PM company ID you are working on here]
ORDER BY bpi.created_at DESC
```
