# **Objective**

<Note>
  **Purpose**

  The purpose of this document is to guide new team members on how to instruct the Property Manager to install the Latchel plug-in that is used to synchronize data between the Property Management software and the Latchel Console for Property Manager's accounts. Instruct the Property Manager on how to do Steps 1, 3, 4, 5, and 6. The Latchel Admin will perform Step 2. 
</Note>

---

<Danger>
  ### \*\*This SOP is internal-facing ONLY. This is to never be shared externally. \*\*
</Danger>

## \*\*Step-by-Step Guide \*\*

### **Step 1- (PROPERTY MANAGER) Install the Chrome Extension**

1. Instruct the Property Manager to follow the steps in [**<u>this customer guide</u>**](https://help.latchel.com/s/article/How-to-Install-the-Latchel-Data-Sync-Extension "https://help.latchel.com/s/article/How-to-Install-the-Latchel-Data-Sync-Extension").
2. Have the PM log into the chrome extension using their Latchel credentials. Once signed into the plugin successfully, the PM will see text on the plugin drop-down that states, "Logged in as _PM COMPANY NAME"_

<Frame>
  ![CSM 49](/images/CSM-49.jpg)
</Frame>

### **Step 2 - (ADMIN) Activate Chrome Plugin in Latchel**

1. [<u>Find and enter the Property Manager Profile Page</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/813729656/How+to+Access+the+Property+Manager+Profile+Page#Howto:AccessthePropertyManagerProfilePage-step1FindandenterthePropertyManagerProfilePage "https://latchel.atlassian.net/wiki/spaces/CS/pages/813729656/How+to+Access+the+Property+Manager+Profile+Page#Howto:AccessthePropertyManagerProfilePage-step1FindandenterthePropertyManagerProfilePage")
2. Under the **Integrations** tab, click "**Activate**" under the Chrome Plugin tile. The "App slug" is the portion of the Property Manager's software URL that comes right after **https://** and before the first **period**.
   <Frame>
     ![CSM 50](/images/CSM-50.png)
   </Frame>
3. Copy the slug and paste it under the "App Slug" field.
4. Configure the installation date as the "Starting Sync Date" **Click SAVE**.
   <Frame>
     ![CSM 51](/images/CSM-51.png)
   </Frame>

### \*\*Step 3 - (PROPERTY MANAGER) Create a user for Latchel \*\*

Instruct the Property Manager to create a user for Latchel within their software system using the following information:

1. \*\*E-mail: \*\*This additional email is to be created by the Property Manager and chosen by the Property Manager. 
2. **Username:** This user name is to be created by the Property Manager and must include the word "Latchel".
   - For example, First Name: "**Team"**, Last Name: "**Latchel"**. 
3. **User Permission:** On-site Manager
4. We don't have a setting yet that allows us to store this username so please get this username or email address from the Property Manager document it [<u>here</u>](https://docs.google.com/spreadsheets/d/16B1d5acwDRvx6w8BkSuL4_QAOZmcXKYwf5HFpMcwRdU/edit?ts=5d3f560b#gid=0 "https://docs.google.com/spreadsheets/d/16B1d5acwDRvx6w8BkSuL4_QAOZmcXKYwf5HFpMcwRdU/edit?ts=5d3f560b#gid=0"). 

### **Step 4 - (PROPERTY MANAGER) Assign a fake work order to the user**

1. Instruct the Property Manager to create a fake work order within their software system.
2. Have them assign this fake work order to the user they created for Latchel.

### \*\*Step 5 - (PROPERTY MANAGER) Log in and out of the Data Sync Plugin and Appfolio \*\*

1. Instruct the Property Manager to sign out and then back into the plugin.
2. Instruct the Property Manager to sign out and then back into Appfolio.

### **Step 6 - (PROPERTY MANAGER) Run the Latchel Data Sync**

1. Instruct the Property Manager to navigate back to their software system's homepage.
   - At the top right of the dashboard, a Latchel Logo will be floating, instruct the Property Manager to click the button labeled **Sync all Data.**
     <Frame>
       ![CSM 52](/images/CSM-52.png)
     </Frame>
   - Once this has synchronized, instruct the PM to "complete" or "close" the test work order in Appfolio. We will run into an issue if the work order is "deleted" from Appfolio.
