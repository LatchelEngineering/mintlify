## **Step-by-Step Guide**

## **Troubleshooting Problems and Solutions**

1. **What to do if the Property Manager states the Latchel Sync is not working:**

- Make sure the customer has installed the latest version of the sync: [<u>https://chrome.google.com/webstore/detail/latchel-data-sync/cefobanegfgmcneibgdbcjbegnnoafob</u>](https://chrome.google.com/webstore/detail/latchel-data-sync/cefobanegfgmcneibgdbcjbegnnoafob)
- Make sure the customer is logged INTO the chrome plugin: (see step 5 here: [<u>https://latchel.com/how-to-install-the-latchel-data-sync-extension/</u>](https://latchel.com/how-to-install-the-latchel-data-sync-extension/))
- Confirm the "Latchel Team" user profile inside of Appfolio is activated. 
- Run a manual work order export using Appfolio's reporting tools to ensure our "Latchel Team" user has been assigned to a work order inside the Appfolio system.
  - If you do not find a work order on the report, proceed with steps 4 - 6 here: [<u>How to: Install the Latchel Data Sync Plug-in (Appfolio</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/813729953 "https://latchel.atlassian.net/wiki/spaces/CS/pages/813729953"))
- Run query P2 from this page: [<u>Job-Aid: SQL Queries Templates</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280/Job-Aid%3A+SQL+Queries+Templates "https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280/Job-Aid%3A+SQL+Queries+Templates") to get all properties with no reference ids for a specific company. 
- Use this SOP to learn how to run a query. [<u>How to: Handle Property and Resident Information Update</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/796296121/How+to%3A+Handle+Property+and+Resident+Information+Update "https://latchel.atlassian.net/wiki/spaces/CS/pages/796296121/How+to%3A+Handle+Property+and+Resident+Information+Update")
- To identify properties <u>without</u> a Reference ID in our database, review the data out after executing the query and locate the ref\_property\_id and red\_unit\_id column. You will see "null' which means there is no Reference ID in our system and the property was created in <u>Latchel</u>.

<Frame>
  ![CSM 53](/images/CSM-53.png)
</Frame>

- If the query identified properties without a Reference ID, check the Property Manager's database in Latchel to see if it is a duplicate property. See [<u>Step 2 - Search for the duplicate property</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/795116246/How+to+Merge+duplicate+properties#Howto:Mergeduplicateproperties-step2Step2-Searchfortheduplicateproperty "https://latchel.atlassian.net/wiki/spaces/CS/pages/795116246/How+to+Merge+duplicate+properties#Howto:Mergeduplicateproperties-step2Step2-Searchfortheduplicateproperty") to search an individual property in the system. 
- If a duplicate exists, merge the properties and keep the record with a Reference ID. Click **merge**.
  <Frame>
    ![CSM 54](/images/CSM-54.png)
  </Frame>
- It's important to also notify the #tech\_escalations channel when this occurs to investigate why this happened in the first place.
- The Latchel data sync is blocked until all properties without a Reference ID are merged or disabled from the account. 
  - Disable properties that did not have a duplicate and provide a list of the properties to the tech team (@Jullian Chavez please check this step? How do you want us to handle these?).

2. **What to do if the Property Manager states work orders <u>from Latchel</u> are <u>not</u> showing up in <u>their system</u>:**
   - The customer has not given us the appropriate permissions. They either need to make us an On-Site Property Manager user in their software system (see [<u>Step 5 - (PROPERTY MANAGER) Create a user for Latchel</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/813729953#Howto:InstalltheLatchelDataSyncPlug-in-step4Step5-\(PROPERTYMANAGER\)CreateauserforLatchel "https://latchel.atlassian.net/wiki/spaces/CS/pages/813729953#Howto:InstalltheLatchelDataSyncPlug-in-step4Step5-(PROPERTYMANAGER)CreateauserforLatchel")) **or** grant the Latchel user the following permissions:
     - Work Order Report
     - Tenant Directory Report
     - Unit Directory Report
     - Property Directory Report
3. **What to do if the Property Manager states work orders <u>from their system</u> are <u>not</u> showing up in <u>Latchel</u>:**
   - First, consider the fact that there is a delay period for imports to process fully. You can check the Latchel Imports (look for the company name) to see if this import is still processing. <u>Note:</u> The import has to go through the entire property list **and then** the work orders synchronize. 
     - See [<u>Step 5 - Check the status of your import/upload</u>](https://latchel.atlassian.net/wiki/spaces/CS/pages/795934730/How+to+Add+Properties+in+the+Latchel+Console+to+a+New+Account#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step5Step5-Checkstatusofyourimport/upload "https://latchel.atlassian.net/wiki/spaces/CS/pages/795934730/How+to+Add+Properties+in+the+Latchel+Console+to+a+New+Account#Howto:AddPropertiesintheLatchelConsoletoaNewAccount-step5Step5-Checkstatusofyourimport/upload")
       - If you do not see the sync, that is a red flag. This should be escalated to tech by slacking #tech\_escalations.
