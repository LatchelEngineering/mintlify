## **1. Case Statuses Descriptions**

### **Open View**

| **New:** New Cases that haven't been viewed yet. |
| :-- |
| **Open:** Cases that have been viewed but still need to be handled. |

### **Pending View**

| **Escalated:** Cases that need a reply from CS but need to be escalated to another department or a CSM (e.g.Deep Dives). If the Case should be handled by another department and CS is not in charge of the comunication (e.g. maintenance issues), once forwarded to the proper department, it should be closed as "Not Actionable". |
| :-- |
| **In Progress:** Cases that have actions in progress but are being handled in our department (CS). |
| **Waiting for Customer:** Cases that need a response from the customer. |
| **Scheduled Call:** A meeting/call has been scheduled but has not yet been finalized. After the call is concluded, the Case should be closed. |
| **On Hold:** Used for cases that shouldn't automatically go to Open but we can't still close. |

### **Closed View**

| **Closed:** Cases that **pertain to Latchel** but don't need additional pending actions. Used when the request, issue or question has been solved. |
| :-- |
| **Closed Unresponsive:** Cases that **pertain to Latchel** but don't need additional pending actions. Used when we don't receive a response from the customer. |
| **Merged:** Only for Cases that have already been merged. This assignment is automated when merging Cases. |
