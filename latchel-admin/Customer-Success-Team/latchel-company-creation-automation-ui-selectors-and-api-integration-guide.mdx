<Note>
  This document provides a comprehensive breakdown of our Selenium-based automation script for provisioning and configuring Property Management Companies in Latchel. For each major workflow—login, company creation, address & URL updates, settings application, product package setup, subscription activation, and HubSpot synchronization—it lists the exact UI selectors (IDs, XPaths, CSS selectors, AngularJS models) that the code depends on. By cataloguing these dynamic touchpoints, engineering teams can quickly identify and update fragile selectors whenever the Latchel UI changes. To further improve reliability, maintainability, and scalability, is also recommended introducing dedicated RESTful API endpoints (e.g., createCompany, updateAddress, assignSubscription) that would allow these operations to be performed directly—bypassing fragile UI automation—and support high-volume, programmatic account management.
</Note>

# **Code Architecture and Steps Followed**

## **1. Login to Latchel**

Navigates to the login page and submits credentials.

- **URL**: `https://app.latchel.com/login`
- **Email input**
  `(By.ID, "login-email") `
- **Password input**
  `(By.ID, "login-password") `
- **Submit** by sending `Keys.RETURN` on the password field
- **Success check**: waits for URL to contain `"work-orders"`

---

## **2. Create a New Company**

- **Navigate**: `https://app.latchel.com/admin/property-manager/new`
- **Form container**

- **Company Name**
  `(By.CSS_SELECTOR, "input[placeholder='Property Management Company Name *']") `

  `(By.CSS_SELECTOR, "input[placeholder='Phone *']") `

# **Why this could break**

Every one of these steps relies on **exact** IDs, placeholders, class names, or XPATH hierarchies. If because a redesign of the ui, new features or changes we:

- Renames an input placeholder
- Moves a field into a different wrapper
- Changes a CSS class on a button
- Adjusts text on a label

then Selenium will immediately throw a `NoSuchElementException` or `ElementNotInteractableException` and the flow will not be completed.

---

## **Suggested API Endpoints**

To avoid this fragility, will be good to expose simple REST endpoints that exactly mirror these flows:

| Endpoint | Purpose |
| :-- | :-- |
| `POST /api/companies` | Create a company |
| `POST /api/companies/{companyId}/subscriptions` | Attach a new subscription package |
| `PATCH /api/companies/{companyId}` | Update addresses, settings, website |
| `PATCH /api/companies/{companyId}/integrations` | Enable AppFolio, toggle features |
| `GET /api/companies/{companyId}` | Retrieve company details (incl. URLs) |
