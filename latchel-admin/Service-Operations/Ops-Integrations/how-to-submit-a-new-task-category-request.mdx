---
title: "How to: Submit a New Task/Category Request"
---

<Note>
  The purpose of this SOP is to provide the Ops Integration Team with a step-by-step guide on how to request the addition of new tasks or categories to the Ops Integration Productivity Tracker. This process ensures that our database remains clean and organized. By using this workflow, we avoid duplicate entries and ensure that all new work types are approved and standardized before being used in daily logging.
</Note>

## Step by step

### **Step 1: Verify Before Requesting**

a. Before submitting a request, please verify that the Task or Category does not already exist.

b. Check the  [**Agent Productivity Tracker - Ops Integration Team**](https://docs.google.com/spreadsheets/d/1dnt8JCPxOkx4D1KQZVGksPdclyWXtv6QQlRueWkSw4M/edit?gid=1143660504#gid=1143660504) under your designated task tab.

<Frame>
  ![image-20251203-201344.png](/images/Intg-10.png)
</Frame>

<Tip>
  Often, a task may already exist under a slightly different name (e.g., "Client Sync" vs. "Client Meeting"). **<u>Only proceed if you are certain the specific activity is not currently listed.</u>**
</Tip>

### **Step 2: Access the Workflow in Slack**

a. Open Slack and navigate to the Ops team channel.

b. Click on the **Worflows(⚡)** button (the lightning bolt icon) located at the bottom left of the message bar.

c. Search for or select **"Task/Category Request"** from the list.

<Frame>
  ![image-20251203-203056.png](/images/Intg-11.png)
</Frame>

### **Step 3: Fill Out the Request Form**

a. A form will appear. Please complete the fields as follows to help the Admin team understand the request:

- **What is the task name?:** Enter the specific name of the activity (e.g., _"Build Automation Workflow Core"_).
- **Does the category already exist for this task?:**
  - Select **Yes** if this new task fits into an existing bucket (e.g., putting a new specific script under _"Internal Latchel Integrations"_).
  - Select **No** if this is a brand new project type that needs a new Category header.
- **If you selected "No," please enter the category name...:** Type the proposed Category name here. If you selected "Yes," leave this blank.
- **Please provide a brief description of the task:** Explain what the activity involves so the Admin knows it is valid work (e.g., _"Creating Python scripts for the new Q4 initiative"_).
- **Please specify where this task or category belongs:** Choose whether it will be part of Regular Tasks or Projects you’ll be working on.
- **When do you need this task/category to be added?:** Select the date using the calendar picker. This helps the Admin prioritize urgent requests.

<Frame>
  ![image-20251203-203519.png](/images/Intg-12.png)
</Frame>

### **Step 4: Submit the Request**

a. Review your answers for spelling and clarity to ensure the new task matches our naming conventions.

b. Click the green **Submit** button.

c. Your request will be sent to the Admin team for review.

### **Step 5: Add to Your Dashboard**

a. Once you receive confirmation from an Admin that the request has been approved:

1. Go to your **Productivity Tracker Dashboard** and refresh it to load the new task/category added.
2. You can now **Clock In** using this new Task/Category.

<Warning>
  ### Important Notes:

  - **Naming Conventions:** Avoid vague names like "Other" or "Misc work."
  - **Approval Time:** Requests with a "Needed by" date of _Today_ will be prioritized, but please try to submit requests at least 24 hours in advance when possible.
  - **Projects vs. Tasks:** Remember that a **Category** usually represents a large Project or Client (e.g., _"Recurring Work Orders"_), while a **Task** is the specific action (e.g., _"Build Automation"_).
</Warning>
