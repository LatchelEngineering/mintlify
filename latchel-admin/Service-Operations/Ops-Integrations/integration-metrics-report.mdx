The purpose of this SOP is to provide a clear and detailed overview of the Integration Metrics Report tab. This document outlines how the report tracks and summarizes integration issues, monitors team performance, and provides actionable insights for stakeholders on a weekly basis.
</Note>

## **Overview**

The **Integration Metrics Report** is a dynamic dashboard that aggregates data from the [Integration Issue PMs ](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=2021689109#gid=2021689109)tab. It serves as a high-level summary, transforming the raw list of issues into measurable metrics. The report is divided into several key sections: a live list of current issues, daily performance trackers for each team, a weekly summary of resolved issues, and a historical timeline of team performance.

---

### **Dashboard Structure: Integration Metrics tab**

This tab provides a comprehensive view of ongoing integration issues and the performance of the teams responsible for resolving them.

#### **1. Weekly list of PMs with integration issues**

This section displays a live list of all Property Managers currently experiencing an integration issue.

**NOTE:** Currently, the Integration Metrics Report tracks only **"Integration hasn’t run"** issues.
</Note>

<Accordion title="Recurring Integration Alerts Automation Overview">
<Note>
The **Recurring Integration Alerts** is an automated monitoring tool designed to identify property management companies experiencing recurring sync issues. It ensures that frequent integration failures are escalated to the appropriate team members for high-priority resolution.
</Note>

#### **How the Process Works:**

1. **Data Archiving & Record Keeping:**
Each time the process runs, the system captures the latest performance data from the "Integration Metrics" dashboard and saves it into a permanent tracking base. It is designed to recognize and skip duplicate entries, ensuring the historical record remains clean and accurate.
2. **Threshold Monitoring (The "3-Strike" Rule):**
The system analyzes the activity for the current month. If a specific company is flagged for integration issues **three or more times** within the same month, they are automatically categorized as a "Systemic Case." This helps the team distinguish between isolated glitches and chronic problems that require a strategic sync.
3. **Automated Slack Alerts:**
Once a systemic issue is confirmed, the system sends a real-time notification to the [#ps-escalation](https://latchel.slack.com/archives/C08475CCBGR) Slack channel. This alert includes:
  - The name of the affected company.
  - The assigned CSM.
  - Clear instructions and links to the next steps for resolution.
4. **Centralized Logging & Prevention of Duplicate Alerts:**
To maintain a professional communication flow, the script logs every alert it sends. It checks this log before sending any new messages to ensure that a company is only alerted **once per month**, preventing the team from receiving redundant notifications for the same issue.
5. **Accountability & Follow-up:**
All alerts are recorded in a "[**Recurring Integration Alerts** **Log**](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=1950306127#gid=1950306127)," which serves as the primary tracking sheet for the team to document outcomes, record alert URLs, and confirm when the systemic issue has been resolved.

The **PS Support team** is responsible for the manual follow-up of each entry to ensure the loop is closed.

**For PS Support:**

- **Update the ASM URL (Column F):** Manually paste the link to the corresponding URL case record.
- **Select the Outcome (Column G):** Choose the final status of the intervention from the predefined dropdown menu. The available options are:
  - **PS Resolved (Latchel Sync Running):** Issues were fixed directly by Product Support.
  - **PM Unresponsive (Escalated to CSM):** The Property Manager did not respond to Support, requiring escalation.
  - **CSM Successful (Latchel Sync Running):** The CSM successfully coordinated with the client to restore the sync.
  - **CSM Unsuccessful (PM Unresponsive):** Despite escalation, the Property Manager remains unresponsive.
</Accordion>
