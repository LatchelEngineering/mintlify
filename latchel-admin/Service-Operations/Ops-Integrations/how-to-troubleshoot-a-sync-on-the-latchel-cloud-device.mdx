<Note>
  The purpose of this SOP is to provide a step-by-step guide on how to access the Latchel cloud device to troubleshoot a client's AppFolio sync.
</Note>

<Warning>
  **Before you begin**, please watch this quick [**Loom video**](https://www.loom.com/share/122caaea15ff465b98188b85be1acca3) where the process is explained.
</Warning>

## Overview

<Tip>
  **TIP:** To identify the last sync date, open the [**Integration Health Metrics**](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=186001220#gid=186001220) dashboard, go to the **PM Metrics** tab, and filter column **Q — Is the integration run by Latchel?** to show only rows marked **Yes**. Then check column **J — Last import date**. **<u>If you don’t have editor access, you can review the [Latchel Internal Sync PMs list](https://docs.google.com/spreadsheets/d/1FZQ_gcofCAjOcqk4Sb9_lIKlRK_3zUIEAUon-PA40q0/edit?gid=0#gid=0) and compare it with the Integration Health Metrics Dashboard</u>** (_You can use CTRL \+ F to search for each PM_).
</Tip>

<Tip>
  **Monitor the internal Latchel syncs daily.** The PM Metrics query runs every day between 7–8 a.m. and 7–8 p.m CST. If you notice a PM whose date isn’t updated to today’s date in the **Last import date** column, you will need to troubleshoot the sync. **<u>It’s best to troubleshoot the sync before the alerts are triggered in the #integration-sync-issues-internal channel</u>**.
</Tip>

## Step by step

<Accordion title="For the Weekend Internal Sync check, please review this step before you begin.">
  1. Wait for the Integration Sync Check Reminder to trigger in the [#operations\_support](https://latchel.slack.com/archives/CUMBLQDRU). Once it is triggered, acknowledge the notification by using a check mark ✔️
  2. In the Integration Metrics Dashboard, identify the PMs whose syncs are run internally by Latchel, as mentioned in the Loom video and the Overview section above. If you identify any sync that is behind today’s date, proceed with Step 1 or Step 2 accordingly.
  3. Don’t forget to mark the notification with the **“Done”** emoji once you’ve completed all the steps. If no sync was needed, you can simply reply in the notification thread with:**“No manual sync was needed.”**
</Accordion>

### **Step 1: Review and Acknowledge Integration Alerts**

a. If, for any reason, an alert notification for the following triggers in [#integration-sync-issues-internal](https://latchel.slack.com/archives/C096E7A625U) (\_The script for these alerts runs every day between 9–10 a.m. and 9–10 p.m. CS\_T):

- Integration hasn’t run in 2 days
- Integration hasn’t run in 4 days

you will need to acknowledge the notification and proceed to step 2. **<u>If this is not the case, please skip step 1 and proceed to step 2 if necessary.</u>**

<Frame>
  ![image-20251216-204807.png](/images/Intg-72.png)
</Frame>

<Note>
  Other notifications, such as:

  - Integration hasn’t run in \+4 days
  - Data Sync Error: Properties, Tenant, Work Orders
  - Latchel Sync Request

  **<u>are currently handled by Trisha. In the future, these will also be monitored by other team members.</u>**
</Note>

### \*\*Step 2: Log in to the Remote Server Cloud \*\*

a. Log in to the designated **Latchel cloud account** where syncs are managed. b. Log in using the credentials provided

<Tip>
  Currently, only members of the **Latchel Integration Team and Ops Support** have access to the Latchel cloud account. If you have been approved for access, <u>reach out to **Trisha** to **request login credentials**.</u>
</Tip>

### **Step 3: Connect to the TeamViewer Virtual Desktop**

a. Once logged in, go to **Devices** and click **Connect**.

<Frame>
  ![image-20260603-212718.png](/images/Intg-73.png)
</Frame>

### **Step 4: Troubleshoot the Sync in the PM's Profile**

a. A virtual desktop will load. **<u>Open the Chrome browser if the PMs tab isn’t already open.</u>** 

<Frame>
  ![image-20260603-212956.png](/images/Intg-74.png)
</Frame>

b. **Select the specific PM profile** you need to check and identify the issue:

<Tip>
  **NOTE:** Each PM has their own Chrome profile. Keep them as they are so the sync runs smoothly. Open the corresponding client Chrome profile to troubleshoot the issue.
</Tip>

1. **If the Chrome page is not responding**, refresh it and then **continue to option #2**.
2. **If the PM’s last sync is behind the current date**, click the floating **Latchel** button on the right sidebar. From the options that appear, select **Sync data in both directions (Slower) and click on Start sync**. This will sync residents, properties, and work orders (for some PMs, depending on their subscription). Owners may appear as well since they are also being synced; however, this is being implemented gradually by Engineering.

<Frame>
  ![image-20251030-192005.png](/images/Intg-75.png)
</Frame>

3. To confirm the sync was completed successfully, go to Latchel and open the PM’s profile.
4. Then, go to the Integrations tab and, under Latest CSVs received for import, **<u>check the syncs for Property and Tenant</u>**. **<u>The date should be updated to today’s date</u>**.

<Warning>
  The sync for **owners** and **work orders (jobs)** doesn’t always run daily, as it depends on whether the PM has work orders to sync and on the subscription type. Also, _owner syncing is being enabled gradually by the Engineering team._
</Warning>

<Frame>
  ![image-20251216-191834.png](/images/Intg-76.png)
</Frame>

<Frame>
  ![image-20251216-191932.png](/images/Intg-77.png)
</Frame>

<Note>
  The sync date that appears in Latchel is in the format **DD/MM/YYYY**.
</Note>

<Tip>
  If, for any reason, the sync doesn’t go through, review whether the PM removed report access. You can check the following [job aid](https://help.latchel.com/how-to-grant-report-permissions-to-latchel-in-appfolio) to learn which reports are required for the sync. **<u>If this is the root cause of the sync issue, communicate it to Trisha</u>**. The alerts will continue to trigger, and once they reach **“Integration hasn’t run in \+4 days,”** the CSM will be notified.
</Tip>

### **Step 6: Keep the Session Active.**

a. After troubleshooting, **<u>ensure ALL the client’s Chrome profile tabs remain open</u>** in the virtual desktop. This allows the syncs to continue running automatically.

### **Step 7: Close the Connection Correctly**

a. Close the remote server cloud tab **from the TeamViewer virtual desktop**.

<Frame>
  ![image-20260603-213320.png](/images/Intg-78.png)
</Frame>

### **Step 8:** If the sync alert was triggered (Step 1), follow this final step

a. Once you have completed all steps, you can mark the notification with a “done” emoji.
