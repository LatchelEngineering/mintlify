### **What do you find in a Churn Alerts Tab?**

The primary source for adding a case to this record is the Slack report in the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) channel. From this report, we will extract and input the company name, the date of the report, and the corresponding link. Subsequently, we will reference HubSpot to verify the current status of the account and to identify the primary and, if applicable, secondary reasons for churn, as well as the MRR at churn request, and total CMRR.

<Frame>
  ![image-20251107-002028.png](/images/Intg-96.png)
</Frame>

| **Column** | **Detailed description** |
| --- | --- |
| A | In here, you find the name of the company that desires to churn |
| B | Here is shown the company’s status at the moment of the report |
| C | This column shows the MRR for the previous month of when the churn request is received. |
| D | The total CMRR is shown in this column |
| E | Here you can see the date when the Churn alert was reported in Slack, in the [#service-changes](https://latchel.slack.com/archives/CKA2W1KJA) Channel |
| F | In this column, you find the main reason why the PM Company desires to churn |
| G | If there is another reason (secondary reason) besides the main one for the PM Company’s desire to churn, you will see it here |
| H | In here, you find the link to the Churn Alert report in Slack |
| I | If there were ticket reports related to this PM Company within 3 months before the churn request the Assembly link(s) will be included here |
| J | If additional details need to be noted, you will see them in this column |

### **What do you find in a Tickets Tab?**

**At the end of each month**, a CSV file will be downloaded from **Assembly** containing all cases reported during that period. We will filter the topics to keep the ones related to **Integrations** and **Emergency WOs issues**. Through a review of the associated messages, we will assess whether the report originated from the customer or a member of Latchel’s team. Additionally, we will identify the underlying reason for the report to categorize the specific issue, and the property management company to which it pertains.

</Frame>

| **Column** | **Detailed description** |

| B | Here is shown the source type of report, if this is a PM/Owners complaint, or a report from a Latchel Ops Associate |

| D | If the report is in reference to an Integrations issue, you will find the specific reason here. The reasons are grouped by color. ### **Analysis Tab:**

This section provides **a consolidated summary of the key data points from the preceding tabs**—[**Churn Alerts**](https://help.latchel.com/churn-alerts-ticket-reports-review-tracker#What-do-you-find-in-a-Churn-Alerts-Tab%3F)\*\* and [**Ticket Reports**](https://help.latchel.com/churn-alerts-ticket-reports-review-tracker#What-do-you-find-in-a-Tickets-Tab%3F).\*\* It presents total figures, significant monthly trends, and details of Ticket Reports linked to churn requests submitted within the three months preceding each churn alert.

### **Ticket Analysis by PM:**

This tab includes a breakdown of tickets submitted by PM. Click the **\+** icon next to a PM’s name to expand and see their submitted tickets. From column F onward you’ll see the total number of ticket issues submitted by each PM per month.
