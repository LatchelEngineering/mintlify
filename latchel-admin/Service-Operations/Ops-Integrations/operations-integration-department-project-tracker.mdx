<Note>
The purpose of this SOP is to outline the standardized process for submitting, tracking, and managing all team projects. This document provides a clear overview of our project management tool, ensuring that all new requests are properly cataloged and all active projects are accurately monitored from start to finish.
</Note>

### **Overview**

The [**Operations Integration Department - Project tracker**](https://docs.google.com/spreadsheets/d/1Azik9dCI80JU4Wk3nN7ZVxU8LKIvn_nuplYCvk9h0m0/edit?gid=1161341563#gid=1161341563)** **is structured into two main tabs: the **Backlog Projects** tab for new requests and the **2025 Project Tracker** tab for projects that are actively being worked on.

<Note>
An [**appendix**](https://docs.google.com/spreadsheets/d/1Azik9dCI80JU4Wk3nN7ZVxU8LKIvn_nuplYCvk9h0m0/edit?gid=734801100#gid=734801100) tab of the **Operations Integration Department Project Tracker** is also available for stakeholders' quick view.
</Note>

### **Dashboard Structure**

#### **1. **[**Backlog Projects Tab**](https://docs.google.com/spreadsheets/d/1Azik9dCI80JU4Wk3nN7ZVxU8LKIvn_nuplYCvk9h0m0/edit?gid=1228511567#gid=1228511567)

This tab is directly linked to our new Slack workflow and will serve as our central hub for **all new project requests from stakeholders**. All requests submitted via the designated Slack channel will automatically populate here.

<Frame>
  <img src="/images/Intg-13.png" alt="image-20251106-213403.png" />
</Frame>

| **Column** | **Description** |
| --- | --- |
| **Project Name** | A clear and concise title for the project. |
| **Assigned To** | The team member responsible for leading or executing the project. |
| **Project Request Status** | It indicates the request status of your project it’s on:Under ReviewApproved & QueuedIn DevelopmentAwaiting DependencyOn HoldDelivered |
| **Target Start Date** | The proposed or scheduled date for the project to begin once approved. |
| **Priority Order** | The order assigned to the project request, which can be set to:BacklogNext UpIn ProgressOn HoldBlockedCompleted |
| **Priority Level** | The level or urgency of the project request, which can be set to: LowMediumHigh |
| **Description** | Details about what the project entails and its scope. |
| **What is the desired outcome?** | The specific goal, deliverable, or success metric for the project. |
| **What key problem will this project solve?** | A brief explanation of the pain point or inefficiency being addressed. |
| **Files related to the project** | A space for any relevant documents, links, or attachments. |
| **Submitted By** | The name of the stakeholder who requested the project. |

---

#### 2.** **[**2025 Project Tracker Tab**](https://docs.google.com/spreadsheets/d/1Azik9dCI80JU4Wk3nN7ZVxU8LKIvn_nuplYCvk9h0m0/edit?gid=1161341563#gid=1161341563)

This is where we will move all approved projects and tasks that the team is actively working on. This tab provides a detailed view of project progress and resource allocation.

<Frame>
  <img src="/images/Intg-14.png" alt="image-20251106-214803.png" />
</Frame>

| **Column** | **Description** |
| --- | --- |
| **Project Name** | The main project or initiative the task belongs to (e.g., Integration Health Metrics Dashboard). |
| **Business Impact** | Describes how this project will affect the business — for example, by increasing revenue, reducing costs, improving productivity, or enhancing customer experience. |
| **Tasks** | A breakdown of the individual, actionable tasks required to complete the project. |
| **Assignee** | The team member responsible for completing the specific task. |
| **Description** | A detailed explanation of what is required for each specific task. |
| **Status** | The current progress of the task. This will be updated to reflect if the task is: - Not Yet Started - In Progress - On Hold - Complete |
| **Priority** | The urgency of the individual task (Low, Medium, or High). |
| **Start Date & Approximate End Date** | The planned timeline for the completion of each task. |
| **Days** | The estimated duration (in days) required to complete the task. |
| **% Done** | The progress of the task, represented as a percentage. |
| **Estimated Hours & Actual Hours** | A field to track the planned time versus the actual time spent on the task. |
