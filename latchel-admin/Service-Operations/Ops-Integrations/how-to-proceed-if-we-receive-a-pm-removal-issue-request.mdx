---
title: "How to: Proceed If We Receive a PM Removal Issue Request"
---

<Note>
The purpose of this SOP is to outline a clear, standardized process for handling PM Removal Issue Requests efficiently and accurately.
</Note>

<Tip>
This workflow is designed for CSMs to ask the CPT—via the [**#integration-notification-feedback**](https://latchel.slack.com/archives/C08TP7NL76W) Slack channel—to exempt property managers from outreach when/before an issue is detected in the [Integration Health Metrics Dashboard](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=186001220#gid=186001220).
</Tip>

## Step by step

### Step 1: Go to the Integration Health Metrics Dashboard

a. Once you receive the request in the _#integration-notification-feedback channel_, go to the **Integration Health Metrics Dashboard** and <u>locate the Property Manager’s company in the PM Metrics tab.</u>

<Frame>
  <img src="/images/Intg-28.png" alt="image-20250522-222837.png" />
</Frame>

### Step 2: Open the PM Exclusions tab

a. **Copy **the name of the Property Manager and **<u>paste</u>**<u> it into the hidden spreadsheet </u>_**<u>PM Exclusions</u>**_.

b. Make sure to select **Yes** in column B, _**Exclude from Outreach?**_

<Frame>
  <img src="/images/Intg-29.png" alt="image-20250530-223506.png" />
</Frame>

c. Once you select it, the Property Manager Company will be excluded from outreach if an issue is detected.

### Step 3: Provide an update to the CSM

a. Use the following template to provide an update in the request thread:

_Good day [CSM Name]! _
_We have processed your request to exempt [Company Name] from outreach and updated the dashboard. Please let us know if you need anything else._

<Note>
If the Property Management Company has already been flagged in the **Integration Issue PMs tab**, follow the previous steps and erase the row’s information leaving the row blank. When the script runs the next day, it will be updated automatically.
</Note>
