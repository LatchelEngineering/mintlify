<Note>
  The purpose of this SOP is to provide a clear and standardized process for all agents to accurately track their time on tasks and projects using the Ops Integration Productivity Tracker. This ensures proper documentation of work, aids in productivity analysis, and maintains accurate project records.
</Note>

### **Overview**

The [**Ops Integration Productivity Tracker** ](https://productivity-tracker-bj1s.onrender.com/)is a web-based application designed to track the time agents spend on various tasks and categories. The tool is divided into two primary interfaces:

1. **The Agent App:** The main interface used by all team members to clock in, clock out, and manage their daily tasks in real-time.
2. **The Admin Dashboard:** A separate view with advanced analytics, accessible only to administrators, for reviewing agent performance and logged hours.

### **Tool Structure**

#### **1. Agent App**

This is the primary screen that all agents will use for daily time tracking. It is designed for real-time logging of tasks and activities.

<Frame>
  ![image-20251104-230757.png](/images/Intg-1.png)
</Frame>

The Agent App is composed of the following sections:

#### **A. Main Controls**

| **Feature** | **Description** |
| --- | --- |
| **Dark Mode Toggle** | Allows the user to switch between a light and dark interface. |
| **Admin Dashboard Button** | A button that navigates to the Admin Dashboard. _(**Note:** This is only accessible to users with admin privileges)._ |
| **Clock In / Clock Out** | The primary buttons used to start and end a task session. The system logs the exact time of each action. |

#### **B. Session Setup**

| **Field/Action** | **Description** |
| --- | --- |
| **Agent** | A dropdown menu to select the agent's name. This must be selected before clocking in. _(**Note:** This is only accesible to users with admin privileges. Agents will be assigned their own session by default)_ |
| **Task** | A dropdown menu to select the specific task being worked on (e.g., _SOP Update / Creation_, _Dashboard Health Check_). New tasks can be added as needed. |
| **Category** | A dropdown menu to assign a broader category to the task (e.g., _Documentation & Processes_, _External Latchel Integrations_). New categories can be added. |

#### **C. Live Tracking Information**

| **Section** | **Description** |
| --- | --- |
| **Current Session** | This panel displays real-time information about the active session, including the selected **Agent**, **Task**, **Category**, the **Start** time of the current task, and the current time (**Now**). |
| **Today Overview** | This panel provides a running summary of the total time spent on each task/category throughout the current day, allowing agents to see how their time has been distributed. |

#### **2. Admin Dashboard**

This section provides a high-level overview and detailed logs of all agent activity. **Please note: Access to the Admin Dashboard is restricted to administrators only.**

<Frame>
  ![image-20251104-231638.png](/images/Intg-2.png)
</Frame>

The Admin Dashboard is composed of the following sections:

#### **A. Navigation and Filtering**

| **Feature** | **Description** |
| --- | --- |
| **Agents Daily Totals** | A sidebar menu that allows the administrator to select a specific agent to view their detailed activity. |
| **Agent Tabs** | The main view is organized by tabs, allowing the admin to switch between an **Overview** and the individual pages for each agent (e.g., _Diana_, _Martha_, _Trisha_). |
| **Back to Agent App Button** | This link allows an admin user to return to the standard agent time-tracking interface. |
| **OID Productivity Tracker** | This button navigates to the master spreadsheet that serves as the database where all records are kept. This provides access to the raw data for all agents for detailed reporting and analysis. |

<Frame>
  ![image-20251104-232257.png](/images/Intg-3.png)
</Frame>

#### **B. Summary Cards**

At the top of the dashboard, four key metrics provide a quick snapshot of the selected agent's productivity:

- **Time Logged Today:** Total hours logged in the current day.
- **Time Logged This Week:** Total hours logged in the current week.
- **Time Logged This Month:** Total hours logged in the current month.
- **Total Daily Tasks:** The number of distinct tasks performed in the current day.

#### **C. Detailed Activity Log**

This table provides a granular, chronological record of all tasks logged by the selected agent.

<Frame>
  ![image-20251104-231706.png](/images/Intg-4.png)
</Frame>

| **Column** | **Description** |
| --- | --- |
| **Date** | The date the task was performed. |
| **Task** | The name of the specific task that was logged. |
| **Category** | The category associated with the task. |
| **Clock in** | The exact timestamp when the agent started the task. |
| **Clock out** | The exact timestamp when the agent completed the task. |
| **Hours** | The total duration calculated for the task. |
