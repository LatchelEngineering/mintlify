---
title: "How to: Submit a Time Correction Request"
---

<Note>
The purpose of this SOP is to provide a step-by-step guide on how to request corrections for the **Ops Integration Productivity Tracker** for time logs using the **Time correction request **workflow in Slack. This process ensures that errors such as forgotten clock-outs, incorrect categorization, or duplicate entries are fixed accurately by the Admin team without compromising the integrity of the main database.
</Note>

## Step by step

### **Step 1: Identify the Error and Session ID**

a. Before submitting a request, review your **Today Overview** on the **Ops Productivity Tracker dashboard** and check your personal tab in the [**<u>Agent Productivity Tracker - Ops Integration Team</u>**](https://docs.google.com/spreadsheets/d/1dnt8JCPxOkx4D1KQZVGksPdclyWXtv6QQlRueWkSw4M/edit?gid=1143660504#gid=1143660504).

b. Identify the entry that needs correction.

c. **Crucial:** Copy the **Session ID** associated with that specific task (e.g., `session-1764...`). This ID is required for the Admin to locate and fix the exact row in the database.

  <img src="/images/Intg-8.png" alt="image-20251202-210013.png" />
</Frame>

### **Step 2: Access the Workflow in Slack**

a. Open Slack and navigate to the [#ops-integration-int-team](https://latchel.slack.com/archives/C0A0VRUJKK8).

b. Click on the **Workflows (⚡)** button (the lightning bolt icon) located at the bottom left of the message bar.

c. Search for or select **"Time Correction Request"** from the list.

<Frame>
  <img src="/images/Intg-9.png" alt="image-20251202-210151.png" />
</Frame>

### **Step 3: Fill Out the Request Form**

a. A form will pop up asking for specific details. Please fill it out as follows:

- **Reason for Correction:** Select the option that best describes your issue (e.g., _Forgot to Clock In/Out_, _Wrong Task/Category_, _Adjust Time_, or _Delete Duplicate_).
- **Session ID:** Paste the ID you copied in Step 1.
- **Issue:** Briefly explain the error and the correct data (e.g., _"I clocked out at 17:00, not 17:30"_).
- **Does the category or task name need to be updated?:** Select **Yes** or **No**.
- **If so, please state which category...:** If you selected Yes, type the correct Category or Task name here.

### **Step 4: Submit the Request**

a. Double-check your information, especially the **Session ID** and the **Issue** description.

b. Click the green **Submit** button.

c. You will receive a confirmation message in Slack (if configured) indicating that your request has been sent.

### **Step 5: Confirmation of Fix**

a. Once the Admin has processed your request and updated the Spreadsheet, they will typically react to your request message (e.g., with a ✅ checkmark).

b. Refresh your **Productivity Tracker Dashboard** or check the [**<u>Agent Productivity Tracker - Ops Integration Team</u>**](https://docs.google.com/spreadsheets/d/1dnt8JCPxOkx4D1KQZVGksPdclyWXtv6QQlRueWkSw4M/edit?gid=1143660504#gid=1143660504) to verify that the change is reflected correctly.

<Warning>
### Important Notes:

- **Do Not Edit the Spreadsheet Directly:** To prevent data corruption, never attempt to manually change cells in the _**Agent Productivity Tracker Spreadsheet**_ yourself. Always use this workflow.
- **Session ID is Key:** Requests submitted without a Session ID may be delayed, as it is difficult to find the specific entry without it.
- **Timeliness:** Please submit correction requests as soon as you notice the error (preferably the same day) to keep the weekly reporting accurate.
</Warning>
