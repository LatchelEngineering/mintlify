<Note>
  The purpose of this SOP is to detail the end-to-end automated process that detects, classifies, alerts, and resolves PM integration issues. This guide explains the automated script and Slack notifications that operate on the data in the **Integration Issue PMs (Internally Flagged)** tab.
</Note>

<Tip>
  A Script monitors the **PM Metrics tab, displays the issues detected in the[<u> </u>](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=2021689109#gid=2021689109)[Integration Issue PMs (Internally Flagged)](https://docs.google.com/spreadsheets/d/1lqN_yI3vZUehb9BZylCvXK0KU-PFfpJCkDvw1_sgWug/edit?gid=1902680244#gid=1902680244)**\*\* tab,\*\* and sends real-time alerts to the appropriate stakeholders team in Slack when a new issue is detected. **Currently, only PMs using AppFolio software are flagged when an issue is detected.**
</Tip>

## Step by step

### **Step 1: Issue Detection**

a. Whenever a problem is identified in the **PM Metrics** tab, a matching row is created in **Integration Issue PMs**.

b. Columns **A – G** are automatically filled with the data pulled from **PM Metrics**.

<Frame>
  ![ChatGPT Image 11 jun 2026, 01\_22\_52 p.m..png](/images/Intg-37.png)
</Frame>

c. Once the issue is detected, **columns H–M** are updated automatically.

<Frame>
  ![image-20251118-230329.png](/images/Intg-38.png)
</Frame>

<Note>
  **The Data Sync Error — Property, Tenant, Work Order** alerts will flag errors detected during the AppFolio→Latchel sync process. Each alert will include instructions to identify the specific issue and will recur **every 12 hours**. **<u>The Latchel Integration Team</u>**<u> is the primary owner of troubleshooting; technical issues will be escalated to engineering.</u>
</Note>

<Tip>
  Additionally, there’s a Slack summary alert triggered in the slack channel [integration-sync-issues-internal](https://latchel.slack.com/archives/C096E7A625U) if 2 or more issues are unresolved.
</Tip>
