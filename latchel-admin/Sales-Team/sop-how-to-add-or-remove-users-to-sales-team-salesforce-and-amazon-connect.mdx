## Adding Users

When a new team member is hired into the Sales team, they will need access to Salesforce, which is where we store all customer accounts.

If the team member will be making and receiving calls from customers, they will also need to have access to the Phone Dialer and will need to have an Account in Amazon Connect, our Telephony System.

## Creating a New User in Amazon Connect - Outbound and Inbound - Sales Team

Use this process if the user will need to be able to make outbound calls and receive inbound calls

### Step 1 - Create a Queue

**What is a queue?** A queue is place to line up calls to be routed to someone to answer. It is like going to the grocery store and having to stand in a line for the cashier. Sometimes there is a line per cashier, sometimes there is one line for all cashiers. Therefore, when we want that a call goes directly to a specific user, we create a queue for that specific user. If we want a call to be routed to any user, we create a shared team queue.

<Warning>
  **ATTENTION – BEFORE CREATING A NEW QUEUE**

  Before creating a new queue, check if there is any unused queue in the team and if so, re-utilize that one instead of creating a new one. Queues cannot be deleted, so as a good housekeeping practice, re-utilized unused ones before adding more to the list!
</Warning>

To create a queue, follow these steps:

1. On the left side icon menu, click on the icon with the three arrows up and select "Queues"

### Step 2 - Create a Routing Profile

**What is a Routing Profile?** It is a configuration that we can use to define which type of calls a certain user will be answering to. Sometimes we will want an user to only receive calls that were indented to them, sometimes we will want that user to answer any calls, sometimes we want the user to be able to handle both. In a routing profile, you can configure which type of calls that "profile" will handle. When creating an user, you will define which profile you want them to have.

<Warning>
  **ATTENTION – BEFORE CREATING A NEW ROUTING PROFILE**

  Before creating a new routing profile, check if there is any unused profile in the team and if so, re-utilize that one instead of creating a new one. Routing profiles cannot be deleted, so as a good housekeeping practice, re-utilized unused ones before adding more to the list!
</Warning>

   - For AEs - include only their individual queue - In the description, include this sentence - "Routing profile for AEs - only individual queue"

### Step 3 - Create a User

1. Click on the user icon to see the menu. Select "User Management"

<Frame>
  ![](/images/Sales-33.png)
</Frame>

2. Click on "Add New Users"

<Frame>
  ![](/images/Sales-34.png)

3. Add user information:
   - Enter First and Last Name
   - Enter the login or username the team member will use to log into Amazon Connect
   - Enter the Latchel email address for both fields
   - Enter the password - when creating the user, as an initial password, create a simple one (like Latchel1) that it is easy to communicate to the user, so they can use that to reset the password.

<Warning>
  **IMPORTANT**

  The **Login MUST** be the same as the Salesforce e-mail address. For password, use anything easy to remember and after communicating that to the user, **ask them to reset the password to the same one they use in Salesforce,** so it is easier for them to log into both places every day.

  ![](/images/Sales-35.png)

4. In settings, you will select:
   - The Security Profile – this defines what level of permissions this person will have within Amazon Connect. For SDRs and AEs. If you only want to give access to manager views to see agent and queues performance, but no ability to change configurations, you can select CallCenter Manager.

### Step 4 - Update the Call Flow

The Sales Team currently has an [IVR](https://en.wikipedia.org/wiki/Interactive_voice_response) set-up to allow the customers to select the person they want to talk to. Therefore, when adding a new user, you will need to add an [IVR](https://en.wikipedia.org/wiki/Interactive_voice_response) option for that new person and add their corresponding call flow in the Amazon Connect

We have 2 call flows in Amazon Connect:

- **Sales Inbound** - This is the entry point call flow and there is no user specific configuration. This call flow will set to collect and record all of the call information, and it will send records to salesforce. This flow will transfer the call to the Sales CS Transfer Flow
- **Sales CS Transfer Flow** - This flow starts with playing a prompt to the customer that the call is being recorded, and presents the customer with the routing options. Press 1 to sales team, and let the customer know that Customer Success needs to be contacted by email. It has also a Press # option if unsure of who they want to contact. This option will route to the Sales shared queue that is under the SDR team routing profile.

<Note>
  If we want to change CS to be able to receive calls, we need to create a "Press 2" option and create a new branch for the team with their own individual routing menu. Also, each team member will need to have their own queue and profile, like it is set-up for the sales team.
</Note>

If the customer "Press 1" for Sales, a new menu will be presented to the customer to select the sales person they want to talk to. Here is where a new user will need to be added. To add the user:

1. First, add a new option to the Sales Team sub-menu. Find the Sales Split box, click on the three dots and select edit settings.

<Frame>
  ![](/images/Sales-38.png)
</Frame>

2. Add the new person to the text, so the text-to-speech feature will let the customer know that option is available

<Frame>
  ![](/images/Sales-39.png)
</Frame>

3. Scroll down and add a "new condition" showing the option that they need to click to route to the new user. Make sure this option matches what you described above. Click Save.

<Frame>
  ![](/images/Sales-40.png)
</Frame>

4. Each option of the menu above will have a connection to where that option should be routed to:

<Frame>
  ![](/images/Sales-41.png)
</Frame>

To create a new branch, you will need to find each block in the block search on the left side of the screen. Once you find the block, you click and drag it to where you want to place it in the call flow.

<Frame>
  ![](/images/Sales-42.png)
</Frame>

6. Next, you will need to configure the block. To know how to configure each block, look at the configuration for another team member and do exactly the same thing. You will need to configure: (1) the input and output arrows connecting the boxes (blocks);

<Frame>

## Creating a New User in Amazon Connect - Outbound only - Customer Success Team

Use this process if the user will only need to be able to make outbound calls, i.e. you don't need the user to be able to receive calls

The CS team is currently set-up to only make outbound calls. If the customer calls back the number the team used to call them, they will get the menu to press 1 for sales and will hear a message to contact the CS team via email.

For outbound calls only, there is no need to create an individual inbound call flow, so the team can share the same queue and routing profile. If in the future, the team decides to have individual call routing, the team set-up will need to mirror the one for the Sales Team.

### Step 3 - Add users to a Permission Set for Amazon Connect

1. Navigate to Setup \> Manage Users \> Permission Set
2. These are the permissions available
   - **AC\_Administrator** - assign to BI or anyone that will do system configurations - see more details on what is included with this permission in [this link](https://amazon-connect.github.io/amazon-connect-salesforce-cti/docs/classic/installation/02-guided-setup/#add-users-to-a-permission-set) Note, with this permission, when trying to sign into the dialer, the user is redirected to an Amazon Connect Console Set-up Page. To be able to see the dialer, this user needs to sign into Amazon Connect into a separate window and refresh Salesforce, so the dialer will load.
   - **AC\_Agent** - assign to everyone using the phone system (added as a call center user) - see more details on what is included with this permission in [this link](https://amazon-connect.github.io/amazon-connect-salesforce-cti/docs/classic/installation/02-guided-setup/#add-users-to-a-permission-set)
   - **AC\_Manager** - assign to everyone that has a direct report using the phone system - see more details on what is included with this permission in [this link](https://amazon-connect.github.io/amazon-connect-salesforce-cti/docs/classic/installation/02-guided-setup/#add-users-to-a-permission-set)
   - **AC\_AmazonQ** - not needed to be assigned, we don't know what this is for
   - **AC\_CallRecordings** - assign to everyone that should be able to listen to recordings. In our case, it is recommended to enable all agents to listen to recordings for training purposes
3. Choose the permission to be assigned.
4. Choose Manage Assignments.
5. Choose Add Assignments
6. Select the users to assign the permissions, then choose Assign.

   - From the IVR options
   - And their routing branch
2. Delete the user
