<Note>
  **Purpose**

  The purpose of this document is to provide a clear, step-by-step guide on creating a tracker for monthly monitoring of aging invoices. This tracker not only helps identify past-due invoices and record the actions taken to address them, but also includes a dedicated sheet for the monthly aging report. This report is the one presented to the Finance Manager to discuss the aging status, ensuring a systematic approach to accounts receivable management and timely follow-up on overdue payments.
</Note>

## Steps in Creating Aging and Payment Check File

### Step 1. Prepare the new Sheet

Open a new Google Sheet file and name it (MONTH) Aging and Payment Check file. Below are the tabs that should be created in the new sheet:

**A. Sheet 1: Past Due Summary Tab**

a. Export all the invoices for the current month in Rillet. In the past due summary tab, put in this formula: `=UNIQUE(Invoices!A:A)`

This will serve as the list of active customers with invoices. Name the columns as follows:

- Column A - Customer Name
- Column B - Payment Terms
- Column C - Autopay
- Column D - Expected failed

**B. Sheet 2: Terms Tab**

a. Export all the customers in Rillet. Delete all columns and only leave columns B (Name) and V (Payment Terms)

b. Rename this sheet as Terms.

c. Delete all other columns and keep only Column B (Customer Name) and Column V (Payment Terms).

**C. Sheet 3: Expected Fail Tab**

a. Confirm that Sheet 3 contains the list of expected failed PMs from the [PMs Vital AR Details sheet](https://docs.google.com/spreadsheets/d/1Y4X5e5zgA7-Ahyz-bPO7LyKwRVgHReYk/edit?gid=816508357#gid=816508357)

**D. Sheet 4: Autopay Tab**

a. Put in the Import range formula to import the [AR CC Fee/Autopay Tracker](https://docs.google.com/spreadsheets/d/1pEcYzTsw3W2rCsJ0wq2GkrRYHCmcn6eZlZCAfHBlPjc/edit?gid=0#gid=0)

### Step 2. Import AR Aging Report

1. From Rillet Reporting Tab, export the AR Aging Report.
2. Import into the new Aging and Payment Check file as a new sheet.
3. Rename the imported AR Aging Summary Report to Aging (MM/DD). Then, delete the AR Aging Details Report.
4. On Past Due Tracker → Column E1, type header: (MM/DD) and "Notes" on F1

In E2, insert formula: `=IFNA(IF(VLOOKUP(A2,'aging 10/06'!A:A,1,FALSE)<>"", "UNPAID"), "PAID")`

<Note>
  The sheet name on the formula above should be updated to reflect the updated imported aging sheet
</Note>

### Step 3. Identify the Past Dues

1. Ensure Columns A–F are complete.
2. Filter Column B (Payment Terms) → select the term you're currently working on.
3. Filter Expected Failed (Column D) → select Yes - these PMs require manual payment processing.
4. Manual Payment Processing

When processing manual payments for Expected Failed accounts:

a. Log in to the payment processing platform.

b. Select the customer tagged as Expected Failed.

c. Manually run the payment against the saved default method.

d. Update the Past Due Tracker → Column E with status.
