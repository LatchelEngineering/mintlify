### Objective:

The purpose of this quality check is to improve the quality of accomplished tasks in order to achieve individual performance metric goals, as well as the team goal. 

‌

### Audit Checklist:

| **Checklist** | **What should be accomplished to achieve a Yes?** |
| --- | --- |
| Did the Data Support Specialist identified that the customer’s account has active Integration to other PMS? | Latchel has 2 type of customers based on how the data are acquired by the system. For customers who sync their properties and resident information to Latchel from a third party software, there are limitations a Data Support can do to update their account data. Hence, before making any actions, the data support must identify if the customer’s account picks data from another property management software. These software are: Appfolio, Rent Manager, and Propertyware. The assignee must indicate if the customer has an active integration at the High Level Summary in their task worksheet. Failure to indicate this in the worksheet will result in a **NO** on this question. |
| Is that status of the Jira ticket changed appropriately? | Jira tickets’ status can be changed by the assignee or Project Manager depending on it’s current progress.  TO DO - means it has not been started yet IN PROGRESS - the assignee has started and is working to execute the task as required. NEED HELP - when a ticket is changed to this status, the reporter is notified via email. This is applicable for tasks that require attention for something that is blocking the assignee to move forward in completing the task. DONE - use to tag the ticket as complete. As soon as the assignee of the task tag the task to Done the task goes to “TO DO AUDIT” status and will be assigned to the Program Manager for audit. Failure to change the status of the Jira ticket appropriately and as needed will result in a **NO** on this question. |
| Did the data support add the STARTED work time of the day in the Jira ticket? | As soon as the data support set the task to IN PROGRESS, a timestamp must be placed in the comment section that tells the time the task is started. This is to ensure that the data support has changed the status of the ticket accurately.  In addition to that, this will identify the legitimacy of the actual tracked time of completing task. No started work time in the comment section of the ticket will have a **NO** as an answer on this question. |
| Does the worksheet include all pertinent reports necessary to accomplish the task, and for audit purposes? | In every task, a worksheet must be attached/linked in the Jira ticket. The worksheet must include the following: **PM Reports** - these are reports forwarded by the customer to Latchel which will serve as the source of truth in updating the customer’s data in Latchel. **Latchel Reports** - these are reports that contains the data in Latchel. These are exported using **pgAdmin**. There can be multiple Latchel report an assignee might need to crossmatch with the PM reports. Visit [How to: Execute SQL Queries in Pg_Admin](/wiki/spaces/CS/pages/865536465) and [Job-Aid: SQL Queries](https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280/Job-Aid+SQL+Queries) to know more about how pgadmin works. **The actual sorting worksheet** - this may contain a combined report from the PM, and Latchel where the assignee sorts both data for comparison **High Level Summary** - This sheet must contain initial inventory vs final inventory of properties, residents, owners, vendors, or any data in the Latchel console that is/are the focus of the task completed. A worksheet without any of the inclusions stated above will merit a **NO** on this question. |
| Did the data support followed the complete description and instructions of the task? | Before completing the task, the data support must made sure that all line items to do as stated in the job description were accomplished accordingly, specially those configurations that are not stressed obviously to the task but could suffice the need of the reporter if executed. An example of the job description to Add and Prioritized A Service Provider” is: “**Please add and prioritized Scott Adams as the ONLY Vendor to be called for property 1234 Chesters Ave Unit 101.**” Then the action steps to do are 1.) add prioritized the service provider to the property, and 2.) the INHERITANCE of the service provider from portfolio/company must be turned off since the instruction would mean the are no other service provider to be called other than Scott Adams. Failure to adhere and/or if at least there’s one step missed from the job description will merit a **NO** on this question. |
| Is the High Level Summary details accurate in the worksheet? | Depending on the task assigned and in progress, the specialist must use the correct High Summary details as stated in the attached spreadsheet below. The data support specialist must be able to provide accurate details in all fields required, including, but may not always be required, any recommendation to improve the customer’s account to achieve great experience and satisfaction. An example recommendation would be, **“portfolios needs to be updated because there are several properties found that are not assigned to existing portfolios.”**  Failure to adhere, and with inaccurate details in the summary will merit a **NO** on this question. |
| Were there NO multiple revisions on the completed work? | Multiple revisions on any tasks due to inaccurate configurations means inconsistency on work done. If the program manager has found significant and unaligned configurations from the desired outcome based on the task description, the task must be returned back to the assignee with feedback and resolution. A **NO** will be given as a score on this question if multiple revisions were made. |
| Does the data support provide updates of the task’s daily progress in Jira ticket? | For any unfinished tasks (In Progress) which are not to be completed due to more urgent tasks that needs to be attended or will require days to complete, the specialist must provide a comment in the ticket of any progress updates including the time spent on working on the task for the day, property time logged, and/or the reason why the task needs to stay in the In Progress status. If the ticket is placed in NEED HELP, the specialist must provide a brief details in the comment section of the reason why the ticket is placed to Need Help.   
The assignee will merit a **NO** on this question if failed to adhere. |
| Is the time tracked/logged correctly in the Jira ticket? | Every tasks that have been started by the assignee must have a tracked/logged time in the ticket regardless if completed or not. This must be done each time the ticket is touched or if updates were made in the customer’s account and in the related worksheet. The time must be accurately set, based on the start time indicated by the assignee when he/she initiated the work.  Failure to log the time, and inaccurate logged time will merit a **NO** on this question. |
| Did the data support provide recommendation to improve the customer’s account? | Maintain and organize the overall health and accuracy of the client’s account is one of the responsibilities of a data support associate. Hence, for every account updates, a recommendation must be provided in the worksheet if there are anything that can be done/accomplish to the customer’s account in order to fulfill this responsibility. The first step into providing recommendation is by finding any discrepancies on the account on top of the task description:  
An example of discrepancies and recommendation would be but not limited to: 1.) While updating properties, the associate found out that there are properties not assigned to portfolios, hence recommendation is to have a portfolio management as the next task. 2.) Assigned PM at the portfolio is not aligned with the assigned PM user in the property level, hence the right recommendation is for the PM to provide a report that indicates the PM assignment of the portfolios or properties to correct discrepancies. 3.) Multiple PMs are no longer active in the company user page but are still assigned as PM to some properties, hence and update of assigned PM users might be needed. An update with escalation email and contact and notification group may also be needed. The data support associate my merit **NOT APPLICABLE** on this question if there are no discrepancies found. A **NO** will be answered if during the audit the auditor/Project Manager found discrepancies and outdated information but the associate failed to see it, resulting to no recommendation provided. |

### Time Tracking

The total time tracked indicated by the associate in the Jira ticket after completing the task will serve as the Resolution Handling time. The associate must be able to indicate the total days, hours, and minutes they took to complete the task. The average will be computed per week. **The current goal must be 16 hours or equivalent to 2 working days.**

**Note:** The days, hours, and minutes must be converted to decimals. Please check the example below.

## The Scoring

In each of the 10 questions/checklists, the associate must get a YES. **The goal is to get at least 85%.** Below is how the score will be computed.

 (Total # of Yes  / 10 questions) x 100 = Total Score for the Audit

Example: (9 Yes / 10 questions) x  100 = 90% score
