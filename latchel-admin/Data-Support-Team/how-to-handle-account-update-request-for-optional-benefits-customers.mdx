### **Introduction**

Every month prior to invoicing we perform account update task for the following customers:

- DS Huber Property Management
- Cal Property Management
- Urban Coast Properties

Handling account updates for Optional Benefits customers requires a monthly audit of their properties and residents before the invoicing date. This process ensures that only the properties and residents included in the provided report remain active in Latchel. The audit is crucial in verifying that only residents with GL codes are maintained in the system, preventing discrepancies and ensuring accurate invoicing.

The account update request is submitted the Customer Profitability Team every first week of the month via [#account-update-reques](https://latchel.slack.com/archives/C05CGVC9FPV)t, and must be completed at the set deadline of their request.

‌

### **Objective**

This SOP outlines the process for auditing and updating Optional Benefits customers' accounts in Latchel. It ensures that only eligible properties and residents with GL codes remain active, maintaining data accuracy and compliance with invoicing requirements.

‌

### **How to Perform the Audit**

1. Look for the assembly ticket and start the ticket timer as soon as you started handling the ticket.
2. Access the attached report from the Assembly ticket and make a copy of it.
   1. Rename the file using the following format: "Worksheet - ASM-ticket # - PM Company Name" (e.g., Worksheet - ASM-23365 - Cal Property Management).
   2. Move the spreadsheet to the corresponding PM folder in Property Manager Files. [Property Manager Files](https://drive.google.com/drive/u/1/folders/1K-OxCEHlyfI6GNhukN9aNPsZho1yzz7T).
   3. If the original spreadsheet contains reports from previous months, delete them in your worksheet and retain only the report for the current month. Rename the tab to "PM Report".

**Watch Loom Recording here:** <custom data-type="smartlink" data-id="id-0">[https://www.loom.com/share/29142e74e96d47bb990b75adf7ca1d4d?sid=d7e6a19f-0d0a-46e1-a259-ad9c3e77060a](https://www.loom.com/share/29142e74e96d47bb990b75adf7ca1d4d?sid=d7e6a19f-0d0a-46e1-a259-ad9c3e77060a)</custom>

3. Download the latest property and resident report from Latchel using the PgAdmin tool.
4. Use SQL R4 to extract the data. You may find the R4 code block here: <custom data-type="smartlink" data-id="id-1">[https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280](https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280)</custom>
5. In the worksheet you created, import the Latchel data exported from PgAdmin and rename the tab to "Latchel Report".
6. Find the matching property and resident record between the PM Report and Latchel Report.
   1. **Disable** any properties or residents that are active in Latchel but not included in the PM Report.
   2. **Enable** any properties or residents that are inactive in Latchel but are included in the PM Report.
   3. **Move** any residents who are not assigned to their correct unit address in Latchel, based on the PM Report.

7. If any properties or residents are missing in Latchel, **DO NOT ADD** them. These customers have active integration.
8. For any missing data, report them to the CP Associate who requested the ticket at the end of your audit.
9. Once the audit is complete, inform the CP Associate of any discrepancies found, along with the total number of active properties and residents in Latchel after the update. Also, report any data present in the PM Report but missing in Latchel.
10. Stop the ticket timer and add a final internal note in the ticket, including:
    1. The worksheet URL
    2. The final total number of active units and residents after update
    3. Start Time and End Time logged as your working time.
11. Finally, change the status of the Assembly ticket to Closed.
