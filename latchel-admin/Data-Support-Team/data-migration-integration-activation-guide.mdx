## **Introduction**

Some customers already have properties and residents set up in Latchel before deciding to activate an integration with their third-party property management software (PMS).

To avoid duplicate records once the sync begins, the Data Support Team performs a Data Migration project. This involves locating the correct reference IDs for all existing properties and residents in the customer’s PMS and adding those IDs to their matching records in Latchel.

Once these matches are in place, the integration can safely sync all data without creating duplicates.

## **Objective**

The goal of this process is to ensure a smooth and accurate integration setup. By identifying and mapping all existing records early on, we prevent duplicate properties, duplicate residents, and unnecessary cleanup work after activation.

This document outlines when a Data Migration project is needed and the guidelines for submitting a request to activate a customer’s integration.

## **Guidelines in Submitting Request to Activate Integration**

### **A. When a Data Migration Project is Required**

Submit a Data Migration request **only if both** of the following are **true**:

1. The customer already has existing properties or residents in Latchel, and
2. They are now choosing to activate integration with their third-party PMS.

In these cases, Data Support must complete the following before integration activation:

- Locate the PMS reference IDs for all existing properties and residents.
- Prepare **two** CSV files:
  - **Properties CSV:** All properties currently in Latchel \+ their PMS reference IDs
  - **Residents CSV:** All residents currently in Latchel \+ their PMS user IDs
- Submit a request to **Activate Integration** in <custom data-type="smartlink" data-id="id-0">[https://latchel.slack.com/archives/C03P9VB4JK1](https://latchel.slack.com/archives/C03P9VB4JK1)</custom> channel with the CSVs attachment.
- **For PS Team, or CS Team:** Please submit a request for the Data Support Team to perform Data Migration Project in <custom data-type="smartlink" data-id="id-1">[https://latchel.slack.com/archives/C05CGVC9FPV](https://latchel.slack.com/archives/C05CGVC9FPV)</custom> using the **Account Update Request** Typeform.

### **B. When a Data Migration Project is NOT Required**

If a customer has **no existing data** in Latchel:

- There’s no matching or reference ID mapping needed; therefore, the customer’s report with reference IDs is not required.
- **The Product Support Team, or the Customer Success Team can directly request integration activation.**
- The request **only needs the customer’s PMS integration credentials**. You can refer to <custom data-type="smartlink" data-id="id-2">[https://latchel.atlassian.net/wiki/x/AQCPmw](https://latchel.atlassian.net/wiki/x/AQCPmw)</custom> for more information about the credentials needed for different 3rd-party management softwares.
