## **Reminders**

**Few reminders to take note:**

- Use the **Account Update Request** form if the customer is **not disputing** their monthly subscription charges. If they are disputing, use the **Active Unit Adjustment** form instead.
- For the request category, you may **only select one option** from the list below. Each category will show the specific data or settings you can update for the customer. If your request falls under another category, please submit a separate typeform.
  - Company
  - Portfolio
  - Property
  - Activate Integration (Data Migration Request)
- For customers with an **active integration** (via any partner third-party software), requests to **add or bulk import new properties are not allowed**. The form will end with a notice explaining this restriction.
- For Data Migration or Integration Activation requests:
  - If the customer already has data in Latchel, attach a **property and resident report with reference IDs**.
  - If no existing data, these reports are **not required**.
  - Provide accurate details of the customer’s credentials as outlined in the <custom data-type="smartlink" data-id="id-3">[https://latchel.atlassian.net/wiki/spaces/PROD/pages/2609840129](https://latchel.atlassian.net/wiki/spaces/PROD/pages/2609840129)</custom>. Inaccurate or incomplete credentials may delay processing beyond the standard ~10 business days.
- The request to **update residents** will only appear if you selected one of the following:
  - Update Property Status (activate/deactivate)
  - Bulk import new properties
  - Merge duplicates
