### **Introduction**

Property management software like AppFolio and Rent Manager helps landlords and property managers handle tasks such as rent collection, maintenance tracking, tenant communication, and financial reporting. It streamlines day-to-day operations, improves efficiency, and provides tools for marketing, leasing, and compliance—all in one platform.

Integrating Latchel with property management software like AppFolio or Rent Manager allows seamless data syncing, ensuring property, tenant, and vendor information stays up-to-date across both platforms. This eliminates manual data entry, reduces errors, and allows Latchel to handle maintenance requests, work orders, and escalations more efficiently.

‌

### **Objective**

The purpose of this SOP is to provide a clear understanding of how these integrations work, offer concise steps for the Data Support Team to check a customer’s integration, and explain the impact of this feature when compromised.

### **The Property and Resident Modification Settings**

For customers with active integrations, the Property and Resident Modification settings are, and **MUST ALWAYS** be, **disabled**. This ensures that no one can manually add new properties or residents, maintaining the credibility of their data.

To check this settings, go to the **Miscellaneous Settings** in the company details tab. These settings are disabled if the the toggle is **blue**.

### **Benefits of Activating 3rd Party Software Integration**

- **Data Accuracy and Consistency**
  - Prevents manual entry errors, ensuring all property and resident information comes directly from the integrated system. This ensures that data in Latchel matches the data in the customer’s Property Management Software (PMS), avoiding discrepancies between platforms.
- **Coordinated updates for improved efficiency**
  - Automatically updates property and resident information when changes are made in the PMS, reducing the need for manual intervention. This eliminates duplicate efforts by relying solely on the integration to manage data updates, saving time for the customer and support teams.
- **Secured system for integrity and accountability**
  - Protects the integration process from conflicts or errors caused by unauthorized manual edits. This guarantees that any modifications to property or resident data are traceable and tied to the source system, improving auditability.
- **Minimized risks**
  - Reduces the likelihood of compromising workflows or automation tied to accurate data, such as maintenance requests or financial reporting.

### **How to Handle Account Update Requests for Customer’s with Active Integration**

When the Data Support Team receives a request to update a customer’s property and resident data in Latchel, it is essential to verify if the customer’s account has an active integration by following the steps outlined above. If their integration is active, below are the limitations that a Data Support associate must follow.

1. **NEVER add property and resident data manually.** If the person requesting the update to the customer’s data in Latchel, based on the most recent property manager report, explicitly asks to add missing data, the Data Support associate must inform them that the customer’s account has an active integration, and manually adding property and resident data would compromise their account.
2. The Data Support Associate CAN enable or disable a property or a resident following the provided report.
3. Updating resident contact information, move-in and move-out dates, and correcting the spelling of their name is allowed.
4. If the resident is assigned to a different property address or unit in Latchel, assign the resident to the correct property based on the report.
5. Other updates that are allowed to be done within a property are:
   1. assign the owner in the property
   2. assign the property to a portfolio
   3. update escalation contact details, service providers prioritization, severity and responsibility setting, etc.
6. Other updates that are allowed to do within a resident profile are:
   1. update contact settings
   2. update schedule availability
   3. add payment method

**Updating schedule availability**, and **add payment method** are not usually requested to the Data Support Team, but can be done as they do not affect with the integration. Defer these types of requests to the Product Support Team.
