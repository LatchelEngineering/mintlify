**What is Data Migration Project?**

The Data Migration task is usually performed for any Latchel customers that started service before activating their third-party software integration since doing so would create duplicate records in the Latchel database. To fix this, we need to collect and assign the “reference IDs” of the properties and residents from the customer’s third-party property management software into our system, so that the integration knows which data needs to be created or updated.

**ATTENTION:**

The estimated time of completion of this project is between 2-3 weeks depending on the amount of records for matching. It is important that the customer is reminded of this ETA to avoid future escalation.

# CONTENTS

## **OBJECTIVE**

The objective of this document is to provide a step-by-step guide to a Data Support Associate on how to find and assign “ref\_ids” of properties and residents to the matching record of it in the Latchel console.

**Who should be reading this document?**

- Customer Success Team
- Data Support Team

## **THE PROCESS**

### **PART 1: HOW TO MATCH RECORDS**

- Export report from Latchel via pgAdmin using the R4 query. You can copy the query from <custom data-type="smartlink" data-id="id-0">[https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280/Job-Aid+SQL+Queries#Residents-Data](https://latchel.atlassian.net/wiki/spaces/CS/pages/848855280/Job-Aid+SQL+Queries#Residents-Data)</custom>.
- Check the file attachment in the request. Make sure it has the resident report with ref\_user\_id, and the property report with ref\_property\_id and ref\_unit\_id. If the reports are incomplete, escalate the ticket and inform the requester of the missing information needed.
- Go down the list of the properties in the Latchel Property Report sheet and check the Property Manager Property Report sheet for a matching record. Pay close attention to the “address 1” and “address 2” fields, and make the best match you can. 
- For each match you find, copy the “ref\_property\_id” and “ref\_unit\_id” columns from the property manager’s property report into the corresponding columns in the Latchel property report. If you cannot find a matching property or are unsure of a match, leave the “ref\_property\_id” and “ref\_unit\_id” fields blank in the Latchel property report.
- Next, go down the list of tenants in the Latchel Tenant report then check the customer tenant report for a matching record. Some of the tenant names may be misspelled in either file, so do your best to make the most educated decision you can on a match, taking into account the name, email, and phone number of the resident.
- For each match, copy the “ref\_user\_id” column from the customer tenant report into the corresponding column in the Latchel Tenant report. If you cannot find a match or are unsure of a match, leave the “ref\_user\_id” blank in the Latchel tenant file.

**Attention:** There will be instances that a ref\_property\_id is assigned to multiple properties, it is an indication that the property is a multi-unit address. On the other hand, ref\_unit\_ids should be **_unique_** in every unit.

- Any unmatched residents and/or properties **must be disabled** in the Latchel console. While Latchel residents with matching records in the customer resident report but are assigned to incorrect properties, **must be moved or transferred** to the correct properties if applicable.

**PRO-TIP:**

You may use different conditional formatting or create sets of formulas in the spreadsheet to find any matching properties or residents. You may use “VLOOKUP” and/or “INDEX \+ MATCH” as a few of the common formulas that can help you find these matches easily.

The assignee of the task is not required to update the resident contact information that is outdated at the time of the data migration task. The reporter must be advised of these discrepancies. A separate task must be posted to update the residents’ contact details using the Task Title “Update Resident Contact Information”.

For more information about these task titles, please refer to the job-aid: “Task Share Program: Standardized Task Titles”

### **PART 2: HOW TO CREATE THE CSV FILES WITH THE MATCHED RECORDS**

- Once all residents and properties have been sorted and all reference ids of the matching records have been placed in the right column, the assignee must create 2 CSV files. The first (.)csv file contains the Latchel “user\_id” of the residents with their corresponding “ref\_user\_id” copied from the property manager tenant report. The second (.)csv file contains the Latchel “property\_id” with their corresponding “ref\_property\_id” and “ref\_unit\_id” copied from the property manager report.
- From the data migration spreadsheet, go to the Latchel Tenant report and copy the Latchel "user\_id" column and the “ref\_user\_id” column. Make sure you are only including residents that have matching records. Paste these columns in a separate spreadsheet.
- Save it as a (.)csv format file and rename it to indicate that the file contains the residents' reference IDs.

![](blob:https://media.staging.atl-paas.net/?type=file&localId=327bc71e-c164-4b04-833a-1c1dceec33f8&id=979a8546-8c5e-4807-8777-9ac85507272b&&collection=contentId-2507866123&height=616&occurrenceKey=null&width=952&__contextId=null&__displayType=null&__external=false&__fileMimeType=null&__fileName=null&__fileSize=null&__mediaTraceId=null&url=null)

- Once done with the residents' reference ID (.)csv file, proceed on doing the same with the properties.
- From the data migration spreadsheet, go to the Latchel Property report and copy the "property\_id", “ref\_property\_id”, and “ref\_unit\_id” columns into another spreadsheet.
- Save it as a (.)csv format file and rename it to indicate that the file contains the properties' reference IDs.
