### **Objective**

This guide is intended to provide clear steps for stakeholders on how to submit data support requests effectively. By following these instructions, stakeholders can ensure their requests are processed promptly and efficiently via Ask Assembly.

### **How to Submit a Request**

1. In the selected channel, click on the **Workflow** tab at the top, select the corresponding workflow based on the type of your request. Alternatively, you may use the workflow button at the bottom of the channel if available.

For this example, let’s submit an Account Update Request.

2. Once the workflow starts, a form will appear. **Fill it out with accurate and relevant information.** Clearly **outline the course of action** required by the Data Support Team.

Ensure you attach the most up-to-date report by providing a Google Spreadsheet link or uploading the file directly for tasks where the company’s data will be the source of truth. Click **Submit**, after completing the form.

Providing unclear or incomplete information may delay your request.

Below is the list of questions in the workflow.

**i. What is the property management company's name?**

**ii. Choose the PM’s Integration. Reminder: Manual adding of properties/residents is prohibited on PM with active integration.**

a. None

b. Chrome Plug-in

c. Rent Manager

d. Propertyware

e. RentRedi

f. API/Webhook (currently being used by Up and Up)

**iii. What type of account update is this task?**

a. Properties and Residents Listing - choose this option if the # of properties and residents in Latchel must match with the PM report.

b. Property Settings Update - make changes to any settings at the property level. This includes all settings that can be configured or changed at the Details tab of the property level.

c. Residents Update - includes updating contact information, and move-in and out dates

d. Portfolio Settings - make changes to any settings at the portfolio level. This also includes assigning properties to portfolios.

e. Service Provider Update - make changes to any service provider settings, or update service provider listings

f. Global Company Settings - make changes or update any global company settings including but not limited to all settings that can be seen at the Details tab of the company level. Note that we use this option if the settings we want to change or configure need to take effect in all portfolios or properties.

g. Work Order adjustments - close out or cancel work orders in the system.

h. Owner Maintenance Settings or Contact Info Update  - **this includes ONLY updating the owner’s contact information and their maintenance communication settings**. However, please note that if your request is to assign owners to the properties, please choose “Property settings update” since we are talking about changing/assigning owners in the property settings.

i. Data Migration - use this if a customer is migrating to third-party software integration.

- For **Excalibur**, please submit a ticket request to Update their Properties and Residents based on their Latchel Concierge Report twice a month.
  - **The First ticket** request is to be submitted **every 1st Monday of the month**, the **deadline is on Friday of that same week.**
    - For example, if you submitted a request on Dec. 04, 2023, the deadline for the task will be on Dec. 08, 2023.
  - **The Second ticket** request is to be submitted **every 3rd Monday of the month**, the **deadline is on Friday of the same week.**
    - For example, if you submitted a request on Dec. 18, 2023, the deadline for the task will be on Dec. 22, 2023.
- For **GC Realty**, please submit a ticket request to Update their Properties and Residents **every 3rd Friday of the Month**. Indicate a **deadline of Wednesday of the following week**.
  - For example, if you submitted a request on Nov. 17, 2023, the deadline for the task will be on Nov. 22, 2023.
- For **DS Huber and Cal Property Management**, a request must be submitted **every last weekday of the month** which must be **completed within 3 working days of submission.**
  - For example, if the request is submitted on February 29, 2024, the deadline for the task will be on March 05, 2024 since the 2nd and 3rd of March fell on the weekend.
- For **Up and Up**, a request must be submitted **every Monday** which must be **completed by Friday of the same week.**
  - All new reports received within the week must be attached in the active request.
  - To avoid duplicate requests, the existing/active request for this customer is always pinned in the <custom data-type="smartlink" data-id="id-6">[https://latchel.slack.com/archives/C05CGVC9FPV](https://latchel.slack.com/archives/C05CGVC9FPV)</custom>  channel and will be unpinned once complete.
- For **IGC Rents, and Homelab**, these are treated as **Onboarding requests**, hence they must be completed within **48 hours upon receiving their automated reports**.

All customers mentioned above have active integrations via the Chrome plug-in. The Data Support Team can only take action on existing data in Latchel and will base their status on the latest report provided with the request. If data is missing in Latchel, customers should be prompted to run their Chrome plug-in extension to sync the missing data.

We recommend that customers follow the help guide, [**_Navigating the Latchel Data Sync Extension_**](https://help.latchel.com/s/article/Navigating-the-Latchel-Data-Sync-Extension), to set up an automatic background sync.
