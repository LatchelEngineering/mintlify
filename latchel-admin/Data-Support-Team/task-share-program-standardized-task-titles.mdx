# Purpose

**Purpose of this document**

This document will give you the standardized titles with a brief description (default scripting) of each of the tasks submitted in the task share program.

For configuration and titles that require adding the account level, the reporter **must indicate in which account level** the task must be performed. 

**Account levels are the following:**

1. Company Level
2. Portfolio Level, or 
3. Property Level.

**Who should read this document?**

Customer Success Team & Technical Success Specialists

# Content

# **Company Management Tasks**

#### **Add Properties/Residents – (Account Level) - (Company Name)**

**Task Description:** _The task is to manually add or import properties and/or Residents for a PM using the provided report. The PM report can either be imported or manually entered into the console._

#### **Update Properties and Residents – (Account Level) - (Company Name)**

**Task Description:** _This task includes making changes for any discrepancies found in the PM Resident and property report. Making changes includes disabling and enabling properties and residents and updating residents’ contact information if necessary. This title can also be used to update addresses or any settings at the property level._ 

_**Note: DO NOT USE** this title if updating is only required for existing residents’ contact information and does not include making changes or updates on any properties, please use the **“Update Resident’s Contact Information”** title instead._

#### **Assign Properties to Portfolio – (Company Name)**

**Task Description:** _The task is to assign all properties to a specific portfolio based on the PM report provided._ 

**ADDITIONAL TASK (_if applicable)_:** _If the portfolio is not yet existing when assigning properties, the reporter must provide details of the portfolio to be added such as the name of the portfolio and the property manager assigned if applicable._

_**Note:** The reporter must provide more information if there are any special instructions/configurations that must be done to complete the task such as assigning property manager to either at the portfolio level or the property level._ 

#### **(Integration Software) Data Migration Project – (Company Name)**

**TASK DESCRIPTION:** _This task requires matching Residents and property profiles in the Latchel console to the PM report. Ref_ids of matching properties and/or Residents from the PM report must be assigned to the Latchel records._

**ADDITIONAL TASK:** _The Data Support Associate is also required to update any discrepancies found, such as:_ 

* _Move Residents to the correct property where they should be active_
* _Merge any duplicate properties and/or Residents_
* _Disable any properties and/or Residents that are in the Latchel console but not in the PM report_
* _After the data migration task is completed, a request to assign the ref_ids of properties and residents to the console should be posted in the Slack channel **#customer-growth-requests**._

**NOTE:** _If the Residents’ contact information needs updating, another task must be created using the **“Update Residents’ Contact Information”** title. The Data Support Associate is not required to add any properties or Residents that are not found in the Latchel report as they will sync to the console once the integration is complete._

#### **Audit Properties – (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to find all discrepancies with the number of properties in the console versus the PM report. The assignee must provide a complete report of the discrepancies found as per the reporter’s request._ 

_This is usually performed for some examples below but not limited to:_ 

* _properties with no ref_ids after migration_
* _mismatch of ref_ids_
* _an audit in which property inherits company service provider prioritization_
* _audit for properties that are not assigned to any portfolio_

**NOTE:** _This task will not require the Data Support Associate to perform any changes to any properties found with discrepancies. If actions need to be done based on the generated discrepancy report, the reporter must create a separate task using the appropriate task title._

#### **Merge Duplicate Properties – (Company Name)**

**TASK DESCRIPTION**_: The task is to ensure that no duplicate properties exist in the company. Merge any found duplicate following the criteria in merging duplicate properties and/or residents._

#### **Home Assistant Audit – (Company Name)**

**TASK DESCRIPTION:** _The task is to make sure that the list of the properties enrolled or not enrolled to Home Assistant is the same as the property manager’s list._

#### **Audit Work Orders – (Company Name)**

**TASK DESCRIPTION:** _The task is to ensure that the work order report of the property manager matches with the work order existing in the console. This is usually performed on property managers that encounter missing work orders or payment disputes._

# **Account Configuration Tasks**

#### **Update Severity and Responsibility Settings – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _The task is to make any changes or updates of the Severity and Responsibility Setting within a company._ 

**NOTE:** _The reporter must indicate to which level(s) of the account the update is needed. Account levels are company level, portfolio level, and company level. If the update needs to be performed on multiple levels, the reporter must indicate the list of properties and/or portfolios._

#### **Update Budget Limit – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _The task is to update the budget limit (Normal and Hard budget limit) for both emergencies and non-emergencies._ 

**NOTE:** _The reporter must indicate the amount that needs to be placed on the normal and hard limits, and the account level to where the budget limit needs to be updated. Account levels are company level, portfolio level, and property level._

#### **Update Assigned Property Manager – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _The task is to assign/update the correct property manager on every property based on the attached report or instruction provided._

**NOTE:** _Assigning property manager can be performed on any of the account levels, however, the Data Support Associate must execute the action at the property level by either using the “Bulk Update” tool or manually assigning at the property level._

**Add/Remove/Update Pre-Dispatch Instruction – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _The main goal is to add or remove pre-dispatch instructions on any account level._ 

**IMPORTANT TO KNOW:**  _Account levels are company level, portfolio level, and property level. The pre-dispatch instruction must only include “must-do” notes that are vital for the Operations Associate to know before assigning or dispatching a service provider to a work order. The pre-dispatch instruction must not include names of service providers that should be prioritized, gate or door lock codes, names of owners, escalation contacts, or any information that can be configured in any part of the company._ 

#### **Add/Remove/Update Critical Notes – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _The task is to add or remove critical notes._ 

**IMPORTANT TO KNOW:** _Critical Notes can be added to a property, a resident, or a service provider. Generally speaking, critical notes should NOT be used for “must-do” information related to work orders. This type of information should be put into pre-dispatch instructions. Critical Notes are best for providing extra information or contextual awareness for our teams such as a Resident who only accepts communication via text or email. Just like the pre-dispatch instruction, this field must not include any information that can be configured in any part of the company’s account._

#### **Add/Remove/Update Home Warranty Information – (Company Name)**

**TASK DESCRIPTION:** _The main goal is to place a note at the property level indicating that the property has an active Home Warranty outside Latchel._ 

**IMPORTANT TO KNOW:**  _This task can only be configured at the property level. Once a note is placed in the home warranty field, the property manager is notified of any non-emergency work orders created for the property while emergency tickets are handled accordingly._ 

#### **Update Notification User Group – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _The task is to set the correct user group to be notified relating to any maintenance or work order updates._ 

**NOTE:** _The reporter must indicate to which account level the update is necessary and the user group to be assigned._ 

#### **Add Default Access Instructions – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _The task is to add default access instructions on either the company level, portfolio level, or property level. The access instructions must only include gate codes, access door codes, or anything that pertains to ways a service provider can gain access to enter the property vicinity._ 

#### **Add/Remove/Update Escalation Contact List – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _The main goal is to make sure that the correct escalation contact list is added on either the company level, portfolio level, and/or property level._ 

**IMPORTANT TO KNOW:** _Escalation contacts are used to communicate with the property manager for budget approvals or any maintenance-related concerns._ 

#### **Add/Remove/Update Failed Escalation Email List – (Account Level) – (Company name)**

**TASK DESCRIPTION:** _The goal of the task is to update the correct escalation email address on either the company level, portfolio level, and/or property level. If multiple email addresses need to be added, use a comma to separate the email addresses with no spaces._

**IMPORTANT TO KNOW:** _This is used to communicate to the property manager if they are unavailable via call._ 

# **Resident Management Tasks**

#### **Audit Residents – (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to find all discrepancies between the Residents assigned in every property in the console versus the PM report. The assignee must provide a complete report of the discrepancies found as per the reporter’s request._ 

_This is usually performed because of the following customer issues but not limited to:_

* _Doubtful number of active Residents assigned in every property_
* _Residents that are assigned to incorrect properties_

**NOTE:** _This task will not require the Data Support Associates to perform any changes to any of the residents with discrepancies. If actions need to be done based on the generated discrepancy report, the reporter must create a separate task using the appropriate task title._

#### **Add/Update Residents’ Move-in and Move-Out Dates – (Company Name)**

**TASK DESCRIPTION:** _The task requires adding move-in and/or move-out dates on all residents in the console assigned in every property based on the PM report._

**NOTE:** _The Data Support Associate is not required to add a resident if not exist in the console. A report should be generated consisting of the discrepancies found if there are any. The reporter must be informed of these discrepancies. If a follow-up action must be performed, such as adding a new resident, the reporter must create a separate task using the appropriate task title._ 

#### **Merge Duplicate Resident – (Company Name)**

**TASK DESCRIPTION:** _The task is to ensure that no duplicate Residents exist in the company. The assignee must follow the procedure and criteria for merging duplicates._

#### **Update Residents’ Contact Information - (Company Name)**

**TASK DESCRIPTION:** _The task is to make sure that all Residents’ contact information is updated based on the property management report._ 

**NOTE:** _This title must only be used if the reporter identified that the company has the updated list of residents in the console and the existing residents have outdated contact details. If the task includes adding properties and new residents, the reporter must use the **“Add Properties and Residents”** as the title of the task._

# **Service Provider Management Tasks**

#### **Add and Prioritize Service Providers – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _This task requires the Data Support Associate to add/import service providers for a company and prioritize them accordingly. The PM report can either be imported or manually entered into the console._

**NOTE:** _The reporter must state to which account level the service providers should be prioritized. Account levels are company level, portfolio level, and property level._

#### **Update Prioritized Service Providers – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _This task requires the Data Support Associate to update prioritized service providers for a company. Updating service providers may include, editing priorities, removing service providers from categories, or updating skills within each category._ 

**NOTE:** _The reporter must state to which account level the service providers should be updated. Account levels are company level, portfolio level, and property level. DO NOT USE THIS TASK IF THE TASK WILL REQUIRE THE DATA SUPPORT ASSOCIATES TO ADD A NEW SERVICE PROVIDER. Please use **“Add and Prioritize Service Providers”** as the title of the task instead._

#### **Update Service Providers Response Buffer Time – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to make sure that the prioritized service providers are given enough time to respond to any available work orders for them to grab. The response buffer time must be entered in minutes. Example: If the buffer time to be set is 2 hours, then the assignee must enter 120 minutes._

**NOTE:** _The reporter must indicate the account level where the service provider is prioritized. Account levels are company level, portfolio level, and property level._

#### **Add/Remove/Update Service Providers Territory Settings – (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to make sure that the prioritized service providers are servicing the correct territories within the company by adding or updating the zip codes where they are servicing._

**NOTE:** _The preferred service provider list from the property manager must indicate the zip codes that the service provider can service. Once the zip code is set, only properties within the zip code are the properties that can only show the preferred service provider._

#### **Audit Service Providers – (Account Level) - (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to find all discrepancies with the service providers assigned and prioritized at the specified account level. The assignee must provide a complete report of the discrepancies found as per the reporter’s request._ 

_This is usually performed for some examples below but not limited to:_ 

* _service providers with missing skills_
* _service providers that are added to the company but not prioritized_
* _incorrect service provider contact details_

**NOTE:** _This task will not require the Data Support Associate to perform any changes to any service providers found with discrepancies. If actions need to be done based on the generated discrepancy report, the reporter must create a separate task using the appropriate task title._

#### **Turn off Service Provider Invoice Collection – (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is for work orders to move straight to Completed instead of Awaiting Invoice receipt. The assignee needs to turn off the invoice collection of the company’s preferred service provider(s) at the service providers' management section at the company level._

#### **Change Service Provider’s Usage – (Account Level) – (Company Name)**

**TASK DESCRIPTION:** _Service providers' usage identifies the kind of job a service provider will be contacted by the automation. The goal of the task is to make sure that service providers are contacted by the system for work orders based on their preferred jobs. Service Provider’s usage can be set to Never for Emergency Jobs (Non-Emergency), Only Emergency Jobs, and Both Emergency and Non-Emergency Jobs._ 

**NOTE:** _The reporter of the task must indicate the name(s) of the service provider(s) and the account level where the service provider(s) is/are prioritized._

#### **Turn Off/On Service Provider Lock-In - (Company Name)**

**TASK DESCRIPTION:** _This feature in the Service Providers Management section at the company level allows any service provider to remove themselves from the work order if the setting is turned off. The goal of this task is to ensure that the service providers may or may not have access to remove themselves from the work orders that are assigned to them._ 

**NOTE:** _The reporter must indicate the name of the service provider(s) needing the update. This task can only be performed at the service provider management section of the company level._

#### **Add/Update Service Provider’s Contact Information - (Company Name)**

**TASK DESCRIPTION:** _The main goal of the task is to add or update service providers’ contact information in order for them to receive necessary notifications from the console._ 

**NOTE:** _This task should only be used for existing service providers in the console. If a service provider needs to be added to the console, please use the “**Add and Prioritize Service Providers**” task title._

#### **Add Service Providers Critical Notes**

**TASK DESCRIPTION:** _The main focus of this task is to add any critical notes to the service provider profile page. The critical note examples can be, “This service provider only receives text messages”, or “Call or text to reschedule work orders”._

**NOTE:** _This task should only be used for existing service providers in the console. If a service provider needs to be added to the console, please use the “**Add and Prioritize Service Providers**” task title instead._

#### **Turn Off/On VIP Service Provider Settings - (Company Name)**

**TASK DESCRIPTION:** _This feature in the Service Providers Management section at the company level allows any service provider to accept a job without scheduling an arrival time._

**NOTE:** _The reporter must indicate the name of the service provider(s) needing the update. This task can only be performed at the service provider management section of the company level._

# **Owner Configuration Tasks**

#### **Add and Assign Owners – (Company Name)**

**TASK DESCRIPTION:** _The task is to create and/or assign owners to the property level for a PM._ 

**NOTE:** _If setting up or turning on maintenance communication settings is needed, the requester must create another task and use the title “**Turn on Maintenance Communication Settings for Owners**”._

#### **Update Owner’s Contact Details - (Company Name)**

**TASK DESCRIPTION**_: The task is to make sure that owners have the updated contact information for them to receive any notification from maintenance communication settings._ 

**NOTE:** _**DO NOT USE** this task if adding a new owner is needed. Please use the title “Add and Assign Owners” instead._

#### **Turn On/Off Maintenance Communication Settings for Owners – (Company Name)**

**TASK DESCRIPTION:** _The task requires the Data Support Associate to turn on or off maintenance communication settings on every owner of a company. The requester must specify which of the 3 communication settings must be turned on or off._

**IMPORTANT TO KNOW:** _There are 3 Maintenance Communication Settings for Owners, and these are the following:_

* _**Work Order Created** - toggle this on if the owner wants to be notified of any new work orders created for any of their properties._
* _**Pre-Approvals** - to notify the owner that approval is needed for a new work order._
* _**Budget Approvals** - turn this on if the owner wants to be notified if a budget increase is needed for a work order._
