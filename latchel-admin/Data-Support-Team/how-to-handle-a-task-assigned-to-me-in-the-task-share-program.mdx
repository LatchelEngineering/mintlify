# Objective

**Purpose**

The purpose of this document is to guide you on what to do when you receive a task from the Task Share Program, including: how you learn about having a task assigned to you, how you access and navigate the task, how you understand what is being asked from you, how you keep the team updated on your task status, how you ask for help, and how you mark your task as completed.

# Content

**What is happening prior to this process?**

The Program Manager has posted in the Task\_Share\_Team channel that next sprint plan has been finalized and certain team members have been assigned a task. The Program Manager will also include a link to the "[Backlog](https://latchel.atlassian.net/jira/software/projects/TS/boards/12/backlog)" page of the Task Share Program, which includes the current sprint, next sprints, and items that we have not assigned yet. 

## Step-by-Step Guide

### **Step 1. How do I know when a task is assigned to me and how do I find it?**

1. When a new task is assigned to you, the Program Manager will post in the Task\_Share\_Team Slack channel that the next sprint plan was finalized, tag everyone that has a tasked assigned, and provide a link to the  "[Backlog](https://latchel.atlassian.net/jira/software/projects/TS/boards/12/backlog)" page.
   1. As you get tagged, you will have a Slack notification to check the Slack channel
   2. You will click on the link provided in the Slack message to open the backlog page
2. In the Backlog page, you will see three sections:
   1. (A in the screenshot below) shows the tasks in the current sprint
   2. (B in the screenshot below) shows the tasks that are being planned for the next Sprint. In the future, we maybe able to have more sprints planned ahead 
   3. (C in the screenshot below) shows the tasks that were requested but not assigned yet (backlog items)
3. To find the tasks assigned to you, you can look at the "assignee" initials on the right side of the task. In this case, it shows JP, that stands for Jennifer Pascon
   1. The system will create those initials automatically. If you are not quite sure which are your name initials, open the task that you suspect it may be yours to confirm.
      1. To open the task, click on it and a new section will show up on the right side of your screen with more information about the task
      2. Look for the "assignee" field to see the name and the initials
      3. After the first time you look at this, you will be able to see the initials that the system has created for you and spot all of your other tasks easily
   2. Now you can go back to all of the other tasks in the backlog page and see the ones that have your initials

### **Step 2. How do I navigate the task details screen?**

1. In the backlog page, click on the name of the task to open the task detail section on the right side of the screen
2. The top line of that section will have icons with the following information and options
   1. Task Type - This is a way to categorize the task so we know which Latchel Products or areas of the company this task will be impacting or supporting. This will be assigned by the requester when creating a task and confirmed by the Program Manager
   2. Task Identification Number - TS stands for "Task Share" and the number is a sequential numerical ID assigned to the task. Requesters and Program Manager may use "TS-5" as a way to refer to the task. Look at this as the task "short name" 
      1. Note: if you click on the Task Identification Number, this task detail section will expand to the whole screen, which may be easier to navigate
   3. Watching Options - Here you can add someone to "Watch" your task (or someone can sign up for it, i.e. the requester). The system will send notifications to "watchers" when there are changes made to that task. If you click on the eyeball, you will see the name of the people watching that task. You can also add names.
   4. Sharing Options - This is used to send an email with the link to the task or other information. We won't use this as we are communicating via Slack
   5. Close this section - if you want to close the section, just click on the "X" icon (this won't delete or close the task)
3. Next, you will see the Task Name and a menu of actions that can be taken:
   1. Task Name - This should be a short sentence showing what the task is asking you to do
   2. Attach a file - Requesters can use this to attach files to the task, or you can use this to attach files to share back with the requester
   3. Add a task child - "Task child" are sub-tasks to this one that may have been assigned to someone else. If a child task is added, you will see it further below in the task details section. You can click on the task number to open that task detail page
   4. Link to another task - This is used when there is dependency between tasks. When tasks are linked, you can specify that dependency. For example, this task is "blocked by" another task. If a task is linked, you will see it further below in the task details. You can click on the task number to open that task detail page
   5. Link to a Confluence Page – This is used to add a link to a Confluence page. When this link is added, you will see the link to the page further below in the task detail section. This is one way the requester may use to share a SOP with you.
   6. Add other Jira Apps - Jira is the name of the software we are using to manage tasks. This program has several other applications that can be added, if we choose to.  You won't be using this.
4. Next you will see the "Task Status" and "Task Description"
   1. Task Status: We have 4 task status to show the level of progress made on the task. To change status, you can click on the arrow down, or go to the [Kanban board](https://latchel.atlassian.net/jira/software/projects/TS/boards/12) and move the card (click and drag it to the appropriate status column)
      1. To Do: It is the initial status. This is what you will see when the task is just assigned to you. 
      2. In Progress: When you start working on the task, move it to this status. You should keep in this status until it is completed. However, if you are facing an obstacle and cannot continue to move forward, you will move it to "Need Help" and [request help in Slack](https://latchel.atlassian.net/wiki/spaces/LTG/pages/867860584/How+to+Use+the+Task+Share+Team+Slack+Channel)
      3. Need Help: Use this when you are "stuck" and can't move forward. Moving the task to this status will help our team to see how many tasks are in this situation
      4. Done: Move the task to this status when you have completed it and have sent the output to the requester (if any)
   2. Description:
      1. Here is where the requester should explain in details what they need your help with. Basically, the description is an explanation of WHAT needs to be done. The instructions on HOW to do the task will come in a SOP that could be linked as a Confluence page, or the link could be provided in the description section. 
      2. This is the template being provided to requesters to include in the description field:
         1. What needs to be done?
         2. Why is this important? What is the impact of this task?
         3. How should this be executed? Link to SOP:
         4. How long do you estimate this task will take? 
         5. Which skills you anticipate will be needed to complete this task? 
5. Next you will see :
   1. Priority - This shows how the task was prioritized when compared to others. High priority tasks will be assigned first when planning a new sprint
   2. Start date - This is when we are planning the task to be started, which will most likely match the date of the start of the sprint
   3. Time Tracking - Here is where you will track how much time you have spent working on the task. This will be a great information for us to know if the time estimated by the requester is becoming reality. We will use this information to improve our planning. It will also help us to see how much time we are spending on doing tasks for each Latchel Product
      1. To log the time you spent on it, click on this field and a new window will open for you to type the time you spent on it. Each time you work on it, you can add more time. [Step 4](#Howto:HandleaTaskAssignedtomeintheTaskShareProgram-step4) of this SOP will provide more details on this
   4. Assignee - Shows who the task was assigned to
   5. Due Date - Shows when this task needs to be completed. This will most likely match the end date of the sprint. However, we may have tasks due after the sprint is over, which means we are being able to work ahead of the requester need
6. Next you will see a few other fields for management purpose:
7. And finally, you will have the activity log section. There you can see:
   1. When the task was created
   2. When the task was last updated
   3. Comments posted by you or others
      1.  if you click on the drop down menu besides the "comments", you can change that to "History" that will show all changes made to this task, or "work log" that will show each time you have worked on this task and logged time in the "Time Tracking" field

### **Step 3. How do I understand what is being asked from me?**

1. Read the task name and the task description (as described in the previous section)
   1. The task name should show a short description of WHAT needs to be done
   2. The task description will show more details on WHAT needs to be done and HOW to do it (usually via a link to a SOP, which could also be provided via a link to a Confluence page)
2. Open link to SOPs to learn HOW to do the task
   1. The link should be provided either in the description field or via a Confluence page link (as described in the previous section)
3. Check if there are other tasks linked to this one and if any of them could be a dependency
   1. Check if there is a "child" task or a "linked" tasks (as described in the previous section)
4. Check start date and end date and verify if that looks feasible to you
5. If you have a question or a concern, use the Slack channel to ask for help: [How to: Use the Task Share Team Slack Channel](https://latchel.atlassian.net/wiki/spaces/LTG/pages/867860584)

### **Step 4. How do I keep everyone updated on my progress?** 

1. When you start working on the task, you will change status from "To Do" to "In-progress"
   1. There are two ways to do this:
      1. In the task detail page section, click on the drop-down menu besides "To Do" and choose the next status
      2. In the [Kanban Board](https://latchel.atlassian.net/jira/software/projects/TS/boards/12), click on the task and drag the task card to the appropriate status column
2. If you run into any question or problem, change task to "Need Help"
3. Every time you work on the task, you will also be required to  log the time worked. 
   1. For example, if you worked 15 min on the task and had to go do something else, log that time right away. Next day, you go back and work for another 10 min and you will add those 10 min. The tool will add up all of those time intervals and keep track of the total time worked on the task. See example below
   2. To log time, click on the clock in the Time Tracking field and a new window will open:
      1. In the "Time Spent" field, type the time you have worked on the task. Type it in minutes, for example: 15m, which means 15 minutes
      2. In the "Time Remaining" field, initially type the time estimate provided in the task description minus the time you have already worked. For example, you just started working on a task that the requester says it will take 30 min. You just worked on it for 10 min. You will update the Time Tracking to show Time Spent = 10m, Time Remaining = 20min
      3. As you work through the task, if you think that estimate is incorrect and needs to be adjusted, you can go back and enter your new estimate of how much time you think you will still need to complete the task. The time tracking will update to show the total time you have already worked on it and how much time you still need to complete the task
4. Add comments to the task summarizing what you have done that day and what you need to do next. If you decided to update the time remaining, also include it in your notes
   1. To add a comment, scroll down to the comments section within the task detail page
   2. Type your comment and click on "save"

### **Step 5. How do I ask for help?**

1. If you need help with a task, you will:
   1. Move the task to "Need Help" status as described in the prior section
   2. Reach out to the team via the slack channel using the appropriate workflow, as explained in this SOP: [How to: Use the Task Share Team Slack Channel](https://latchel.atlassian.net/wiki/spaces/LTG/pages/867860584)

### **Step 6. How do I mark my task as complete?**

1. If you have completed the task, you will:
   1. Make sure you have logged all time you have spent on it
   2. Refer to the requester SOP to see how the request wants to receive information back, if there is a need for it. In this case, follow the SOP to send the information requested
   3. Move the task status to "Done" and celebrate! <custom data-type="emoji" data-id="id-0">:slight_smile:</custom>
