### **Introduction**

The Data Support Team receives automated reports from customers, which consist of an owner directory and/or a property-resident directory. Each of these tickets must be completed according to the prescribed cadence.

The Data Support Team receives automated reports from the following property management companies:

* IGC Rents 
* Homelab Property Management
* Up and Up

_Note that this list may change._

### **Objective**

The objective of this document is to provide a step-by-step guide on how to process and manage each customer’s automated report, ensuring that their most up-to-date list of properties, residents, and portfolio and/or owner assignments are accurately maintained.

### **Property Management Companies with Automated Reports**

#### **A. IGC Rents**

The automated reports from this customer are sent to the Data Support Team in a daily basis. Their report consists of Owner Directory, and Property-Resident Directory. 

Here’s how to complete and respond to their automated reports:

1. Always use the most up-to-date report attached within the most recent Assembly ticket received. 
2. The properties of this customer are well-managed based on their portfolio assignment which are named after their owners.
3. Export the following reports from Latchel via PgAdmin. These exported data will be use for comparison to their automated reports.

    1. Property-resident report
    2. Owner Report
    3. Portfolio report
    
4. Start working with this sequence:

    1. Compare properties and residents between their automated report, and Latchel exported report. Find the discrepancies.
    
        1. Active properties/residents in Latchel that are not in the automated report must be marked as “to be disabled”.
        2. Missing properties/residents in Latchel that are in the automated report but not in Latchel must be marked as “to be added”.
        3. Disabled properties/residents in Latchel that are in the automated report must be marked as “to enable”.
        4. Residents with outdated contact information must also be checked and updated. 
        5. Residents’ unit assignment on multi-unit properties must also be considered. Make sure that the resident is active to their respective unit in Latchel based on the automated report. Otherwise, they must be moved in to the right unit.
        6. Residents assigned to multi-unit properties must be moved out from old properties instead of being disabled. Remember, disabling a resident profile will deactivate them in all of their assigned units.
        
    2. Check property owners and assign the owner to the property based on the owner directory report. 
    
        1. Add missing owners in Latchel, and assigned them in the property.
        2. There is no need to disable an owner who is not in the PM report. If the property is still listed in the property-resident directory, but the owner of the property is not listed in the owner directory report, we need to confirm the ownership of the property to the PM.
        3. Update owner contact information. 
        4. Create a portfolio for any new owners that is not yet in Latchel.
        
    3. Finally, assign the properties to the owner portfolio.
    
5. Once done with processing the reports, close the assembly ticket. Make sure to add your internal notes that the ticket has been processed with the final count of active properties and residents.

**B. Homelab Property Management**

The automated reports from this customers is sent to the Data Support Team in a daily basis. Their report consists of Owner Directory, and Property-Resident Directory but in a separate ticket. 

Follow the same workflow as IGC Rents. The only difference is that this customer does not have owner portfolio assignments. However, all properties must have assigned owners based on their current owner directory report.

**C. Up and Up**

The Data Support Team receives partial property-resident report from this customer. Their automated report only consist of properties and residents that are recently added to their system.

Take appropriate action, by adding properties/residents if they are not yet in Latchel. Enable them if existing but only previously disabled.

### **Escalating Discrepancies**

1. For properties without an assigned owner or any information that needs confirmation from the PM, follow the standard escalation procedure and escalate the issue to the PS Team. When escalating, ensure that all data requiring validation is compiled in an Excel or CSV file. If the data is a short list, an Excel or CSV file is not necessary but list the discrepancies in bullet form. Make sure that the ticket status changed to ESCALATED.
2. Make sure to keep an eye on the escalated ticket. Take note that if the child ticket has been responded to, the parent ticket will automatically set to NEEDS RESPONSE status.

    1. Make appropriate action on the discrepancies. Once done, you may close the ticket. If additional information is needed after the first child ticket is closed, create another child ticket to escalate to the PS Team until ticket is resolved.
