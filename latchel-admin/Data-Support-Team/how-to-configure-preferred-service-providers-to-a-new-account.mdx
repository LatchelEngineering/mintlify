# **Objective**

**Purpose**

The purpose of this document is to provide an overview process on how to set-up Preferred Service Providers to a new Property Management account and provide links with detailed process for each step of the process.

# **Content**

**What is happening prior to this process?**

The customer has created their account in the console using an account creation link provided to them by the Sales team. 

The Sales team has also requested Service Provider information by sending them our spreadsheet and / or requesting access to their database with that information 

# **Step-by-Step Guide**

# **Process Overview**

1. Once the Property Management account has been set-up into the Latchel Console, you will be able to start adding and configuring Preferred Service Providers.
2. There are multiple steps involved in fully setting up Preferred Service Providers.
   1. The process starts with collecting basic information from Property Managers on the Service Providers they use and which skill categories they provide.
   2. That information will be collected on a template that is easier for customers to fill in and we will need to convert that information into a format that can be uploaded into the system.
   3. After all service provider accounts are created in the system, you will be able to go to each one of those to finalize configuration and customize.

# **Step 1**- **Get Preferred Service Provider Information**

There are multiple ways to collect information on Preferred Service Providers: (1) Property Managers can provide a data extract from their current system or they can provide their login information to us and we will do the data extract; (2) Property Managers can fill in the spreadsheet we provide to them (see templates in this [folder](https://drive.google.com/drive/folders/1fPMYgh3niGJUwrAZFBP5dFtOeIkeWzlP)); (3) Property Managers can provide any other spreadsheet they use to keep track of their service providers information

The initial request for Service Provider information is done by the Sales team before the onboarding meeting. If this information is not received, please follow-up with the Property Management company using the template in this link: [(Draft) Job-Aid: Onboarding Flow & Communications](https://latchel.atlassian.net/wiki/spaces/CS/pages/851247114)

# **Step 2**- **Prepare Preferred Service Providers Data for Upload**

1. Because the information on Preferred Service Providers can be provided to us in different formats and the Latchel Console has some requirements to enable the data upload, you will have to make some adjustments to the data to meet those requirements.
   1. Overall, the data will need to be converted into another spreadsheet showing the information of each Service Provider on each line. This spreadsheet needs to be saved in ".csv" format. The spreadsheet does not need to have "headings" or field names, but it is recommended to use the first line to show the field name so you can use it to make sure you are matching the information correctly in the upload tool, as described in the [step3](#Howto:ConfigurePreferredServiceProviderstoaNewAccount-step3).
   2. Therefore, the spreadsheet should show the field name in the first line (for example: company name, company phone number, company email address, etc), and each other line will contain the information for one Service Provider per line.
      1. Note that the field name in the spreadsheet does not need to exactly match the field name options you will see in the Latchel Console, but you need to make sure that the information in the subsequent lines are related to the field name, so the information populated in that section of the Service Provider profile is correct. See example below: 
   3. While preparing the spreadsheet, there are a few rules that need to be followed otherwise the upload tool will give an error message:
      1. Skill Categories
         1. If there are multiple Skill Categories for one Service Provider, you will list them separated by pipe symbol "|" and without spaces between the pipes and the skill name. For Example: Plumbing|Appliances 
            1. Note that there is no space between the pipe and the word "Appliances"
            2. Note that this pipe is key to separate Skill Categories that has a comma in their names. For Example: Cleaning, Sanitation & Restoration Services. If you were to upload this skill with Plumbing and Electrical, the field of the importer will need to be: Plumbing|Electrical|Cleaning, Sanitation & Restoration Services
         2. If there is a typo in the Skill Category name, or you are attempting to upload a Skill Category that we don't have in the system, you will get an error
            1. Please make sure you have it typed correctly. Consult [↔ Job Aid: Service Provider Skills and Definitions](https://latchel.atlassian.net/wiki/spaces/OPS/pages/791119550) to see all current Skill Categories listed in the system.
      2. Phone and email formats are not enforced, so you can enter it anyway
2. If the input data is coming from a system extract, you will follow the SOPs listed in [step1](#Howto:ConfigurePreferredServiceProviderstoaNewAccount-step1) to change the input data into the required format to upload into Latchel Console
3. If the input data is coming from the spreadsheet we provided  (see templates in this [folder](https://drive.google.com/drive/folders/1fPMYgh3niGJUwrAZFBP5dFtOeIkeWzlP)), use the following SOP to change the input data into the required format to upload into Latchel Console: [(Draft) How to: Convert our Preferred Service Provider Information Template into "Latchel Imports" Required Format](https://latchel.atlassian.net/wiki/spaces/TST/pages/889947002)

# **Step 3**- **Upload Preferred Service Provider Information to Property Manager Account**

1. If the list of Service Providers is short (i.e. 1 or 2) companies, you can create the Service Provider profile individually in the Latchel Console by following this SOP: [↔ How To: Onboard a New Service Provider](https://latchel.atlassian.net/wiki/spaces/OPS/pages/794034545)
2. If the list of Service Providers includes more than 2 companies, use the upload process described below: 

Once the spreadsheet has all of the Service Providers information matching the template described on step 2, follow this process to upload the information into the Property Manager account in the Latchel Console:

1.  Go to the Property Manager Profile Page: [How to: Access the Property Manager Profile Page](https://latchel.atlassian.net/wiki/spaces/TST/pages/813729656)
2. Click on the "three dots" icon on the up right side of your screen
3. You will see a configuration menu. Click on "Import vendors"
4. The importer tool will open. Click on the "Select CSV file" button and navigate to the folder in your computer where the spreadsheet you created on [step2](#Howto:ConfigurePreferredServiceProviderstoaNewAccount-step2) is saved.
5. Select the ".csv" file with the Service Providers' information and click on "Open"
6. The information you have in the spreadsheet will populate in the Latchel Console, however, you will notice that some of the field names populated in the Latchel Console (line A) doesn't match what you have in your spreadsheet (line B). In that case, you can change the field name in the drop-down menu
7. To change the filed name, click on the drop-down menu and select the field name that reflects the information you have in that column of your spreadsheet
8. Select the appropriate field names to match the information in the spreadsheet and click on "Save Vendors"
   1. Note 1: After you have matched the field names, you can exclude that line so that information is not uploaded as a new service provider. However, if you forget to do that, you can simply delete this service provider as it will be described later in this process
   2. Note 2. If you have "empty" lines in the spreadsheet, i.e. a line added but without any data, you will need to delete those; otherwise you will get an error message saying "The following rows are missing a required field". To delete those "empty" cells, click on the "trash can" icon:
9. You will get a confirmation that the data was successfully imported. From there you can track the progress of the import or do another import
10. To double-check if the information was uploaded correctly, return to the Property Manager Profile and go to the "Vendor" tab and "Vendor Management" section. There you will see the list of all Service Providers you had in your spreadsheet
    1. Verify that all Service Providers listed in the spreadsheet is showing up on the list in the Latchel Console
    2. Verify that all Skill Categories listed in the spreadsheet for each Service Provider were correctly uploaded. If not, you will need to edit the Service Provider Profile to add any Skill Category missing: [↔ How to: Add or Remove Service Provider Skills](https://latchel.atlassian.net/wiki/spaces/OM/pages/791281864)
       1. Note that we don't have skill categories for Handyman or General Contractor, so those won't be automatically added and you will need to add all Skill Categories that are applicable to them and remove the skills within those categories that they are not capable of handling. Follow this SOP for guidance on [(CS/Admin) How to: Set-up Skills for Handyman and General Contractors](https://latchel.atlassian.net/wiki/spaces/CS/pages/796623267) . 
          1. Note:  Although we don't have those skill categories (Handyman and General Contractor) in the system, we are collecting information from Property Managers this way because it is easier for them.
       2. Note that the last line has the information from the spreadsheet headings (the field names) that was used to help matching the information in the spreadsheet to the fields of the Latchel Console. This could happen if you forgot to delete that line during the uploading process. You can delete that line by clicking on the "three dots" on the right side and selecting "Remove Preferred Vendor" 

# **Step 4**- **Track Progress of your Import (Upload)**

1. To check the list of imports (uploads) that are being processed by the system, check this link ([https://stage.latchel.com/admin/imports](https://stage.latchel.com/admin/imports)) 
2. To see the progress details for a specific import, click on the file name and another page will open. There you can see how many items are pending and errors that happened during the import. To return to the list of imports, click on "All Imports" link at the top left of the page.

# **Step 5**- **Add Preferred Service Providers to a Portfolio**

If this Property Manager has Portfolios of properties, there might be a need to specify which service providers should be used for this portfolio. If this process is followed, only the Service Providers selected will be made available to be assigned in the tool to assign Service Providers, i.e. setting up Service Providers at the Portfolio level will override the list of Service Providers set-up at the company level for this specific Portfolio.

To set-up Service Providers at the Portfolio level, follow the steps below:

1. Go to the Property Manager account, click on the "Portfolios" tab and on the name of the portfolio you want to modify
2. The profile page for that portfolio will open. Click on the "Vendors" tab
3. You will see all of the Skill Categories that this Property Manager has Preferred Vendors. For each Skill Category, click on "\+Include Vendor" to see all of the available Service Providers within that Skill Category.
4. Click on "\+ include" to add this Service Provider to the list of providers that can work on this Portfolio. 
5. Next you will be able to select which skills within that category this Service Provider is approved to use for this Portfolio. Select or remove skills and click "Save"
   1. You can always go back to edit those skills or remove this Service Provider from this Portfolio. To do that, click on the "three dots" on the right hand side of the Service Provider you want to edit and select the appropriate option.

# **Step 6**- **Add Preferred Service Providers to a Property**

If this Property Manager has a specific requirement for a property to be only serviced by a specific group of Service Providers, you will need to set-up the Preferred Service Providers at the property level. If this process is followed, only the Service Providers selected will be made available to be assigned in the tool to assign Service Providers, i.e. setting up Service Providers at the Property level will override the list of Service Providers set-up at the company and portfolio level for this specific property.

To set-up Service Providers at the Property level, follow the steps below:

1. Go to the Property profile by clicking on "Property" on the top menu of the Latchel Console, typing the address information and clicking on the link. 
   1. Note, if you don't remember the address or you are not finding the property this way, go to the property manager account and click on the "Properties" tab to see the list of properties under that account 
2. The property profile page will open. Click on the "Vendors" tab to see a list of Skill Categories that this Property Manager has Preferred Service Providers in the account. Click on the "\+Include Vendor" button to see a list of Service Providers that have that Skill Category
3. Click on the "\+Include" link to add this Service Provider to this Property
4. Next you will be able to select which skills within that category this Service Provider is approved to use for this Property. Select or remove skills and click "Save"
   1. You can always go back to edit those skills or remove this Service Provider from this Property. To do that, click on the "three dots" on the right hand side of the Service Provider you want to edit and select the appropriate option.

# **Step 7**- **Configure Preferred Service Provider Accounts**

Each Service Provider will need to be configured for the following items:

1. **Invoice Collection**
   1. Define if we will collect invoices from Preferred Vendors or if they will send those directly to the Property Manager
      1. When this setting is turned off, Work Orders will move straight to Completed instead of Awaiting Invoice Receipt and Service Providers are expected to send their invoices directly to the Property Manager. If this setting is turned on, we will collect invoices for them and send to the Property Manager for payment. In this case, Work Orders will move to Awaiting Invoice Receipt.
         1. To configure this item, go to the Property Manager profile page, click on "Vendors" tab and "Vendor Management" Section. Then, click on the the "three dots" icon on the right side of the Service Provider you want to configure and select "Preferred Vendor Settings" 
         2. Click on the Toggle button to turn it on or off
2. **Vendor Lock-in**
   1. Define if this Service Provider should be locked to a work order or not. If a Service Provider has this setting turned on and is assigned to a work order, the service provider won't be able to remove themselves from that work order, i.e. the work order will need to be completed by that Service Provider. 
      1. To configure this item, go to the Property Manager profile page, click on "Vendors" tab and "Vendor Management" Section. Then, click on the the "three dots" icon on the right side of the Service Provider you want to configure and select "Preferred Vendor Settings" and click on the Toggle button to turn it on or off
3. **Service Provider Profile (other settings)**
   1. Each Service Profile will have their own account within our system. Their profile page will have multiple sections that we can enter additional information that we were not able to include in the upload (for example, their 24/7 availability) or that were provided by the Property Manager verbally during the onboarding meeting, or sent to us in a follow-up email, or provided in the "Additional Information" column of the Service Provider data collection spreadsheet we sent to them.
   2. Because those additional information were not included in the upload, you will need to navigate to the Service Provider profile page and add them manually. Refer to [↔ How to: Navigate Service Provider Profiles](https://latchel.atlassian.net/wiki/spaces/OM/pages/791183477) for guidance on each section 
4. **Configure Service Provider Skills Categories and Priorities**
   1. **Service Provider Skills Categories and Skills**
      1. Each Service Provider is able to provide a list of services and those are described as "skills" in the Latchel Console. "Skills" refer to the abilities or services that a Service Provider offers to their customers. Skills are very specific to what needs to be done to resolve the maintenance request and will be grouped into categories. To learn more about the skills available, refer to [↔ Job Aid: Service Provider Skills and Definitions](https://latchel.atlassian.net/wiki/spaces/OPS/pages/791119550)
         1. If there are new skills you want to add to the list, please follow [↔ How to: Request Changes or Maintenance to Vendor Skills](https://latchel.atlassian.net/wiki/spaces/LTG/pages/794525751)
      2. To configure the skills applicable to each Service Provider, follow this SOP: [↔ How to: Add or Remove Service Provider Skills](https://latchel.atlassian.net/wiki/spaces/OM/pages/791281864)
         1. When setting up a new Service Provider, you will include the Skill Categories applicable to that Service Provider during the upload process ([step3](#Howto:ConfigurePreferredServiceProviderstoaNewAccount-step3)). As mentioned in that section, there are some limitations on which Skill Categories can be uploaded, so you will need to add some Skill Categories and their underlying skills manually. 
         2. If a Service Provider was market as a "Handyman" or "General Contractor" they will need to be assigned multiple Skill Categories and only certain skills within each Category will be applicable to them.  Follow this SOP to adjust their skills appropriately: [(CS/Admin) How to: Set-up Skills for Handyman and General Contractors](https://latchel.atlassian.net/wiki/spaces/CS/pages/796623267)
      3. Unless we get specific information from the Property Manager on what skills that Service Provider does not have, we will select all skills within that category to that Service Provider
         1. Note that "not having" a skill is different than " not wanting" a Service Provider to execute a certain type of repair. If the Service Provider is able to perform that skill, include the skill in their profile. You will be able to "disable" this skill for this Property Manager in specific during the Property Manager prioritization process.
   2. **Preferred Service Providers Prioritization** 
      1. When adding a new Service Provider, we will select the skills ([↔ How to: Add or Remove Service Provider Skills](https://latchel.atlassian.net/wiki/spaces/LTG/pages/791281864)) they are able to perform. For Preferred Service Providers, the skills added in their profile will not automatically be added to all Property Managers that have them as Preferred Vendors. So, after adding a Service Provider as “Preferred” to a Property Manager, you will need to specify which priority this new Preferred Service Provider will be given (within each Skill Category) for that Property Management account and which skills they will use in that account. The same process needs to happen if you update a skill in the Service Provider Profile, i.e. the prioritization list also needs to be updated.
      2. To add a Preferred Service Provider to a Property Manager Priority List and prioritize them within a skill category, follow this SOP [(PREMIUM) How to: Add or Remove Service Providers and Skills from a Property Management Priority List](https://latchel.atlassian.net/wiki/spaces/LTG/pages/791347980/%28PREMIUM%29+How+to%3A+Add+or+Remove+Service+Providers+and+Skills+from+a+Property+Management+Priority+List) 
   3. **Property Manager Skill Prioritization**
      1. Here you will be able to specify which skills Property Managers want each Preferred Service Provider to be able (or not) to perform on their work orders
      2. Follow this SOP to add/enable or remove/disable Service Provider skills for a specific Property Manager account [(PREMIUM) How to: Add or Remove Service Providers and Skills from a Property Management Priority List](https://latchel.atlassian.net/wiki/spaces/LTG/pages/791347980/%28PREMIUM%29+How+to%3A+Add+or+Remove+Service+Providers+and+Skills+from+a+Property+Management+Priority+List) 
         1. Note: As a result of this configuration, you may see that a Service Provider works with a certain skill, but it ends up not showing up in the list of available Preferred Service Providers for a certain work order because that Property Manager removed that skill from their list. It is important to highlight that if a Property Manager removes a skill in their prioritization list, that skill will not be removed from the Service Providers' profile, so they can still be assigned for that skill for other Property Management companies.
