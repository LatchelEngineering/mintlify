Slack Channels:
1.
#ps-team-chat - Private team slack channel
Morning Standup - Scheduled message sent in the AM to share queues and workload
volume.
Pulse Check - Scheduled message sent in the afternoon to check in on queues and see
where support may be needed.
ASM Template Request - Workflow to submit a request for a template to be added to
salesforce to be used for cases.
DD Conclusion Workshop - Workflow to fill out original DD Conclusion and then share a
new updated, PM-facing conclusion. Used to help PS improve communication skills and
how we present these conclusions in a digestible manner.
PS SOP Request - When PS cannot find an answer or next steps, they should use this
workflow to submit for a new SOP or add an update to an existing SOP.
FYI Heads Up PMs - Scheduled message sent every Friday for PS to fill out any PM
Tickets that should be shared with the CSM as a heads up! (This should be different from
tickets already escalated to the CSM).
#ps-asm-escalate - Used for PS to submit cases they need assistance with, do not know
the answer, are not sure the next steps to take, etc.
ASM Ticket Escalate - Workflow for when any ticket that PS has questions on or gets
stuck on. Escalation point to the PSM.
#ps-dd-sddi - Used for PS and CSM to submit WOs to be deep dived by PS.
DD Request - Workflow for all deep dives needed; complaints regarding delays, missed
appointments, confusion on automation, invoice disputes, etc.
SDDI Request - Workflow for dives for negative WO Reviews, CSATs, Google Reviews,
etc.
#pd-cn-actionset - Used for PS and CSM to submit pre-dispatch instructions, critical notes
and action sets for review, approval and execution.
PD and CN Request - Workflow for any pre-dispatch instructions or critical note that a
PM, Owner or SP wants added to their account. Submit for review and approval.
Action Set Request - Workflow for any action set that a PM wants added to their
account.
#csm-escalation - Used for ASM Tickets that PS needs to escalate to the CSM to handle.
PS Escalate to CSM - Workflow for cases that have been deemed scenarios that require
escalation to the CSM to handle.
For reference: Product Support Team vs Customer Success Managers
#ps-cs-sync - Used a cross functional PS <> CS comms channel; not to be in place of the
formal official #csm-escalation channel. This is just used to confirm details with the CSM,
ask leadership advice for how to handle a situation, give a quick “heads up” to the CSM,
etc.
