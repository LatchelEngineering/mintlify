# How to Handle: Feature Requests

How to Handle: New Requests
When a new feature is requested by a property manager, PS should gather as many as details
1.
as possible from the PM. Such as:
What is the problem we are solving for? We want to focus on the problem vs the potential
solutions. PMs are notorious for requesting an update or feature without sharing the “WHY”
or “What is happening?” that has led to this request. We need to understand what we are
working to solve for.
Solution Recs are great! And this is valuable to gather from the PM too.
Specific examples - Work Orders, Email template, SMS templates, etc.
Based on the request, review and follow if applicable:
Job Aid: Top PM Requested Action Sets
SOP: Pre-Dispatch Instructions & Critical Notes
If request cannot be achieved via an Action Set or PD/CN, PS should send the Feature
2.
Request for PM Experience SF Template to the PM.
PS submit the request in the #feature-requests Slack channel via the Feature Request
3.
workflow.
Once submitted, copy the Slack Url and paste it to the Case Thread.
4.
PS move the case to Closed.
5.
The template with the CSAT Survey will send 24 hours after the case is moved into the
6.
Closed state.
How to Handle: Follow-Ups on Requests
PS should send the Follow-Up on Feature Request SF Template.
1.
If the PM pushes back and/or gets frustrated, PS should escalate the case to the CSM.
2.
If the PM is unresponsive to either email template, PS does not need to send the
Unresponsive Follow-Up Template. PS can simply close the case as Closed Unresponsive.
