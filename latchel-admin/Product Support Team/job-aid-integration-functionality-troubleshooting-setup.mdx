Discrepancy of Work Orders Syncing from Appfolio to Latchel
Description: Work orders are in Appfolio are not syncing into Latchel.
Reconciliation Steps:
Identify Integration Sync Method:
1.
Review and confirm the Property Manager's Integration Sync Method
under the Miscellaneous section in Account Settings.
Here is how each setting option impacts which work orders sync from
Appfolio to Latchel:
Assigned Work Orders: Only WOs assigned to a user with the name
that includes “Latchel” will sync from Appfolio to Latchel. All Latchel
work orders will be created in Appfolio but only WOs that are
assigned to the Latchel user in Appfolio will be updated from
Appfolio. This is used by some customers who want the work order
created but subsequently want to make changes without them being
overwritten but a future update.
All Work Orders: All WOs in the Appfolio account will sync to
Latchel, all WOs in Latchel sync to Appfolio
No Work Orders: No WOs in the Appfolio account will sync to
Latchel, No WOs created in Latchel will sync to Appfolio
Check Last Sync:
2.
Run the query outlined in the Standard Operating Procedure to
determine when the last synchronization occurred.
Check Plugin Version via SQL Query.
3.
Run query in pgAdmin to see whar version the PM is on. Cross
reference with Chrome store version.
Report a Problem:
4.
If the version is up-to date, integration sync method setting is
configured correctly, and last sync was recent but the issue persists,
have the PM use the "Report a Problem" feature in their plugin.
Escalate the Issue:
5.
If applicable, submit issue to the #tech-high-sev-escalation channel for
urgent attention.

Unexpected Property Reactivation During Synchronization
Description:
Property is manually deactivated.
1.
Synchronization process executes.
2.
Property is unexpectedly reactivated.
3.
Reconciliation Steps:
Check reports
1.
Access the Integration Report
Navigate to the Customerʼs Integration Tab in Latchel.
Download the property report.
Locate the property in the report.
If the property is marked as active (identified by a 1 integer), it should
be active in Latchel.
If the Property or Resident is NOT in the Report:
This indicates one of the following scenarios:
Latchel doesnʼt have access to the record.
The record is hidden in AppFolio (e.g., due to filters or permission
settings).
The property or resident is deactivated in AppFolio.
Important Synchronization Limitation
Latchel only syncs status changes from active to inactive.
If a previously inactive property is reactivated in AppFolio, this change
will NOT be reflected in Latchel.
In these cases, the property must be manually re-added or activated in
Latchel if needed.
Next Steps Based on Findings:
If the property should be active in Latchel but isnʼt: activate it.
If the property should NOT be active in Latchel but is: disable it.
If the record is missing from the report, instruct the PM to verify
visibility and access settings in AppFolio.
Report a Problem:
2.
If the version is up-to date and the issue still persists, have the PM use
the "Report a Problem" feature in their plugin.
Escalate the Issue:
3.
Submit issue to the #tech-high-sev-escalation channel for urgent
attention.

5. Troubleshooting Queries
SQL Query to see the last time the Integration (any integration) was run successfully:
select i.company_id, c.name, i.type, i.status, i.total_rows, i.created_at,
i.source
FROM public.import i
JOIN public.company c
ON i.company_id = c.company_id
WHERE i.company_id = ####
ORDER by i.created_at desc
LIMIT 10
SQL Query to see what version of the Latchel plugin the PM is running (Appfolio clients):
SELECT al.created_at, al.plugin_version
FROM appfolio_log al
WHERE al.pm_company_id = ####
ORDER BY al.created_at DESC
LIMIT 1
SQL Query that shows big red flags for duplicate work orders. The query should produce zero
results. If it outputs results, escalate to #tech-high-sev
SELECT j.ref_job_id, j.created_at, i.settings-
>>'last_sync_time' as last_sync_time FROM job j
JOIN property p ON p.property_id = j.property_id
JOIN company c ON c.company_id = p.pm_company_id
JOIN integration i ON i.pm_company_id = c.company_id
JOIN pm_company_settings pcs ON pcs.pm_company_id =
c.company_id
WHERE c.company_id = 549 AND j.ref_job_id IS NULL AND
j.created_at < DATE(i.settings->>'last_sync_time') AND
j.created_at > pcs.integration_sync_date;
