## What is the Product Release?

Latchel Users can now see a transcript of all SMS and Latchel Web Support chats that are tagged to a work order on the work order detail page.

<Frame>
  ![Work order transcripts overview](/images/Eng-153.png)
</Frame>

## What was the Previous State?

Previously, there was no way to reference historical SMS or Latchel Web Support chats related to a specific work order.

## Who Is Impacted?

PMs, Latchel Admins, and Latchel Ops can all access these transcripts.

## When Is It Relevant?

The transcripts can be accessed on a specific work order detail page. It's important to note that if an SMS or Latchel Web Support conversation is not tagged to a specific work order, it will not show up in the transcripts.

<Warning>
  It's important to note that the transcripts are meant to be a historical reference and are read only. If you'd like to have a live SMS or Latchel Web Support conversation with somebody then you will need to start the SMS or Latchel Web Support using the existing methods today (i.e. SMS button on work order page/person's profile page/etc. or the messaging icon in the home navigation bar for Latchel Web Support).
</Warning>

## What Are Some Risks or Ways Things Could Go Wrong?

SMS or Latchel Web Support conversations will not show up in the work order's transcripts unless it has been correctly tagged to that work order. In addition, if there are any conversations that are incorrectly tagged, then they will not make sense in relation to the work order.

Please reference these articles for more details on how to correctly tag:

- PM SMS Tagging - [Filter SMS by Portfolio & PM SMS Tagging](https://latchel.atlassian.net/wiki/spaces/PROD/pages/2678947841)
- Service Ops SMS Tagging - [Service Ops SMS Tagging & PM 360 Degree Communication View](https://latchel.atlassian.net/wiki/spaces/PROD/pages/2613182465)

## When Will This Change Be Effective?

SMS transcripts was deployed on 6/27 and Latchel Web Support transcripts will be deployed on 11/30.
