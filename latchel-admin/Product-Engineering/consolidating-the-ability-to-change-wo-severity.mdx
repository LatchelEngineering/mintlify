title: "Consolidating the Ability to Change WO Severity"
---

## What is the Product Release?

We have consolidated the ability to change a WO's severity within the Take Action drop down. With this, there is only a "Change Severity" option.

<Frame>
  ![Take Action drop down with Change Severity option](/images/Eng-46.png)
</Frame>

## What was the Previous State?

Previously, the ability to change the severity lived in two different places:

1. "Escalate to an Emergency" through Take Action Drop Down - This was also problematic because we would show this option even when the work order was already in Emergency severity.

<Frame>
  ![Escalate to an Emergency option in Take Action drop down](/images/Eng-47.png)
</Frame>

2. Severity Drop Down - While this drop down is more intuitive for the User, it did not contain any logic other than changing the severity label for the work order.

<Frame>
  ![Severity drop down](/images/Eng-48.png)
</Frame>

In order to reduce confusion and prepare for the WO 2.0 deployment, we've cleaned it up so that it's consolidated under a new "Change Severity" option in the Take Action drop down. This was the option that was easiest/fastest to deployment.

## Who Is Impacted?

Only Latchel Admin users are impacted. PMs already do not have the ability to change severity today so we kept the scope to only Ops for this deployment. However, we plan to release a more user friendly design for PMs at a later date.

## When Is It Relevant?

Any time Latchel Ops needs to change the severity of a work order.

## How Is It Used?

The "Change Severity" option is dynamic and only shows certain options based on the current severity of the work order. See demo in this [loom](https://www.loom.com/share/1144db68f44c4d99bdf0711f0a74c03a).

1. When a work order is **Normal**, Latchel Ops will have the option to:

- Escalate to an Emergency - This follows today's workflow for when Latchel Ops chooses "Escalate to Emergency" under the Take Action drop down.
- Escalate to Urgent - This simply changes the severity label of the work order to Urgent.

2. When a work order is **Urgent**, Latchel Ops will have the option to:

- Escalate to an Emergency - This follows today's workflow for when Latchel Ops chooses "Escalate to Emergency" under the Take Action drop down.
- De-Escalate to Normal - This simply changes the severity label of the work order to Normal.

3. When a work order is in **Emergency**, Latchel Ops will have the option to:

- De-Escalate to Urgent - This simply changes the severity label of the work order to Urgent.
- De-Escalate to Normal - This simply changes the severity label of the work order to Normal.

<Warning>
  Please note that Latchel Ops will not have the option to change severity until after a work order has been moved out of Troubleshooting.
</Warning>

## What Are Some Risks or Ways Things Could Go Wrong?

There is little to no risk with this deployment. It takes all the workflows that Latchel Ops uses today and just consolidates it into one place.
