## Introduction

To ensure a swift and efficient response to critical incidents, it's crucial to understand when to post an issue in the #tech-high-sev-escalations channel. This guide will assist in determining when an issue merits escalation, facilitating engagement with an on-call engineer.

## Definition

An incident is considered high severity when it meets the following criteria:

1. **Customer Impact -** The issue must affect customers at a functional level. This means the problem should not merely be cosmetic or typo-related, but something that directly hampers the user's experience or their ability to perform necessary actions within the system.
2. **Multi-Customer Impact -** The incident must impair more than one customer, indicating a wider implication and a higher urgency to resolve the problem. \*Exception: For verified **integration or Latchel data sync** issues, report directly to #tech-high-sev-escalations.
3. **Affected Functionality -** The functionality affected is in one of the following areas:
   - Integrations or plugin sync
   - Work Order (WO) creation
   - Assigning and scheduling a service provider or resident
   - Communication features
   - Invoicing or payment features
   - System running exceptionally slow or not responding
   - Access to resident benefits

<Note>
  This definition is continuously evolving and not perfect! Use your best judgment. If business-critical functionality is impacted, don't hesitate to ask for help! If you can't get a timely response, it's acceptable to create a high severity escalation, provided you've thoroughly read and understood this guide.
</Note>

## When to post an issue in the High Severity Channel

You should post an issue in the #tech-high-sev-escalations channel when:

- You encounter an incident that fulfills the high severity communication criteria outlined above.
- You are reasonably confident that the issue is not expected functionality (a "bug" rather than a feature request).
- The immediate resolution of the incident is crucial for maintaining customer satisfaction or safeguarding essential system functionality.

## How to post an issue in the High Severity Channel

To ensure the on-call engineer has all the necessary information to address the issue effectively, please use the shortcut in the #tech-high-sev-escalations and fill out the required fields.
