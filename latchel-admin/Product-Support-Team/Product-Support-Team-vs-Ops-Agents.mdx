### **What’s the difference between Product Support and Ops Agents?**

\*\*Product Support (PS)\*\*manages issues involving product knowledge, platform behavior, and helping property managers (PMs) update workflows to meet their goals. PS addresses work order escalations, Deep Dive submissions, invoice disputes, feature requests, and service issues, focusing on functional product problems and certain business outcomes. PS also handles initial responses, troubleshooting, and gathering additional details on customer support issues before escalating to ensure the Customer Success Manager (CSM) receives comprehensive information.

**Ops Agents**handle straightforward requests and account updates. Any request beyond a basic Tier 1 issue will be escalated to PS for more specialized assistance.

---

Below is a summary of the roles of Product Support and Ops Agents; and**how you can tell the two apart**:

| **Product Support** | Involves straightforward assistance with technical problem solving and troubleshooting. |
| --- | --- |
| Answering customer questions about the product. |  |
| Fulfilling specific customer requests. |  |
| Looks into isolated interactions such as work orders, reviewing calls and dispatch attempts and configuration setting issues. |  |
| Subscription questions and system functionality. |  |
| Account Configurations. |  |
| Focuses on immediate issues (short-term view). |  |
| **Ops Agents** | Assists customers in updating contact information. |
| Basic settings listed below and account updates. |  |
| Assist SPs in making updates to their profiles. |  |
| Escalating to PS more complex product questions and account updates. |  |

Follow these steps: 1. Check if the Property Manager contacting us has an active user profile associated with the account.   1. If the requester does not have an active profile, do not proceed. 2. Ask the requester to confirm the email address where they expect to receive the login information or password reset link.   1. Cross-check this email address against the one listed on the user’s profile in the system.   2. If the email addresses do not match, do not send any login information. Advise the requester that, for security reasons, login details can only be sent to the email address on file. Only proceed with sending the login information once both points have been confirmed. |
| [Enable](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/How-to-Re-Enable-a-property#how-to-re-enable-a-property) / [Disable](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/How-to-Remove-Properties) Properties |
| [Enable / Disable Residents](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/How-to-Enable-or-Disable-a-Resident-in-Latchel#how-to-enable-or-disable-a-resident-in-latchel) |
| Add or Remove [Escalation Contacts](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/How-to-Update-Your-Escalation-Contacts#how-to-update-your-escalation-contacts) ONLY if the below scenario applies: - Upon request from the PM, the Ops Agent should ask: Does this change apply to ALL properties on the account? - If YES, the Ops Agent may proceed with the changes. - If NO, escalate to PS.   - For On-call related WOs, escalate to Product Support. |
| Add, Update, or Edit a property address ONLY if the Property Manager is NOT on an active integration. - How to Check if a PM is on an integration:   - Review the PM’s integration status and confirm whether it is **Active**, **Inactive**, **Active but Paused**, or whether the **Property Modification** and **Resident Modification** settings are enabled. Refer to [How to: Check and Handle Property Management Company Integration Status](https://latchel.atlassian.net/wiki/spaces/OPS/pages/3084845060/How+to+Check+and+Handle+Property+Management+Company+Integration+Status)   - **Note:** The example shown represents a Property Manager with **no active integration**. <img src="attachments/2977234945/3017539592.png?width=742" width="742" /> |

Below are settings that Ops Agents can handle without escalating to PS (anything outside of the below should be escalated to PS):

| **SP Requests that Ops Agents can handle:** |
| --- |
| LCN/LVN Service Providers: - Update and/or Add Skills - Update/Add Contact Info - Add SP User to Account   - When on the SP Company page, scroll to Service Provider Status block to see if the SP is part of the Latchel Service Provider Network (LCN).   - If the toggle is ON, then the Ops Agent may proceed in updates. <img src="attachments/2977234945/3143991315.png?width=368" width="368" /> |
| Preferred Services Providers: - Update/Add Contact Info - Add SP User to Account - For requests in adding a preferred service provider, refer to [How To: Add a New Service Provider](https://latchel.atlassian.net/wiki/spaces/OPS/pages/794034545/How+To+Add+a+New+Service+Provider). |
