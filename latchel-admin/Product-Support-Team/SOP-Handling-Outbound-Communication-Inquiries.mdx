<Info>
  This document outlines the procedures for handling outbound communications by our team, focusing on requests, complaints, and issues/bugs.
</Info>

### How to Handle: Requests

_For requests like "Call me for X" or "SMS me when X happens":_

1. **Standard System Communications**:
2. Utilize existing user group configurations for predefined communication preferences.
3. [**Create User Groups**](https://help.latchel.com/s/article/How-to-Create-User-Groups)\*\*:\*\*Set up user groups when necessary to manage specific communication preferences efficiently.
4. [**Emergency Escalation Contacts**](https://help.latchel.com/s/article/How-to-Update-Your-Escalation-Contacts)\*\*:\*\*Refer to emergency escalation contacts for urgent requests that fall outside the norm.
5. **Pre-Dispatch Instructions & Critical Notes**: Ensure reference to critical notes using the pre-dispatch instructions here: [https://latchel.atlassian.net/wiki/x/AYD\_pw](https://latchel.atlassian.net/wiki/x/AYD_pw)
6. **Additional Non-Standard Communications Solutions**: Implement or adjust action sets as needed: [<u>Action Sets</u>](https://latchel.atlassian.net/wiki/x/AoA-tg).
7. **Unique Complaints Not Covered:?**
8. Follow the escalation protocol on [https://latchel.atlassian.net/wiki/x/B4BKvQ](https://latchel.atlassian.net/wiki/x/B4BKvQ) .

### How to Handle: Complaints

_For complaints such as "Too often" or "Not enough" communications:_

1. \*\*Initial Contact:\*\*Email the customer using [the Outbound Comm Complaint SF Template](https://latchel1.lightning.force.com/lightning/r/EmailTemplate/00XUS00000CgFzp2AF/view) to gather details.
2. \*\*Response Procedure:\*\*Based on customer feedback, proceed with one of the following:
3. Refer to steps 1-2 in the “How to Handle: Requests” section above.

### How to Handle: Issues/Bugs

_For issues like "Not getting notified when X happens":_

1. **Troubleshooting Steps:**
2. For email issues, refer to troubleshooting: [<u>Email Troubleshooting</u>](https://latchel.atlassian.net/wiki/x/AYA5kQ).
3. For SMS issues, follow these steps: [<u>SMS Troubleshooting</u>](https://latchel.atlassian.net/wiki/x/PwBQvw).
4. **Escalation**:
5. If troubleshooting does not resolve the issue, report the bug in the #bug-report Slack Channel using the Bug Report workflow.
