# Purpose

<Info>
  **Purpose of this document**

  The purpose of this document is to align PS CS Auditors on how to assess the Product Support Associates' interactions with our customers. This document also serves as a reference for Product Support to understand Latchel’s expectations and provide an excellent customer experience.
</Info>

<Info>
  **Other resources to review:**

  - [SOP: Product Support vs Customer Success Managers](https://latchel.atlassian.net/l/cp/vtkv8BA4)
  - [PS CX Tenets](https://latchel.atlassian.net/l/cp/yr81an1F): Speed, Accuracy and knowledge, Clarity, Empathy, Transparency, Seek Help & Escalate, Full Resolution.
  - [Case Types & Motives Explained](https://latchel.atlassian.net/wiki/x/HADnlg)
</Info>

# Content

### Process Overview

Auditors should follow these guidelines while completing [this form](https://docs.google.com/forms/d/e/1FAIpQLSd_G_7r_l4KI7rAoMDZ6Ib4YSPlnOV9LYh17nPo8MS3kscuvA/viewform) for the PS CX Weekly Audit.

| **Question ** | **Guidelines ** | **Tips & Remarks ** |
| :-- | :-- | :-- |
| Product Support Associate | Select the name of the agent you are reviewing. |  |
| Auditors Name | Select your name. |  |
| Case URL Link | Paste the link to the Salesforce Case. |  |
| Case ID | Paste the Case Number ID found in the Case Details box. |  |
| Case Type | Select the Case Type that was chosen for the case. |  |
| Case Motive | Select the Case Motive that was chosen for the case. |  |
| Did PS use the correct Case Type/Motive? | The agent should have selected the accurate case type/motive combination based on the [<u>SOP: Case Types & Motives Explained</u>](https://latchel.atlassian.net/l/cp/x1n0uuWw "https://latchel.atlassian.net/l/cp/x1n0uuWw") |  |
| If NO, what would have been more appropriate? | Review the main topic of conversation from the case thread and share what would have been more appropriate based on the SOP. | If you feel there is not an option available/an idea of additional case type/motive, please DM Maddie to discuss! [<u>SOP: Case Types & Motives Explained</u>](https://latchel.atlassian.net/l/cp/x1n0uuWw "https://latchel.atlassian.net/l/cp/x1n0uuWw") |
| Did we lead with Options for the PM vs a solution? | We want to provide our PMs with the best practices and options along with any concerns/caveats that are good to know with the desired outcome. | Don’t solve or take the PMs word for it - We want to solve the overall problem and/or answer the question but we want to ask clarifying questions and serve as the PMs advisor. _\*Best Practice Templates coming to Confluence. _ |
| If escalation was required to another department, was a link to the escalation documented in the case thread (Slack link, JIRA link, etc.)? | The agents should always document where their questions, escalations and workflows are submitted. | You will see these links as a Post in the case thread. |
| If escalation was required via Slack, was the correct channel/workflow used? | The agents should be knowledgeable about the different resources and workflows available to them. They should be asking questions/escalation situations through the proper Slack channels through the workflow. | See here: [<u>Job Aid: Slack Channels & Escalation Workflows</u>](https://latchel.atlassian.net/l/cp/0Dzs3wf5 "https://latchel.atlassian.net/l/cp/0Dzs3wf5") |
| Did PS consistently show empathy and "treat the PM as a friend" throughout the conversation? Did it feel personal and not robotic or scripted? | Begin your case with “Dear/Hi/Hello \[customer name\],” Start the body of your email with a warm greeting such as “I hope this email finds you well.” Acceptable verbiage includes: <ul><li>“Thank you for reaching out!”</li><li>“We appreciate you reaching out to us - We are happy to help!”</li><li>“We apologize for this inconvenience.”</li></ul> | If leveraging ChatGPT or other AI resources, did the agent ensure that the responses still felt personal? Not like a robot? |
| Was PS clear and concise in their responses using the appropriate grammar and spelling, writing complete sentences? | PS provided a solution or answer in a short response that was easy for someone without maintenance expertise to understand. PS agent answered the question(s) asked. | Was the response clear with key points while retaining customer obsession? We want to remove any added “noise” or “fluff” while still treating the PM as a friend. |
| Did PS provide the accurate help guide and give a brief summary of the process? (Use N/A if a help guide was not needed). | PS should be knowledgeable on the Help Guide Directory available to them. They should help assist the PM while provided a Help Guide to help make the PM self-sufficient. | [<u>Published Help Article Directory HERE</u>](https://latchel1.lightning.force.com/lightning/o/Knowledge__kav/list?filterName=00B5e00000MxX7eEAF "https://latchel1.lightning.force.com/lightning/o/Knowledge__kav/list?filterName=00B5e00000MxX7eEAF"). Latchel Help Center: [https://help.latchel.com/s/](https://help.latchel.com/s/) |
| Did PS set the correct expectations and next steps? | PS should share next steps with the PM so they feel informed and what to expect next. | <ul><li>Are they submitting a DD?</li><li>Will finance be reaching out with an updated invoice?</li></ul> |
| Did PS give the PM consistent follow-ups and share the progress of their request when needed? Did they keep the PM in the loop throughout the lifecycle of the case? | Everytime the case re-opens, the PS should be sending a follow up with either: A) An update to the PM on the situation (if it was escalated/reported to another department)\> B) Or this [<u>Follow Up template</u>](https://latchel1.lightning.force.com/lightning/r/EmailTemplate/00XUS000002Tryp2AC/view?queryScope=userFolders "https://latchel1.lightning.force.com/lightning/r/EmailTemplate/00XUS000002Tryp2AC/view?queryScope=userFolders") if the PM was unresponsive to our original message. | It is important for the PM to feel informed! Is PS consistently following up with the PM so they feel in the loop? Did PS follow up on the Latchel side for an answer? |
| Did PS use the Follow-Up template when reaching out to the PM (instead of copying and pasting the original message)? | Instead of copying and pasting the original message full of details, the agent should be doing a quick follow-up response. | _\*Case SOP coming to Confluence. _ |
| Did PS fully answer the PMs question? Did PS ask clarifying questions (if needed) to ensure they answered the PM accurately? | <ul><li>Did PS ask clarifying questions?</li><li>Did PS get the full scope of the PMs question **without** assuming?</li></ul> | We never want to assume we know what the PM is talking about. The PS Agent should ask the PM to elaborate when needed to ensure we are providing the best recommendation. |
| Was the answer/resolution that PS provided accurate? Did PS provide the best resolution? | Did PS paint the full picture and close the loop with the PM? We PS to always be explaining and sharing with the PM, “If you request X, then Y and Z will happen. With the outcome…” | PS should be able to share the outcomes and any concerns/caveats that come along with the workflow. |
| Did PS send this “PM Close Case - Send Survey Link” Template that includes the Typeform Survey link when closing the case? | Regardless of if the conversation was positive or negative, we want to always close the case with this template which includes a link to the [<u>PS CSAT Survey Typeform</u>](https://latchel.typeform.com/to/TjsKozdo "https://latchel.typeform.com/to/TjsKozdo"). | We can learn just as much (maybe even more) from negative surveys as positives! All survey responses are captured [<u>here</u>](https://docs.google.com/spreadsheets/d/1BwoOI4tACxCvWul5PwrnNj8-xfqHm1B-P-bwK-8Mpz8/edit#gid=1271016591 "https://docs.google.com/spreadsheets/d/1BwoOI4tACxCvWul5PwrnNj8-xfqHm1B-P-bwK-8Mpz8/edit#gid=1271016591"). |
| What was great about this interaction? | <ul><li>Let the agent know what they did great!</li><li>The agent might have done something not currently covered in our training. If it was helpful, please note it so we can consider sharing with the rest of the team.</li></ul> |
| Was there anything we could have done better? Please provide any other feedback and improvement opportunities! | <ul><li>The reviewer should note anything observed that could lead to a PM complaint.</li><li>For any questions that receive a “No” answer, provide tips and suggestions or sample verbiage to the agent on how they can improve to receive a “Yes” next time.</li><li>Questions answered "Yes" may also have areas of improvement. Note any tips or suggestions in this field too.</li></ul> |
