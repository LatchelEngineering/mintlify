<Info>
  This job aid outlines the steps for setting up, managing, and troubleshooting the integrations between Appfolio, Yardi, Propertyware, and Rent Manager. It ensures consistency in how these integrations are handled across different teams and projects.

  Applies to all personnel involved in setting up and managing integrations across the various platforms. It covers:

  - Initial setup of new connections
  - Monitoring and maintaining integration functionality
  - Troubleshooting common issues
</Info>

<img src="https://latchel.atlassian.net/wiki/images/icons/grey_arrow_down.png" />Discrepancy of Work Orders Syncing from Appfolio to Latchel

**Description:** Work orders are in Appfolio are not syncing into Latchel.

| **Reconciliation Steps:** |
| --- |
| 1. **Identify Integration Sync Method:** Review and confirm the Property Manager's Integration Sync Method under the Miscellaneous section in Account Settings. Here is how each setting option impacts which work orders sync from Appfolio to Latchel: - **Assigned Work Orders:Only WOs assigned to a user with the name that includes “Latchel” will sync from Appfolio to Latchel. All Latchel work orders will be created in Appfolio but only WOs that are assigned to the Latchel user in Appfolio will be updated from Appfolio. This is used by some customers who want the work order created but subsequently want to make changes without them being overwritten but a future update. - All Work Orders: All WOs in the Appfolio account will sync to Latchel, all WOs in Latchel sync to Appfolio - No Work Orders: No WOs in the Appfolio account will sync to Latchel, No WOs created in Latchel will sync to Appfolio 2. [Check Last Sync](https://latchel.atlassian.net/wiki/spaces/SR/pages/3175776273/Job+Aid+Integration+Functionality+Troubleshooting+and+Setup+Steps#5.-Troubleshooting-Queries):** Run the query outlined in the Standard Operating Procedure to determine when the last synchronization occurred. 3. **Check Plugin Version via[SQL Query](https://latchel.atlassian.net/wiki/spaces/SR/pages/3175776273/Job+Aid+Integration+Functionality+Troubleshooting+and+Setup+Steps#5.-Troubleshooting-Queries).** Run query in pgAdmin to see whar version the PM is on. Cross reference with Chrome store version. 4. **Report a Problem:** If the version is up-to date, integration sync method setting is configured correctly, and last sync was recent but the issue persists, have the PM use the "Report a Problem" feature in their plugin. 5. **Escalate the Issue:** If applicable, submit issue to the #tech-high-sev-escalation channel for urgent attention. |

| **Suggested Scripting:** |
| --- |
| **Assigned Work Orders Method Sync** Hi \[PM\], I hope you're doing well! I wanted to clarify how your current AppFolio-Latchel sync settings work to ensure everything is functioning as expected. This setup allows you to have work orders created automatically but make changes in AppFolio without them being overwritten by future sync updates. Could you let me know if this integration is operating as expected on your end, or if you're noticing any discrepancies? If Plugin is out of date, include steps to reinstall below **(remove this step if the most recent version is installed)**: Lastly, I noticed you're on an old version of the plugin. Please reinstall the latest version by following these steps: - Log out of AppFolio. - Remove the current Latchel Data Sync extension and refresh the page. - Reinstall the latest version using this guide: [How to Install the Latchel Data Sync Extension](https://help.latchel.com/s/topic/0TO5e000000h9wqGAA/How-to-Install-the-Latchel-Data-Sync-Extension#how-to-install-the-latchel-data-sync-extension).. - Log into the Latchel Data Sync extension using your Latchel credentials. - Log back into AppFolio and try running the sync again. <img src="https://latchel.atlassian.net/wiki/images/icons/grey_arrow_down.png" />Unexpected Property Reactivation During Synchronization

**Description:**

1. Property is manually deactivated.
2. Synchronization process executes.
3. Property is unexpectedly reactivated.

| **Reconciliation Steps:** |
| --- |
| 1. **Check reports** Access the Integration Report Navigate to the Customer’s Integration Tab in Latchel. Download the property report. Locate the property in the report. If the property is marked as active (identified by a 1 integer), it should be active in Latchel. If the Property or Resident is NOT in the Report: This indicates one of the following scenarios: - Latchel doesn’t have access to the record. - The record is hidden in AppFolio (e.g., due to filters or permission settings). - The property or resident is deactivated in AppFolio. **Important Synchronization Limitation** Latchel only syncs status changes from active to inactive. If a previously inactive property is reactivated in AppFolio, this change will NOT be reflected in Latchel. In these cases, the property must be manually re-added or activated in Latchel if needed. **Next Steps Based on Findings:** If the property should be active in Latchel but isn’t: activate it. If the property should NOT be active in Latchel but is: disable it. If the record is missing from the report, instruct the PM to verify visibility and access settings in AppFolio. 2. **Report a Problem:** If the version is up-to date and the issue still persists, have the PM use the "Report a Problem" feature in their plugin. 3. **Escalate the Issue:** Submit issue to the #tech-high-sev-escalation channel for urgent attention. |

| **Suggested Scripting:** |
| --- |
| I'm reaching out to address the issue with properties reactivating unexpectedly after being manually disabled. We're providing a quick overview of how property status sync works to help you understand the current limitations and behavior of the system. This information is important because it explains _why_certain changes, like reactivating a property in your property management system, might not reflect in Latchel. - Latchel can only change properties from active to inactive. - If you reactivate a previously inactive property in Appfolio, this change won't be reflected in Latchel. - When a property is manually disabled in Latchel, any activation changes made in AppFolio won't update in Latchel. Thank you for your patience and understanding. I'm here to assist you every step of the way. |

## **5. Troubleshooting Queries**

SQL Query to see the last time the Integration (any integration) was run successfully:

| select i.company\_id, c.name, i.type, i.status, i.total\_rows, i.created\_at, i.source FROM public.import i JOIN public.company c ON i.company\_id = c.company\_id WHERE i.company\_id = #### ORDER by i.created\_at desc LIMIT 10 |
| --- |

SQL Query to see what version of the Latchel plugin the PM is running (Appfolio clients):

| SELECT al.created\_at, al.plugin\_version  FROM appfolio\_log al  WHERE al.pm\_company\_id = ####  ORDER BY al.created\_at DESC  LIMIT 1 |
| --- |

SQL Query that shows big red flags for duplicate work orders. The query should produce zero results. If it outputs results, escalate to #tech-high-sev

| `SELECT j.ref_job_id, j.created_at, i.settings->>'last_sync_time' as last_sync_time FROM job j JOIN property p ON p.property_id = j.property_id JOIN company c ON c.company_id = p.pm_company_id JOIN integration i ON i.pm_company_id = c.company_id JOIN pm_company_settings pcs ON pcs.pm_company_id = c.company_id WHERE c.company_id = 549 AND j.ref_job_id IS NULL AND j.created_at < DATE(i.settings->>'last_sync_time') AND j.created_at > pcs.integration_sync_date;` |
