## Case State Automation:

| **Grouping Category** | **Case State** | **Description** | **Goes back to Needs Response after 24 Hours (if no response from Customer or update from the escalated team).** | **Goes back to Open after 48 Hours (if no response from Customer).** | **Goes to Needs Response when a new email response is received.** | **Goes to Open.** |
| --- | --- | --- | --- | --- | --- | --- |
| Active | New | Brand New Cases that have not been viewed or triaged by PS yet. | - | - | - | - |
| Open | Cases that have been responded to by PS and have an ongoing conversation happening. | - | - | - | - |
| Waiting for Customer | Case that needs a response from the customer. PS sent the last reply and is waiting for a response. | - | - | - | - |
| Pending | Escalated | Cases that have actions in progress and a concern/question has been escalated to another Latchel Department. To be used with PS is waiting on a response/next steps from Latchel Department. | - | - | - | - |
| *Cases moved to the CS Team bucket | Cases that are escalated to the CSM or PSM based on the concern/issue in question. More details here:  [https://latchel.atlassian.net/wiki/x/VoDpnQ](https://latchel.atlassian.net/wiki/x/VoDpnQ)  [https://latchel.atlassian.net/wiki/x/VQDamw](https://latchel.atlassian.net/wiki/x/VQDamw) | - | - | - | - |
| On Hold | Cases that should not automatically go to Open but cannot be closed yet. | - | - | - | - |
| Scheduled Call | Can be used when a meeting has been scheduled (if used, typically by the CSM). PS does **not** use this state. | - | - | - | - |
| Complete | Closed - No Workflows | Used when the case has no actions needed. Use this for PS Escalations from Ops or SPAM emails. | - | - | - | - |
| Closed | Use this when completing the case when the customer is unresponsive. PS has sent the **Explanation - Unresponsive Follow-Up** or **Bias for Action - Unresponsive POC Follow-Up** and have not gotten a response, use this close the case. | - | - | - | - |
| Merged | Never move a case to this state manually. This only shows on the case state when merging cases together. | - | - | - | - |
